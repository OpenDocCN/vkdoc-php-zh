# 将 404 错误指向自定义错误页面

将 404 错误指向自定义页面非常有用，因为它可以向用户提供关于页面被删除原因的更多信息、与网站升级进度相关的更新，甚至提供一个搜索界面。结合 PHP 使用，这样的页面还能尝试识别用户试图访问的页面，并将其相应地转发；向网站管理员发送邮件，告知她已发生错误；创建自定义错误日志；或者执行你希望的任何其他操作。本节将演示如何使用 PHP 收集与缺失文件相关的一些统计信息，并将这些信息通过邮件发送给网站管理员。希望这个示例能为你提供一些思路，帮助你开始创建适合自己特定需求的自定义 404 处理程序。

> **注意：** 本章描述的某些概念已由 Apache Web 服务器的 URL 重写功能高效地处理。然而，请记住，许多读者使用共享服务器来托管网站，因此无法奢求对其 Web 服务器的行为拥有如此强大的控制权。尽管如此，此处描述的概念旨在鼓励读者在并非所有工具都可用的情况下，考虑替代解决方案。

##### 创建自定义 404 错误处理程序

在本示例中，你将创建一个脚本，该脚本会向网站管理员发送一封包含错误详细报告的电邮，并显示一条请求用户谅解的消息。首先，创建一个`.htaccess`文件，将 404 错误重定向到自定义脚本：

```
ErrorDocument 404 /www/htdocs/errormessage.html
```

如果你希望此行为在整个网站中生效，请将其放置在网站根目录。如果你不熟悉`.htaccess`文件，请参阅 Apache 文档以获取更多信息。

接下来，创建通过邮件通知网站管理员并显示适当消息的错误处理脚本。该脚本如清单 13-4 所示。

**清单 13-4.** *邮件通知与简单消息显示*

```php
<?php

// 服务器

$servername = $_SERVER['SERVER_NAME'];

$recipient = "webmaster@example.com";

$subject = "检测到 404 错误: ".$_SERVER['PHP_SELF']; 
$timestamp = date( "F d, Y G:i:s", time() );

$referrer = $_SERVER['HTTP_REFERER'];

$ip = $_SERVER['REMOTE_ADDR'];

$redirect = $_SERVER['REQUEST_URI'];

$body = <<< body

在 $timestamp 检测到 404 错误。

服务器: $servername

缺失页面: $redirect

来源文档: $referrer

用户 IP 地址: $ip

body;

mail($recipient, $subject, $body, "From: administrator\r\n");

?>

<h3>文件未找到</h3>

<p>

请谅解，我们的网站当前正在进行维护。

因此，您可能会偶尔遇到访问文档和/或服务的困难。

网站管理员已收到关于此事的详细事件日志邮件。

</p>

谢谢，<br />

网站团队
```

当然，如果你的网站特别庞大，你可能需要考虑将错误信息写入日志文件或数据库，而不是通过电子邮件发送。
