# 总结

在本章中，我们构建了面向操作的 API 的核心功能。我们通过以下方式建立了坚实的基础：

-   为操作输入和输出定义了清晰的模型
-   定义了一种创建操作服务的具体方式（`OperationInterface` 和 `OperationMetadata`）
-   实现了对可能实例化较重的操作进行懒加载
-   实现了一个用于存储操作的集中式集合，以及一个用于高效检索和执行操作的专用处理器
-   使用专用验证处理器验证操作数据，保护 API 免受无效载荷的攻击
-   精心设计了作为所有 API 请求入口点的端点，简化了用户交互和请求处理
-   结合 Symfony Monolog 包和 Symfony 事件系统，记录操作请求的接收和终止，以便日后分析 API 系统的健康状况

有了这些核心功能，我们的 API 就能够处理各种操作并提供所需的功能。然而，安全性仍然至关重要。在下一章中，我们将把重点转移到加强端点和单个操作的安全性上，以确保 API 环境的安全。
