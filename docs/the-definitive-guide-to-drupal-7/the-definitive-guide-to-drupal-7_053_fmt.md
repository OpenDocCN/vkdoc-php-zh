# 制作标签和替换标记表单

对于一个完全自定义的模块，你可以跳过设置表单，直接硬编码你希望对内容应用的处理。无需为管理员制作用户界面，也无需为其他模块制作 API。由于 Drupal 7 没有满足`DefinitiveDrupal.org`需求的包装标签替换方案，并且你可能希望管理员能够在不修改模块的情况下更改过滤器，你可以尝试制作一个同时包含 UI 和 API 的模块。

通常你会从 API 开始。在本例中，Drupal 为过滤器设置表单提供了 API，但没有为过滤器设置数据提供 API，因此你可以先实现表单，然后反向工作来理解 Drupal 如何处理数据。

每组标签和替换标记需要包含三部分数据：被替换的标签、替换该标签开放版本的标记，以及替换该标签闭合版本的标记。通过采用 HTML 闭合标签包含斜杠的约定，你可以只要求用户输入闭合标签，并通过移除斜杠来推导出开放标签。例如，表单可以接受`{{/pony}}`作为标签，然后在内容中，`{{pony}}`将被替换为表单中输入的开始标记，而`{{/pony}}`将被替换为结束标记。
