# 索引

索引与 PostgreSQL 性能调优有关，因此我们在此仅作简要介绍。

`索引` 是旨在提升数据库操作整体速度的数据库对象。索引基于这样一个假设：绝大部分数据库操作都是读取操作。索引能提升搜索操作的速度，但会降低插入、删除和更新操作的速度。通常，使用索引的好处远大于其弊端。

你可以在一个表上创建一个或多个索引，每个索引针对一列或一组列。当表在特定列上建有索引时，其行会根据该列的值和索引类型被索引或进行物理排列。这使得对该列的搜索操作非常迅速。例如，如果对 `department_id` 建立了索引，然后你搜索 ID 值为 934 的部门，那么搜索会执行得非常快。

索引的弊端在于，它们可能会拖慢添加新行或更新现有行的数据库操作，因为每次执行这些操作时，都必须更新索引（或重新排列表行）。

关于索引，你应记住以下几点：

- 索引能极大地加快数据库的搜索操作，但会降低更改数据库的操作（删除、更新和插入操作）的速度。

- 索引过多可能会降低数据库的整体性能。一般规则是，在 `WHERE`、`ORDER BY` 和 `GROUP BY` 子句中频繁使用，或用于表连接的列上设置索引。

- 默认情况下，系统会自动在主键表列上创建唯一索引。

你可以使用专用工具在压力条件下测试数据库在有特定索引和没有特定索引情况下的性能；事实上，一个严谨的数据库管理员在决定最有效的索引组合之前，会想要运行一些此类测试。
