### 转换数字格式

为了展示 PEAR 的强大功能，有必要提一个包，它很好地说明为什么在尝试解决任何重要的编程任务之前，你都应该定期查看代码库。虽然有些人可能认为选择这个特定的包有点奇怪，但它的目的是表明，即使对于你认为不太常见以至于不会有人开发出对应包的特别棘手的问题，也可能存在可用的包，因此不妨在代码库中搜索一下现有的解决方案。这个包是 `Numbers_Roman`，它使阿拉伯数字和罗马数字之间的转换变得轻而易举。

回到问题，假设你最近被雇来为一位电影制片人创建一个新网站。众所周知，任何严肃的制片人都使用罗马数字来表示年份，产品经理告诉你，网站上出现的任何日期都必须以此格式显示。花点时间思考一下这个需求，因为它并不像听起来那么容易实现。当然，你可以上网查找转换表并硬编码这些值，但你如何确保页面页脚中的网站版权年份始终是最新的呢？

当你正准备花一整个晚上编写代码时，你停下来思考一下是否有人遇到过类似的问题。“不可能，”你喃喃自语道，但花点时间搜索 PEAR 肯定是值得的。你导航过去，果然，遇到了 `Numbers_Roman`。

出于本练习的目的，假设 `Numbers_Roman` 包已经安装在服务器上。现在不必太担心这个，因为你将在下一节中学习如何安装包。那么，你如何确保当前年份显示在页脚中呢？通过使用以下脚本：

[www.it-ebooks.info](http://www.it-ebooks.info/)

```php
<?php

// 使 Numbers_Roman 包可用

require_once("Numbers/Roman.php");

// 获取当前年份

$year = date("Y");

// 将年份转换为罗马数字

$romanyear = Numbers_Roman::toNumeral($year);

// 输出版权声明

echo "Copyright &copy; $romanyear";

?>
```

对于 2005 年，此脚本将产生：

```
Copyright © MMV
```

这个故事说明了什么道理？即使你认为某个特定问题很冷门，其他程序员很可能也遇到过类似问题，如果你足够幸运，可能已经有一个现成的解决方案可供你使用。

### 安装和更新 PEAR

管理 PEAR 包最简单的方法是通过 PEAR 包管理器。这是一个命令行程序，提供了一个简单高效的界面来执行诸如检查、添加、更新和删除包以及浏览代码库中的包等任务。在本节中，你将学习如何在 Unix 和 Windows 平台上安装和更新 PEAR 包管理器。由于许多读者在共享托管服务提供商上运行网站，本节还将解释如何在不运行包管理器的情况下利用 PEAR。

##### PEAR 安装指南

##### 概述

`PEAR`已成为高效 PHP 编程中如此重要的一个方面，以至于自 PHP 4.3.0 版本起，其稳定发行版已包含在 PHP 发行包中。因此，如果你运行的是该版本或更高版本，请直接跳转到“更新 PEAR”章节。如果你在 Unix 上运行 PHP 4.2.X 或更早版本，或者使用 Windows 平台，安装过程非常简单，你很快就能学会。

##### Unix

在 Unix 上安装`PEAR`是一个相当简单的过程：从`http://go-pear.org/`网站获取一个脚本，并用 PHP 二进制文件执行它。打开终端并执行以下命令：

```
%>lynx -source http://go-pear.org/ | php
```

请注意，你需要安装`lynx`网页浏览器，这是 Unix 平台上一个相当标准的程序。如果你没有安装它，请搜索特定操作系统发行版对应的程序仓库，它肯定在那里。或者，你也可以直接使用标准 Web 浏览器（如 Firefox）访问上述 URL，保存获取到的页面，然后用 PHP 二进制文件执行它。

安装过程开始后，系统会提示你确认一些配置设置，例如 PHP 根目录和可执行文件的位置。你通常可以接受默认答案（方括号内提供的）而无需修改。在这一系列问题中，系统还会询问是否安装六个可选默认包。目前这是一个全选或全不选的方案。因此，如果你希望立即开始使用其中任何一个包，直接同意请求即可。

##### Windows

`PEAR`在 Windows 发行版中默认不安装。要安装它，你需要运行位于 PHP 发行版根目录下的`go-pear.bat`文件。该文件会安装`PEAR`命令、必要的支持文件以及前面提到的六个`PEAR`包。通过切换到 PHP 根目录并执行`go-pear.bat`来启动安装过程，命令如下：

```
%>go-pear.bat
```

系统会提示你确认一些配置设置，例如 PHP 根目录和可执行文件的位置。你通常可以接受默认答案（方括号内提供的）而无需修改。在这一系列问题中，系统还会询问是否安装六个可选默认包。目前这是一个全选或全不选的方案。因此，如果你希望立即开始使用其中任何一个包，直接同意请求即可。

安装过程结束后，会创建一个名为`PEAR_ENV.reg`的注册表文件。执行该文件将为许多`PEAR`特定的变量创建环境变量。虽然这不是必须的，但将这些变量添加到系统路径中，可以让你在 Windows 命令提示符下从任何位置方便地执行`PEAR`包管理器。

> **注意：** 执行`PEAR_ENV.reg`文件会修改你的系统注册表。尽管这种修改是无害的，但你仍应考虑在执行该脚本之前备份注册表。操作方法是：转到“开始”➤“运行”，执行`regedit`，然后通过“文件”➤“导出”导出注册表。

##### PEAR 与托管公司

如果你的托管公司不允许用户在其服务器上安装新软件，请不要担心，因为它很可能已经为最常用的包提供了至少基本的支持。如果`PEAR`支持不明显，请联系客户支持，询问他们是否考虑让某个特定包在服务器上可用。如果他们同意，那就万事大吉。如果他们拒绝你的请求，也不用担心，因为你仍然可以使用这些包，只需通过其他方式将它们包含在项目中即可。
