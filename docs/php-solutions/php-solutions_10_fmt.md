# 准备工作：开始使用 PHP

既然你已决定使用 PHP 来丰富你的网页，就需要确保拥有继续阅读本书后续内容所需的一切。虽然你可以在远程服务器上进行测试，但通常在本地计算机上测试 PHP 页面会更方便。你需要安装的所有软件都是免费的。在本章中，我将介绍 Windows 和 Mac OS X 的各种选项。Linux 系统通常默认安装了必要的组件。

本章涵盖的内容：

- 检查你的网站是否支持 PHP
- 为什么 PHP 5.4 应是最低版本
- 决定是否创建本地测试环境
- 在 Windows 和 Mac OS X 中使用现成的集成包
- 存储 PHP 文件的位置
- 检查本地和远程服务器上的 PHP 配置

## 检查你的网站是否支持 PHP

判断你的网站是否支持 PHP 最简单的方法是询问你的托管公司。另一种方法是将一个 PHP 页面上传到你的网站，然后查看它是否能正常运行。即使你知道网站支持 PHP，也请执行以下测试来确认运行的版本：

打开文本编辑器（如记事本或 TextEdit），在空白页面中输入以下代码：

```php
<?php echo phpversion(); ?>
```

将文件保存为`phpversion.php`。务必确保你的操作系统不会在`.php`后添加`.txt`扩展名。Mac 用户还应确保 TextEdit 不会以富文本格式（RTF）保存文件。如果你不确定，可以使用本书配套文件`ch02`文件夹中的`phpversion.php`。

像上传 HTML 页面一样，将`phpversion.php`上传到你的网站，然后在浏览器中输入 URL。假设你将文件上传到网站的根目录，URL 将类似于 [`www.example.com/phpversion.php`](http://www.example.com/phpversion.php)。

如果屏幕上显示一个三位数的版本号（如 5.6.1），说明你成功了：PHP 已启用。该数字告诉你服务器上运行的 PHP 版本。你至少需要 5.4.0 版本才能使用本书中的所有代码。

如果收到类似"Parse error"的消息，说明 PHP 支持，但你在文件中输入代码时出现了错误。请改用`ch02`文件夹中的版本。

如果你只看到原始代码，则说明 PHP 不受支持。

PHP 5.3 的官方支持已于 2014 年 8 月终止。如果你的服务器运行的是 PHP 5.3 或更早版本，请联系你的主机服务商，告诉他们你需要最新的稳定版 PHP。如果主机服务商拒绝，那就该更换托管公司了。

### 为什么 PHP 5.4 应是最低版本

作为一般原则，PHP 在次级版本之间（即仅版本号中第一个点之后的数字发生变化）会尽量保持向后兼容性。然而，PHP 5.4 移除了一些过时特性，并为数组引入了新语法。

尽管本书中的大部分代码在较旧版本的 PHP 上也能正确运行，但如果你的服务器仍依赖这些被移除的特性，你可能会得到意想不到的结果。影响本书代码的最重要变化是安全模式和魔术引号的移除。

安全模式常用于共享托管环境中。它的影响包括限制包含文件的位置以及可读写文件的权限。由于 PHP 5.4 移除了安全模式，这些限制不再存在。

魔术引号是一个错误的尝试，旨在通过在用户提交的数据中的引号前插入反斜杠，来让 PHP 对缺乏经验的开发者更安全。其目的是防止被称为 SQL 注入的恶意攻击。不幸的是，魔术引号引发的问题比解决的问题更多，经常使文本中充斥着不必要的反斜杠。如果你在 PHP 5.3 或更早版本上运行本书的代码，且魔术引号未被禁用，你就会遇到不必要的反斜杠。

本书中的代码还使用了数组的简化语法，这在较旧版本的 PHP 中无法运行。

不使用旧版 PHP 最重要的原因是安全性。当发现漏洞时，仅会对当前版本和之前两个版本提供安全更新。在撰写本书时，当前版本是 PHP 5.6。这意味着 PHP 5.4 和 5.5 将受益于任何安全更新。但一旦下一个版本发布，PHP 5.4 将不再修补安全威胁。使用最新版本的 PHP 不仅仅是获得最新功能的问题；它有助于保护你的网站和宝贵数据免受恶意攻击。

### 决定测试页面的地点

与普通网页不同，你不能直接在 Windows 文件资源管理器或 Mac 的 Finder 中双击 PHP 页面并在浏览器中查看。它们需要通过支持 PHP 的网络服务器进行解析或处理。如果你的托管公司支持 PHP，你可以将文件上传到你的网站并在那里进行测试。但是，每次修改后都需要重新上传文件。在早期阶段，你可能会发现由于代码中的一些小错误需要频繁上传。随着你经验的积累，你仍然需要频繁上传文件，因为你想要尝试不同的想法。

如果你想立即开始使用 PHP，完全可以把自己的网站当作测试平台。然而，你很快就会发现搭建本地 PHP 测试环境的必要性。本章的其余部分将专门介绍如何在 Windows 和 Mac OS X 上完成此操作。

## 本地测试环境所需组件

要在本地计算机上测试 PHP 页面，你需要安装以下软件：

- Web 服务器：这是一个显示网页的软件，而非独立的计算机
- PHP
- MySQL 以及 MySQL 的基于 Web 的前端`phpMyAdmin`，这是使用数据库所必需的

> **提示：** 某些 Linux 发行版会安装 MariaDB（ [`mariadb.org/`](https://mariadb.org/) ）作为 MySQL 的直接替代品。本书中的代码与 MariaDB 完全兼容。

所有需要的软件都是免费的。你唯一需要付出的成本就是下载必要文件的时间，当然，还要确保所有设置正确无误的时间。在大多数情况下，你应该能在不到一小时内（很可能远少于这个时间）完成配置并运行。只要你有至少 1GB 的可用磁盘空间，即使配置一般的计算机也能安装所有软件。

> **提示：** 如果你的本地计算机上已有 PHP 测试环境，则无需重新安装。只需查看本章结尾题为"检查你的 PHP 设置"的部分即可。
