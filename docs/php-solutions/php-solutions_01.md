# 引言

当《PHP 解决方案》第一版出版时，我曾担心副标题“动态网页设计轻而易举”显得过于雄心勃勃。即便是第三版，我仍然有些不安，担心不切实际地提高读者的期望。PHP 并不难，但也不像速成蛋糕粉那样：加水搅拌即可。每个网站都各不相同，因此不可能抓来一个脚本，粘贴到网页中就指望它能运行。我的目标是帮助那些对编程所知甚少甚至一无所知的网页设计师，让他们获得信心，能够深入代码并根据自身需求进行调整。

事实上，这本书自 2006 年首次出版以来一直广受欢迎，这表明许多读者接受了这一挑战。波士顿 PHP 社区的成员便大量参与其中：他们将第二版作为三系列 PHP Percolate（面向初学者的虚拟自学小组）的教材。数百人报名每周研读一章。这套方法对他们有效，因此我希望它同样对你也有效。

## 新版内容

来自 PHP Percolate 参与者及其他读者的一个宝贵反馈是：当我粗略讲解某段高级代码，仅解释其功能而非运作原理时，会令他们感到失望。这个遗漏在本版中已得到纠正。偶尔，我会指出你可能想跳过详细解释，但如果你对某种技术的工作原理感到好奇，相关内容就在那里。因此，第 3 章的参考部分已扩展，涵盖了诸如可变变量等深奥内容。不，这不是笔误；“可变变量”是 PHP 中的一个真实概念，而且相当实用。

本版内容已更新至 2014 年 8 月发布的 PHP 5.6。由于主机服务商通常升级其所提供 PHP 版本的速度较慢，我已将本书代码的最低版本设为 PHP 5.4。PHP 5.4 做出了一些重要变更，引入了简化的数组语法，并移除了对安全模式和“魔术引号”的支持。除了更新代码，我还逐行修订了每一章，澄清了解释。同时，我消除了一些错误——希望没有引入新的错误。

最大的变化在于第 6 章和第 8 章中用于上传文件和创建图像缩略图的自定义类。它们现在使用命名空间来避免与其他第三方代码的命名冲突。更重要的是，类定义已被大规模重写以提高效率。另一个重要变化是在第 9 章和第 17 章中使用新的密码哈希函数。这些函数直到 PHP 5.5 才引入，但你可以在 PHP 5.4 中通过将`password_compat`库包含在脚本中来模拟它们。关于如何获取该库（仅由一个文件组成）的详细信息，请参见第 9 章。

有关数据库操作的章节已重新组织，使其更易理解。我还加强了对预处理语句的解释，同时使用了 MySQL 改进版（MySQLi）和数据库无关的 PHP 数据对象（PDO）。某些 Linux 发行版现已安装 MariaDB 作为 MySQL 的直接替代品。为避免不必要地重复，我通常只提及 MySQL，但本书中的所有 PHP 解决方案同样适用于 MariaDB。

## 本书的组织结构

每一章都会引导你完成一个项目中的一系列阶段，每个阶段都建立在前一个阶段的基础之上。通过逐章学习，你将全面了解各部分如何整合在一起。之后，你可以参考个别阶段来重温某项特定技术。虽然本书并非参考手册，但第 3 章是 PHP 语法入门，某些章节也包含简短参考部分——尤其是第 7 章（文件读写）、第 9 章（会话）、第 10 章（MySQL/MariaDB 数据类型）、第 11 章（PHP 预处理语句）、第 13 章（四个基本 SQL 命令）以及第 14 章（处理日期和时间）。

那么，“简单”到底有多简单？我已尽力为你铺平道路，但并没有神奇药水。这需要你付出一些努力。不要试图一次性完成所有内容。循序渐进地为你的网站添加动态功能。理解它们如何运作，你的努力将获得丰厚回报。将 PHP 和 MySQL/MariaDB 加入你的技能库，将使你能够构建提供更丰富内容和交互式用户体验的网站。
