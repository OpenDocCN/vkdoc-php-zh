## 程序代码

PHP 程序是一组由解释器执行的指令。这些指令必须经过适当格式化，并传递给解释器执行。相关计划主要围绕解决以下两个任务展开。

- 编写程序代码
- 启动程序执行

第二个任务比第一个任务复杂得多。

那么，让我们创建你的第一个 PHP 程序。具体来说，我们先明确程序要做什么。按照惯例，第一个程序通常显示一条消息。我们也这样做。你的程序将在输出窗口（终端）中显示一条欢迎消息。程序如清单 1-1 所示。

```
清单 1-1
第一个程序
```

程序以`<?php`语句开始，以`?>`语句结束。这是所有 PHP 程序的标准格式。要执行的命令放在这两个指令之间。在本例中，只有一个命令。它调用了内置函数`print()`，并将`"Hello, PHP!"`作为参数传递。

| 详细信息 |
| --- |
| 函数是一个命名的代码块（命令块），可以通过调用函数（必要时指定函数名及其参数）来执行。函数可以在程序中创建，也可以使用内置函数。在众多内置函数中，有一个`print()`函数。文本值用双引号括起来。此外，文本值也可以用单引号括起来。这两种风格几乎等价，但存在一些重要区别，稍后会讨论。另外值得注意的是，PHP 中的每条命令都以分号结尾。 |

![](img/613529_1_En_1_Figa_HTML.jpg)信息图标。

**注意**

坦率地说，`print()`并非严格意义上的函数，而是 PHP 语言的一种特殊语法结构。但由于其属性与函数相似，我们将其视为函数。在所有类似情况下，我们都采用同样的处理方式。

正如你可能猜到的，该命令在终端中显示一条消息。消息文本作为参数传递给`print()`函数。以下是运行程序的结果。

```
程序输出（来自清单 1-1）
Hello, PHP!
```

这个程序并不复杂。剩下的问题只是弄清楚如何运行这个程序。

| 详细信息 |
| --- |
| 调用`print()`函数时，可以不带括号指定参数。也就是说，可以使用`print "Hello, PHP!"`语句来代替`print("Hello, PHP!")`命令。`print()`函数的另一个替代方案是`echo`语句。具体来说，可以使用`echo "Hello, PHP!"`命令来显示消息。 |



## 解释器模式

让我们探讨几种方案，您可以选择最喜欢的一种。

> **注意**
>
> PHP 程序的执行有两种模式。第一种是在客户端计算机上使用解释器执行程序。第二种模式是将 PHP 代码封装在带有 HTML 标记的文档中。在这种情况下，您必须使用外部或本地服务器。实现这种方式比使用解释器执行代码稍复杂一些，但它更“自然”，也更贴近 PHP 的实际应用。

因此，首先要做的是创建一个包含程序的文件，如清单 1-1 所示，并将该文件保存为`.php`扩展名。例如，将文件命名为`hello.php`，放置在`D:\Books\php\codes`文件夹中。接下来，执行该文件。在此阶段，您需要一个解释器，即位于 PHP 安装目录下的`php.exe`文件。假设 PHP 安装在`C:\PHP`文件夹中。启动程序的简单方法是：在终端命令行中，必须执行`php.exe`，并将`hello.php`文件（包含程序的文件）作为参数传递给它。您可以简化操作。您需要在命令行中指定`php.exe`文件的完整路径，并用空格隔开，指定`hello.php`程序的完整路径。相应的命令如下所示。

```
C:\PHP\php.exe D:\Books\php\codes\hello.php
```

不过，指定文件的完整路径并不十分方便。原则上，您可以切换到`php.exe`文件所在的目录，并将包含 PHP 代码的文件保存在同一位置。因此，如果 PHP 安装在`C:\PHP`文件夹中，要切换到该目录，请在命令行中运行以下命令。

> **详情**
>
> 如果需要更改目录和磁盘，请使用`/d`选项。例如，如果需要从`D:\`盘切换到`C:\`盘，命令格式为`cd /d C:\`。

```
cd C:\PHP
```

如果`hello.php`文件与`php.exe`文件位于同一文件夹中，则使用以下命令运行程序。

```
php hello.php
```

在 Windows 操作系统下，如果将 PHP 文件夹的路径添加到`Path`环境变量中，然后切换到程序文件所在的文件夹，也可以使用相同的命令。

> **详情**
>
> 在计算机属性中选择**高级**系统设置项，以获取`Path`环境变量。在**系统属性**窗口中，选择**环境变量**图标。在**环境变量**窗口中，选择**Path**项并修改相应变量的内容。

## 服务器模式

另一种测试 PHP 程序运行的模式是使用服务器来执行程序。我们可以使用外部或本地计算机作为服务器。我们先从第二种情况开始。

你将把 PHP 代码封装到 HTML 文档中。

## HTML 基础

要创建网页，需要使用一种特殊的文档标记语言。这种标记语言定义了文档的结构以及其各个区块的格式——也就是声明式语言 HTML（超文本标记语言的简称）。相应的文档包含纯文本和称为标签或描述符的特殊指令。这些指令是为浏览器准备的，浏览器会根据这些指令来显示文档。

让我们使用清单 1-2 中所示的 HTML 代码。

```
The First Program
清单 1-2
第一个项目的 HTML 代码
```

此代码必须保存在一个 `.php` 文件中。它是位于 `D:\Books\php\codes` 目录下的 `hi.php` 文件。

## HTML 基础

一个 HTML 文档以 `<!DOCTYPE html>` 声明开头，该声明定义了文档的规范。文档的内容放置在 `<html>` 和 `</html>` 标签之间。文档的元数据放置在由 `<head>` 和 `</head>` 标签定义的区块中。在这种情况下，只有一个元素，由 `<title>` 和 `</title>` 标签（标题元素）分隔。该元素中包含的文本 `The First Program` 用作网页的标题（显示在浏览器的文档标签页上）。

由浏览器渲染的文档内容放置在由 `<body>` 和 `</body>` 标签标记的区块中。其中只有一段与清单 1-1 类似的 PHP 代码。当服务器执行该代码时，文本会被添加到网页文档的相应位置。`print("Hello, PHP!")` 命令用于打印文本。换句话说，文本 `"Hello, PHP!"` 被添加到网页中，这些内容在你用浏览器打开文档时都能看到。但如何打开网页则是另一个问题。

如果你尝试用浏览器打开 `hi.php` 文件，结果可能并不完全是你所期望的。具体来说，文件的代码会显示在浏览器窗口中，如图 1-19 所示。

要获得不同的结果，你需要运行一个本地服务器。为此，使用 `-S` 选项运行 `php.exe`，后跟 `localhost` 关键字，并用冒号分隔，指定本地服务器的端口号。原则上，端口号可以是任意的，但建议大于 5000。

| 详细信息 |
| --- |
| 通常，端口号是 `1` 到 `65535` 之间的一个数字。系统进程会使用主要的端口号。 |

之后，在命令行中指定 `-t` 选项以及你想要标识为本地服务器根目录的文件夹链接。更具体地说，命令如下所示。

```
php -S localhost:6789 -t D:\Books\php\codes
```

该指令使用端口 `6789` 运行本地服务器。服务器的根目录被标识为 `D:\Books\php\codes` 文件夹。如果 `Path` 环境变量包含 `php.exe` 文件的路径，或者命令是从 `php.exe` 文件所在的文件夹执行的，那么这样的命令是正确的。通常，命令可能如下所示。

```
C:\PHP\php.exe -S localhost:6789 -t D:\Books\php\codes
```

这里指定了 `php.exe` 文件的完整路径。

如果一切操作正确，本地服务器就会启动。现在，打开浏览器并在地址栏中输入以下指令。

```
localhost:6789/hi.php
```

结果，在打开 `hi.php` 文件时，你的计算机会模拟服务器的工作。结果应如图 1-20 所示。

## 总结

- 接下来的章节将讨论 PHP 语言的特性。这些章节中涉及的代码功能可以在解释器模式下测试。为此，请将程序保存为扩展名为 `.php` 的文件，然后在命令行中执行相应的指令。

- 开发 PHP 项目的一个好选择是 NetBeans 开发环境。它允许你创建项目，既可以在解释器模式下测试，也可以使用本地服务器进行测试。

- 要测试程序在服务器模式下的执行正确性，可以使用以下模板（将 PHP 代码放在 `<?php` 和 `?>` 指令之间的代码块中）。
