# 选择加密方法

第 9 章中的 PHP 解决方案采用**单向加密**方式——密码一旦加密，就无法逆转该过程。这既是优点也是缺点。它为用户提供了更高的安全性，因为以这种方式加密的密码会保持隐秘。然而，丢失的密码无法重新发放，因为即使是网站管理员也无法解密。唯一的解决方法是重置密码。

另一种方案是使用**双向加密**，它依赖于一对函数：一个用于加密密码，另一个用于将密码还原为明文，从而可以轻松地向健忘的用户重新发放密码。双向加密使用一个密钥，该密钥被传递给这两个函数以执行转换。密钥只是你自己定义的一个字符串。显然，为了确保数据安全，密钥必须足够难以猜测，并且绝不应存储在数据库中。但是，你需要将密钥嵌入到注册和登录脚本中——无论是直接嵌入还是通过包含文件——因此一旦脚本暴露，安全性就会彻底崩溃。

MySQL 和 MariaDB 提供了多种双向加密函数，但`AES_ENCRYPT()`被认为是最安全的。它采用美国标准与技术研究院（NIST）批准的**高级加密标准**，密钥长度为 128 位（AES-128），可用于保护机密级（SECRET）机密材料（最高机密（TOP SECRET）材料需要 AES-192 或 AES-256）。

单向加密和双向加密各有优缺点。许多安全专家建议密码应频繁更换。因此，由于密码无法解密而强制用户更改遗忘的密码，可被视为一项良好的安全措施。另一方面，用户每次忘记现有密码时都需要处理新密码，这可能会让他们感到沮丧。我将由你决定哪种方法最适合你的情况，我仅专注于技术实现。
