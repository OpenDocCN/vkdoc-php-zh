# 设置目录管理页面

网站的目录管理部分将由 `admin.php` 页面以及多个其他 PHP 文件和 Smarty 模板组成。你将逐一构建这些组件。



对于每个组件，您首先需要实现表示层，然后编写业务层代码，最后编写数据层方法。

然而，在构建管理页面之前，您需要建立一种机制来保护对它们的访问。您当然不希望任何人访问您的管理页面！另外，为了进一步提高安全性，如果管理页面需要通过互联网（而不仅仅是本地内网）访问，您可能还需要通过`SSL`加载它们。

您的管理员将能够登录并执行各种管理任务。在本章中，您只实现目录管理功能，但在开发的第二阶段，您还将让他们管理客户订单。在实现自己的订单处理系统时，您将自行处理客户账户，并存储敏感数据，例如客户信用卡号、电话号码等。

这清楚地表明了提前规划为网站敏感区域实现安全连接的重要性。

## 使用安全连接

`HTTP`不是一个安全的协议，即使您的网站使用密码（或其他形式的身份验证）保护敏感区域，传输的数据也可能被截获和窃取。为避免这种情况，您需要设置应用程序以使用`SSL`（安全套接字层）连接，即`HTTPS`协议（超文本传输协议 - 安全版）。

为此，您首先需要做一些准备工作。除非您的`Web`服务器已经使用`SSL`连接，否则您很可能没有正确的配置。此配置涉及为您的服务器获取安全证书并将其安装在您的`Apache` `Web`服务器上。（如果托管服务由第三方提供，该托管服务可能也提供启用`SSL`的选项。）安全证书本质上是公钥-私钥对，类似于异步加密算法中使用的那些。如果您的域控制器被配置为证书颁发机构，您可以生成这些证书，但如果您不是受信任的证书颁发机构，此方法可能会出现问题。数字签名的`SSL`证书可能会导致使用这些证书的浏览器无法验证您的证书颁发机构的身份，从而怀疑您的安全性。

当某人访问安全页面，而其证书不是由受信任的证书颁发机构颁发时，浏览器将显示警告消息。当保护供公司人员访问的页面时，这并不算灾难，但如果出现此类警告消息，例如在他们尝试支付订单时，肯定会打击客户的信心。

要自行设置`Apache`，我们建议您查看`http://www.sitepoint.com/article/securing-apache-2-server-ssl`上的文章。由于启用`SSL`可能是一个耗时的过程，出于测试目的，您可以从`http://www.devside.net/web/server/free/download`获取一个已配置好的`Apache`版本。有关更多详细信息，请参阅附录 A。

另一种方法是向知名的、专注于`Web`安全的信誉良好的组织获取`SSL`证书，例如：

*   `VeriSign`（`http://www.verisign.com/`）
*   `Thawte`（`http://www.thawte.com/`）
*   `InstantSSL`（`http://www.instantssl.com/`）

`Web`浏览器内置了来自此类组织的根证书，并且能够验证由它们提供的`SSL`证书的数字签名。这意味着不会出现警告消息，并且可以轻松建立`SSL`安全连接。

## 为 SSL 配置 HatShop

如果您决定使用`SSL`，则需要安装一个`SSL`证书，如下几页所示。使用`SSL`时，还建议强制任何敏感页面通过`SSL`访问；也就是说，如果有人尝试通过`http://`访问敏感页面（例如登录页面），请求将被自动重定向到`https://` `URL`。

但是，如果您想推迟处理`SSL`并希望暂时专注于构建管理页面，您可以这样做。为了使解决方案可配置，您在第 3 章中将以下配置选项添加到了您的`include/config.php`文件中。

```php
// We enable and enforce SSL when this is set to anything else than 'no'
define('USE_SSL', 'yes')
```

如果您现在不想使用`SSL`，只需将`USE_SSL`常量设置为`no`。

### 获取 SSL 证书

获取证书是一个相对简单的过程。我们在此介绍从`VeriSign`获取证书所需的步骤，但其他提供商的过程也类似。完整说明可在`VeriSign`网站（`http://www.verisign.com/`）上找到。

您还可以从`VeriSign`获取测试证书，这些证书在试用期内可以免费使用。以下是基本步骤：

1.  在`VeriSign`网站上注册试用证书。
2.  在您的`Web`服务器上生成证书签名请求（`CSR`）。这涉及填写各种个人信息，包括您的网站名称等。为此，您需要在`Web`服务器中安装一个`SSL`模块，如附录 A 所述。
3.  将生成的`CSR`的内容复制到`VeriSign`请求系统中。
4.  不久之后，您将收到来自`VeriSign`的证书，将其复制到您的`Web`服务器以安装该证书。

步骤比这要多一点，但如前所述，`VeriSign`网站上有详细说明，您不应该遇到任何困难。

### 强制使用 SSL 连接

安装证书后，您可以通过`SSL`连接访问`Web`服务器上的任何网页，只需将用于访问页面的`URL`中的`http://`部分替换为`https://`即可（假设您的防火墙设置为允许`SSL`连接，默认使用端口`443`）。
