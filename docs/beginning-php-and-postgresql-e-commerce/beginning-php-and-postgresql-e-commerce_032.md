# 以最小精力满足长期需求

除了你需要一个快速的网站之外，我们在第 1 章中讨论的每个开发阶段都会带来必须满足的新需求。

每当你进入一个新阶段，你都希望能够**重用**大部分现有解决方案。仅仅因为需要添加一个新功能就重新设计整个网站（不仅包括视觉部分，还包括代码！）是非常低效的。通过提前规划，你可以使重用解决方案变得更容易，这样任何需要添加的新功能都可以轻松地嵌入，而不是每次更改都带来新的麻烦。

在构建网站时，实现一个由可插拔组件组成的**灵活架构**，允许你通过将新功能（如购物车、部门列表、产品搜索功能）编码为独立组件，并将它们插入到现有应用中。实现良好的灵活性水平是应用架构的目标之一，本章将展示如何将其付诸实践。

你会发现，灵活性与设计和实现它所需的时间成正比，因此我们将尝试找到一个折衷方案，在不过度复杂化代码的情况下提供最大的收益。

所有在线应用共有的另一个主要需求是拥有一个**可扩展的架构**。可扩展性被定义为增加资源以产生服务能力的线性增长。换句话说，理想情况下，在一个可扩展的系统中，客户端请求数量与处理这些请求所需的硬件资源之间的比率（比例）是恒定的，即使客户端数量增加也是如此。一个不可扩展的系统无法处理不断增长的客户端数量，无论提供多少硬件资源。因为我们对客户数量持乐观态度，我们必须确保网站能够向大量客户提供其功能，而不会抛出错误或性能低下。

**可靠性**对于电子商务应用来说也是一个关键方面。借助一致的错误处理策略和强大的关系型数据库，你可以确保数据完整性，并确保非关键性错误得到妥善处理，而不会导致网站崩溃。
