# 网站主题设计

有许多优秀的社区基础主题，如 AdaptiveTheme、Genesis 和 Zen，它们能很好地适配 Drupal 7，并已通过无障碍测试。从稳固的基础框架入手，有助于确保主题的无障碍性。

开发主题时，别忘了为所有图片添加 `alt` 标签。图片常被用于向用户传递视觉含义，因此花心思思考如何向无法看到图片的用户表达这层含义至关重要。使用自动化测试工具（后文详述）是发现图片缺失 `alt` 标签的有效方法。

Drupal 无障碍网站主题设计指南（`drupal.org/node/506866`）相当出色，其中提供了多种简化无障碍网站创建工作的途径。借助 CSS3，如今你可以实现过去需要依赖图片才能达到的效果，例如圆角与精美的 3D 特效。CSS 的 `border-radius` 属性以及针对 Webkit 和 Mozilla 浏览器的私有扩展，都能为元素增添立体感。减少图片使用不仅能让网页更易访问，还能提升加载速度，便于维护。

使用 CSS 的 `display:none` 属性时需格外谨慎，因为屏幕阅读器会严格遵循该属性的含义。当元素被设置为 `display:none` 时，它不仅在屏幕上不可见，对屏幕阅读器也同样隐藏。请确保所有链接列表（如菜单）都设有标题（通常为 H2）。若需对视力正常的用户隐藏该标题，可使用 Drupal 新增的 CSS 类 `.element-invisible`。

遵循标准进行开发，能够在项目生命周期内节省时间和成本。确保主题通过 W3C 指南验证，不仅有助于网站兼容未来技术，还能确保其对多种浏览器呈现良好效果。使用 W3C（`validator.w3.org`）或 Validator.nu（`validator.nu`）等免费工具，可以帮助你发现 CSS 和 HTML 中的问题。
