# 测试如何帮助编写整洁的代码

即使你在应用中使用单元测试，代码仍然可能变得混乱。单元测试不会替你完成工作。我将在第`4`章更详细地讨论这个话题。

## 规划

规划是创建优秀软件中非常重要的一部分，但像我和你这样的人常常缺乏耐心，无法将每个细节都规划周全。我们喜欢尽快动手。不幸的是，这种编码方式会制造问题。使用单元测试会促使我们在开发过程中进行深思熟虑（或规划）。如果我想给一个 Web 应用添加新功能，我当然会思考它如何运作。如果连这点规划都没有，我就无法编写代码。当代码能运行时，我的工作就完成了——至少是直到有人发现 bug 之前。当我编写测试时，我不得不思考可能且可接受的输入、输出以及返回值。编写测试有助于我们从远处、在更抽象的层面上思考，而测试驱动开发（TDD）则能帮助我们从中获得最大收益。

## 重构

只要你的测试没有失败，你就可以确信你测试过的代码按预期运行。测试有助于检查代码中的修改是否破坏了什么。因为单元测试是隔离运行的，它们有助于生成更好、更简单的代码。

有了测试，你无需担心改变函数签名或返回值的类型。测试会指出在哪些地方需要为函数调用进行相应修改。

如果你的测试不只是几行代码，这可能是你的代码中违反了单一职责原则的标志。如果你在重构代码后修改了测试，这种情况常常发生。记住这一点，你最终将得到更整洁的代码。
