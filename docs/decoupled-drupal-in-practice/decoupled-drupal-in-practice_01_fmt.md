# 前言

18 年前，当我在一间狭小的比利时宿舍里开始 Drupal 项目，夜以继日地开发一个原本只是为朋友们搭建的留言板时，我从未想过我的软件项目会产生如此巨大的影响。如今，Drupal 为整整 2%的在线网站提供支持，并拥有一个来自世界各地、各行各业、超过百万人的忠诚开源社区。

我也无法想象，在那些日子里，除了曾是 Drupal 主要焦点的网站之外，我们今天能在如此多样化的地方遇到 Drupal。就在过去几年里，我们看到 Drupal 被用于驱动大学校园的数字标牌、飞行娱乐系统、邮轮上的互动信息亭，甚至是一个主要交通系统中下一班到站列车的实时更新信息。

大约两年多前，我在我的博客`https://dri.es`上写道，Drupal 致力于打造雄心勃勃的数字体验。然而，当我们谈论雄心勃勃的数字体验时，我们关注的不仅仅是网站；我们还指虚拟现实和增强现实、对话式界面以及刚刚起步的物联网应用。

如今，Drupal 已具备强大的 API 优先特性，开箱即用地提供 Web 服务。解耦式 Drupal 是轻松构建我们想要在世界上看到的雄心勃勃数字体验的关键组成部分。Drupal 如今被认为是在内容管理系统中解耦架构的领导者，这绝非偶然。得益于 Drupal 的 Web 服务，任何应用程序都可以在 Drupal 上创建和编辑内容。我们拥有 JSON API、GraphQL 和 CouchDB 等知名标准，以及一个围绕解耦式 Drupal 不断发展的工具生态系统，使开发者能够轻松地使用任何技术构建应用程序。

这就是为什么我对这本书的出版感到兴奋，并且它如此重要。Preston So 是 Drupal 社区中一位受人尊敬的声音，其 Drupal 专业知识可追溯到 2007 年。2012 年，我首次与他直接合作于 Spark 计划，这是一项旨在改善 Drupal 编辑界面用户体验的社区努力，当时他还是一名大学生。从三年前他开始探索解耦式 Drupal 以来，Preston 通过其关于该主题的写作和演讲而享誉全球，并且他带头努力采用 React 来构建 Drupal 核心中的新管理界面。他在推动 Drupal 社区采用解耦架构方面发挥了关键作用。我一直与 Preston 并肩致力于这些工作，我可以告诉你，没有人比他更适合写这本书。

你手中的这本书是一本必备的解耦 Drupal 指南，涵盖了从这一趋势如何形成的历史和背景，到如何处理 Drupal Web 服务 API 细节的所有内容。当你读完时，我想你会同意，解耦式 Drupal 是 Drupal 光明未来的一个基本要素。

如果你刚接触 Drupal，我热烈欢迎你，并邀请你在 Drupal.org 上报告问题，加入我们的众多 Drupal 活动之一，并考虑贡献你一点点时间，无论你是从事代码、文档、设计、可用性测试还是其他任何工作。我很激动这本书现在可供 Drupal 社区和更广泛的 Web 开发世界使用，他们现在可以以新的视角发现 Drupal。
