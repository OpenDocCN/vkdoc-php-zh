## 静态字段和方法

先前示例中类所描述的字段和方法都是普通的，即非静态的。但是，如前所述，一个类也可以包含静态成员——*静态*字段和静态方法。

从形式上看，创建静态字段或方法很容易。你可以使用 `static` 关键字来描述它。但是，静态字段或方法与静态字段或方法有何区别？如果普通的（非静态）字段和方法“依附”于对象，那么静态类成员不属于任何对象。它们独立存在（但在所描述的类的上下文中）。

要引用静态字段或类方法，请指定类名，后跟双冒号（`::`），然后再跟字段或方法的名称（方法名称后跟带参数的括号）。如果在类代码中调用静态方法或引用静态字段，则使用 `self` 关键字代替类名。

`MyClass::hello()` 和 `MyClass::rename(“Bravo”)` 命令用于调用静态方法。在第一种情况下，会显示 `MyClass` 类的静态字段 `$name` 的值，而第二个命令将该字段设置为 `“Bravo”`（之后在程序中再次调用 `hello()` 方法）。

## 私有字段与方法

## 特殊方法

有一组方法用于解决“特殊”任务。这些方法的名称以双下划线开头。接下来我们将探讨它们。

> **注意：** 你可能已经猜到，`__construct()` 和 `__destruct()` 方法（构造方法和析构方法）也是特殊方法。

我们先从 `__toString()` 方法开始。每当一个对象出现在本应是字符串的位置时，该方法就会被自动调用。例如，当对象使用点运算符与字符串进行拼接时，或者当对象作为参数传递给 `print()` 函数或 `echo` 语句时。我们来看一下清单 7-8 中的示例。

```php
<?php
class MyClass{
public $name;
public $code;
function __construct($name,$code){
$this->name=$name;
$this->code=$code;
}
// __toString() 方法：
public function __toString(){
$txt="对象的字段：\n";
$txt.="字段 \$name = $this->name\n";
$txt.="字段 \$code = $this->code\n";
return $txt;
}
}
// 创建该类的对象：
$obj=new MyClass("Object",123);
// 显式调用 __toString() 方法：
echo "显式调用 __toString() 方法：\n";
echo $obj->__toString();
// 隐式调用 __toString() 方法：
echo "隐式调用 __toString() 方法：\n";
echo $obj;
$text=$obj."程序结束";
print($text);
?>
```


该程序描述了 `MyClass` 类，它有两个字段（`$name` 和 `$code`）。该类还有一个带有两个参数的构造方法，用于定义字段的值。此外，它还描述了 `__toString()` 方法。该方法没有参数，并返回一个字符串作为结果。该字符串通过几个步骤形成，包含了对象字段值的信息。

基于 `MyClass` 类，创建了 `$obj` 对象。在 `$obj->__toString()` 语句中，像普通方法一样显式调用了 `__toString()` 方法。但是，当执行 `echo $obj` 指令时，`__toString()` 方法会从 `$obj` 对象中自动调用，并且该方法的结果会替换对该对象的引用。当执行 `$text=$obj."程序结束"` 命令时，该方法也会被隐式调用。在这里，你将对象与一个字符串进行拼接。该操作执行如下：从 `$obj` 对象调用 `__toString()` 方法，然后将该方法返回的字符串结果与第二个字符串操作数进行拼接。

> **PHP 8 标准：** 在 PHP 8 中，如果某个类中声明了 `__toString()` 方法，那么该类会自动实现 `Stringable` 接口。接口将在后面讨论。

让我们再来探讨另外三个特殊方法：`__set()`、`__get()` 和 `__call()`。它们在引用类的成员时使用。也就是说，如果调用了不存在的对象方法，则会调用 `__call()` 方法。

**详细信息**

如果从一个对象调用某个方法，而该对象没有这个方法，那么会从该对象自动调用 `__call()` 方法。`__call()` 方法的第一个参数是不存在的方法的名称。`__call()` 方法的第二个参数是一个数组，其元素是传递给该不存在方法的参数。

清单 7-9 是一个使用 `__call()` 方法的示例。

```php
<?php
class MyClass{
function show($txt){
echo $txt,"\n";
}
function __call($name,$args){
if(strlen($name)>5){
echo "参数个数：",sizeof($args),"\n";
echo "方法 $name() 不存在！\n";
}else{
for($k=0;$k<sizeof($args);$k++){
echo "[",($k+1),"] ",$args[$k],"\n";
}
}
}
}
$obj=new MyClass();
// 调用已存在的方法：
$obj->show("MyClass 类");
// 调用不存在的方法：
$obj->display("Alpha");
$obj->hello("Bravo","Charlie","Delta");
?>
```

以下是程序执行的结果。

```
程序的输出（来自清单 7-9）
MyClass 类
参数个数：1
方法 display() 不存在！
[1] Bravo
[2] Charlie
[3] Delta
```

该程序使用了 `MyClass` 类，该类描述了一个带有一个参数的普通 `show()` 方法。当调用该方法时，会显示传递给该方法的参数值。需要使用该方法来表明 `__call()` 方法的存在不会影响类中普通方法的操作。

特殊方法 `__call()` 声明了两个参数。第一个参数 `$name` 表示从对象调用的、但该对象不存在的方法的名称。第二个参数 `$args` 是一个数组，包含传递给这个不存在的被调用方法的参数。

在方法体中，使用了一个条件语句来测试 `strlen($name)>5` 这个条件。如果被调用（不存在）的方法的名称由超过五个字符组成，则该条件为真。如果为真，则 `echo "参数个数：",sizeof($args),"\n"` 会显示一条消息，告知传递给该方法的参数数量。`echo "方法 $name() 不存在！\n"` 则会显示一条表明该方法不存在的消息。

如果 `strlen($name)>5` 条件为假，并且被调用方法的名称不超过 `5` 个字符，则会启动一个循环语句，遍历 `$args` 数组（传递给不存在方法的参数）的元素并显示它们的值。

**详细信息**

`sizeof()` 函数用于计算数组的大小。字符串的长度使用 `strlen()` 函数计算。

基于 `MyClass` 类，创建了 `$obj` 对象。如果从该对象调用已存在的 `show()` 方法，一切都会正常进行，没有意外。

`$obj->display("Alpha")` 命令从 `$obj` 对象调用了一个不存在的 `display()` 方法，参数为 `"Alpha"`。该方法的名称由超过五个字符组成。因此，当自动调用 `__call()` 方法时，它会显示一条消息指出该方法不存在，并且还包含传递给该方法的参数数量信息（在这种情况下，有一个参数）。

另一个不存在的方法由 `$obj->hello("Bravo","Charlie","Delta")` 指令调用。方法名称现在有五个字符长，因此自动调用的 `__call()` 方法会显示传递给 `hello()` 方法的参数列表。

当访问不存在的对象字段时，`__set()` 和 `__get()` 方法会被自动调用。具体来说，如果尝试给不存在的字段赋值，则会调用 `__set()` 方法。`__set()` 方法的参数是被赋值的字段名称和赋值的值本身。

**详细信息**

默认情况下，如果给一个对象的不存在的字段赋值，则该字段会被添加到对象中。如果在类中定义了 `__set()` 方法，那么当给一个不可访问（不存在或私有）的字段赋值时，`__set()` 方法会被自动调用。



`__get()`方法在尝试读取不存在的字段时被调用，该字段的名称作为参数传递给`__get()`方法。

## 总结

*   对象由类创建，可以包含字段和方法。
*   类声明以`class`关键字开头，后跟类名，并在花括号内包含类描述。
*   类中声明的字段和方法称为类成员。方法的描述与普通函数相同。描述字段时，字段的值可以省略。
*   要确定访问级别，可以使用关键字（访问级别限定符）`private`（私有类成员）、`protected`（受保护类成员）和`public`（公有类成员）。如果没有指定访问级别限定符，则表示相应的类成员是公有的。
*   要基于类创建对象，请使用`new`指令，后跟类名。相应表达式的结果是对所创建对象的引用。该引用被赋值给一个对象变量。
*   在赋值对象变量时，不会创建对象的副本，两个变量引用的是同一个对象。要创建对象的副本，赋值语句中应包含`clone`指令。
*   构造函数是在创建对象时自动调用的方法。它名为`__construct()`，可以接受参数。如果是这样，构造函数的参数将在创建对象的语句中指定，位于类名后面的括号内。
*   析构函数是在对象从内存中移除时自动调用的方法。析构函数名为`__destruct()`，没有参数。
*   一个类可以包含静态字段和方法。这些类成员使用`static`关键字声明。静态类成员不绑定到任何特定对象，并且无论是否创建了相应类的任何对象，它们都存在。
*   要访问对象的字段，请使用`$object->field`形式的指令，字段名称中不包含`$`符号。从对象调用方法使用类似`$object->method(arguments)`的命令。对静态类成员的访问以`class::$field`和`class::method(arguments)`的格式进行。
*   `$this`关键字在类中用作标识符，表示调用方法的对象。`self`关键字在类中用作类标识符，用于访问静态字段和方法。
*   有一些具有特殊用途的特殊方法，它们会被自动调用。这些方法的名字以双下划线开头。这些方法包括`__toString()`（自动调用来将对象转换为字符串）、`__call()`（尝试调用不存在的方法时自动调用）、`__get()`（尝试读取不可访问字段的值时自动调用）以及`__set()`（尝试给不可访问字段赋值时自动调用）。
*   从 PHP 8 开始，可以将构造函数中的参数描述与字段声明合并。在这种情况下，需要为参数提供一个访问级别限定符，这会自动创建相应的字段，并根据传递给构造函数的参数来赋值。

## 创建子类

假设你有一个类，想对其进行修改（为其添加一些新的属性和方法）。原则上，你可以直接修改对应类的代码。但实际情况比较复杂，因为程序中已经使用了该类。因此，修改现有类代码的想法并非最佳方案。从头开始描述一个新类，除了新增内容外还要重复原类的内容，这种做法也并不可取。首先，这样做需要重复现有代码。在编程中，这种风格通常是程序设计不良的表现。其次，如果后续需要修改原类（例如修改其中一个方法的代码），你可能需要同步修改新创建的类。那么，你打算怎么做呢？在这种情况下，合理的选择是使用*继承机制*。

思路很简单：基于现有类创建一个新类。这样一来，新创建的类会自动继承原类中所有公有（public）和受保护（protected）的方法。（类的私有成员如何处理属于特殊情况，稍后会讨论。）被继承的类称为*父类*或*基类*。基于父类创建的类称为*子类*或*派生类*。

> 一个圆形图标中心有一个字母 i。注意

PHP 支持多层继承：可以从父类创建子类，再从该子类创建另一个子类，以此类推。因此，子类和父类是相对概念：一个类可以是某个类的子类，同时又是另一个类的父类。

不过，PHP 不支持多重继承：一个子类只能有一个父类。

从技术角度来看，创建子类非常简单：在类描述中，类名后面加上 `extends` 关键字和父类名称。子类的描述中只需指定在父类成员基础上，需要追加到子类的字段和方法。父类的成员在子类中会自动可用。子类声明的模板如下所示。

`Alpha` 父类拥有 `$number` 字段和两个你想要在子类中覆盖的公有方法 `set()` 和 `show()`。`show()` 方法没有参数。当调用该方法时，它会显示一条包含 `Alpha` 类名称以及调用该方法的对象的 `$number` 字段的消息。`set()` 方法用于给 `$number` 字段赋值，它被描述为可以接受任意数量的参数（形式上，它使用前面带有省略号的 `$args` 参数来描述）。要理解为什么这样做，需要考虑以下重要情况。在子类中覆盖方法时，该方法必须使用与父类中相同的参数进行声明。父类中的 `set()` 方法为一个字段赋值。在子类中，使用该方法为两个字段赋值。因此，使用一个技巧：用任意数量的参数描述该方法，并且只使用需要的参数。在父类中，这是第一个参数，通过 `$args[0]` 指令访问。在方法体中，通过 `$this->number=$args[0]` 命令将值赋给 `$number` 字段。

> **PHP 8 标准**: 在 PHP 8 中，具有任意数量参数的方法可以覆盖具有固定数量参数的方法（需满足类型兼容性）。

`Bravo` 子类是通过继承 `Alpha` 类创建的。同时出现了 `$symbol` 字段，并且覆盖了 `set()` 和 `show()` 方法。在 `set()` 方法的描述中，`parent::set($args[0])` 命令调用了从父类继承的 `set()` 方法的版本。这里使用了一条通用规则，即如果要访问父类方法，应该在其前面加上 `parent` 关键字和双冒号（`::`）。特别地，`parent::set($args[0])` 语句意味着，当从 `Bravo` 类的对象调用 `set()` 方法时，必须首先调用父类（即 `Alpha` 类）中该方法的版本。该版本方法（来自 `Alpha` 类）的第一个参数由传递给 `set()` 方法（来自 `Bravo` 类）的第一个参数定义。结果，`$number` 字段被赋值。为了给 `$symbol` 字段赋值，使用 `$this->symbol=$args[1]` 命令，根据该命令，第二个参数决定了传递给 `set()` 方法的字段的值。为了覆盖 `show()` 方法，使用了以下命令。首先，`echo "-----------------\n"` 语句显示一条“装饰性”线条，纯粹为了美观，用以分隔 `show()` 方法的不同调用结果。`parent::show()` 命令调用了 `Alpha` 父类中 `show()` 方法的版本，之后 `echo "The class Bravo\n"` 命令显示一条消息，然后显示 `$symbol` 字段的值（通过 `echo "字段 \$symbol: ",$this->symbol,"\n"` 命令）。`$A=new Alpha` 命令创建基于 `Alpha` 类的 `$A` 对象。该对象的 `set()` 和 `show()` 方法按照 `Alpha` 类中的描述执行。`Bravo` 子类的 `$B` 对象是通过 `$B=new Bravo` 命令创建的。你可以直接验证，对于该对象，`show()` 和 `set()` 方法的执行方式不同（与 `$A` 对象相比），即根据它们在 `Bravo` 类中被覆盖的方式执行。

> **注意**: 你可以通过使用 `final` 关键字声明方法来禁用方法覆盖。

`Alpha` 类有一个私有的 `$code` 字段和两个公有方法：`set()` 方法允许你给字段赋值，`get()` 方法返回字段的值。基于 `Alpha` 类，通过继承创建了 `Bravo` 类。它继承了公有的 `set()` 和 `get()` 方法，但没有（根据常规术语）继承私有的 `$code` 字段。结果发现 `Bravo` 类拥有 `set()` 和 `get()` 方法，这些方法需要 `$code` 字段才能正常运行，但该字段本身并未被继承。这怎么可能？要回答这个问题，有必要澄清“未继承”的含义。事实是，私有字段 `$code` 并没有在 `Bravo` 类中继承，因为你不能在 `Bravo` 类的主体中直接访问 `$code` 字段。你可以将这种情况解释为该类“不知道”这个字段的存在，但从技术上讲，这样的字段是存在的，并且 `set()` 和 `get()` 方法可以操作它。`set()` 和 `get()` 方法在 `Bravo` 类构造函数中使用。`$this->set($code)` 命令为 `$code` 字段赋值，`$this->get()` 指令用于获取字段的值。

## 类的受保护成员
