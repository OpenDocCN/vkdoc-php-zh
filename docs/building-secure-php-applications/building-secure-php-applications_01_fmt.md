# 引言

如今，我们正见证 Web 应用构建方式的重大转变，尤其是“左移”范式的重要性日益凸显。随着安全领域对左移的重视，开发人员从一开始就在设计和代码中构建安全性的责任也随之增加。鉴于我们跨行业的合规性要求多种多样，忽视安全性的代价实在太高，不容忽视。基于此，本书旨在向 Web 应用开发者介绍 Web 应用安全背景，特别是在 `PHP` 环境下的安全。

开发人员处理的应用类型多种多样，包括全新项目、现有项目、混合项目（`Kubernetes` 和 `OpenShift`）以及云原生项目。为了理解这一点并获得实用见解，本书聚焦于在构建不使用任何框架的核心应用时需要理解并实现的安全方面，然后深入探讨帮助我们构建 Web 应用的各种流程背后的安全协议，最后介绍 `Laravel` 等企业框架中普遍存在的安全实践。

软件开发生命周期包含多个阶段，安全需要从一开始就融入每个阶段。本书提供了实用见解，介绍如何与利益相关者讨论安全、理解不同阶段（如开发、测试、部署、基础设施即云、云安全和维护）的安全背景。

本书最后详细介绍了安全的未来以及将纳入开发者生命周期的一些有用工具。书中分享了贯穿始终的概念和代码范例，不仅有助于学习，也有助于在实际项目中使用。
