# 使用示例文件

本书所需的所有文件均可通过 Apress 网站上的**下载源代码**按钮获取，该按钮位于 [`www.apress.com/9781484243374`](http://www.apress.com/9781484243374) 处。

按照第 2 章所述，搭建 PHP 开发环境。解压文件，并将 `phpsols-4e` 文件夹及其所有内容复制到 Web 服务器的文档根目录中。每章的代码都存放在以章节名称命名的文件夹中：`ch01`、`ch02`，以此类推。请遵循每个 PHP 解决方案中的说明，并将相关文件复制到站点根目录或指定的工作文件夹。

当某个页面在一章中经历多次修改时，我会对不同的版本进行编号，例如：`index_01.php`、`index_02.php` 等。复制带有数字的文件时，请从文件名中删除下划线和数字，这样 `index_01.php` 就变成了 `index.php`。如果你使用的程序在将文件从一个文件夹移动到另一个文件夹时会提示更新链接，请不要更新。文件中的链接在设计上可以在目标文件夹中正确引用相应的图片和样式表。我这样做是为了让你可以使用文件比较工具来检查你的文件与我的文件之间的差异。

如果你没有文件比较工具，我强烈建议你安装一个。它能为你节省数小时试图找出你的版本与我的版本之间差异而绞尽脑汁的时间。在几十行代码中，缺少一个分号或输错一个变量名可能很难发现。Windows 用户可以从 [`http://winmerge.org/`](http://winmerge.org/) 免费下载 WinMerge。我使用 Beyond Compare（[`www.scootersoftware.com`](http://www.scootersoftware.com)），它现在支持 Windows、macOS 和 Linux。它不是免费的，但功能出色且价格合理。Mac 上的 BBEdit 包含一个文件比较工具。如果你习惯在 Mac 上使用终端，系统默认安装了 `diff` 工具。

为了使本书尽可能清晰易懂，全文采用了以下文本约定：

-   重要词汇或概念首次出现时通常以**粗体**突出显示。
-   代码以 `等宽字体` 呈现。
-   新的或更改过的代码通常以 **`加粗等宽字体`** 呈现。
-   伪代码和变量输入以 *`斜体等宽字体`* 书写。
-   菜单命令以“菜单 ➤ 子菜单 ➤ 子菜单”的形式书写。
