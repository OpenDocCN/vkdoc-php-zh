# 名称和描述

访客输入的文本可以通过以下几种方式进行搜索：

**精确匹配搜索：** 如果访客输入的搜索字符串由多个单词组成，系统将按原样在目录中搜索该字符串，而不会拆分单词并分别搜索。

**全词搜索：** 访客输入的搜索字符串会被拆分为单词，从而搜索包含访客输入的每个单词的产品。这与精确匹配搜索类似，它仍然搜索所有输入的单词，但这次单词的顺序不再重要。

**任意词搜索：** 搜索字符串中的至少一个单词必须匹配到某个产品。

这个简单的分类绝不完整。搜索引擎可以像现代搜索引擎提供的工具一样复杂，提供多种选项和功能，并显示排序后的结果列表；也可以简单到只在数据库中搜索访客提供的精确字符串。

`HatShop` 将支持任意词和全词搜索模式。这一决策影响了搜索功能的视觉设计（见图 5-1）。

## 教数据库自己搜索

在数据库中实现搜索主要有两种选择：

- 使用 `WHERE` 和 `LIKE` 实现搜索。

- 使用 `tsearch2` 模块进行搜索。

让我们分析一下这些选项。

### 使用 WHERE 和 LIKE 进行搜索

最直接且常用于实现搜索的方法，是在 `SELECT` 语句的 `WHERE` 子句中使用 `LIKE`。让我们看一个简单的例子，它返回描述中包含单词“war”的产品：

```sql
SELECT name FROM product WHERE description LIKE '%war%'
```

`LIKE` 运算符匹配字符串的一部分，百分号通配符（`%`）用于指定任意数量的字符（零个或多个）。这就是为什么在前面的例子中，模式 `%war%` 会匹配所有其 `description` 列中某处包含单词“war”的记录。此搜索不区分大小写。

如果您想检索所有产品名称或描述中某处包含单词“war”的产品，查询语句如下所示：

```sql
SELECT name FROM product
WHERE description LIKE '%war%' OR name LIKE '%war%';
```

这种搜索方法有三个重要的缺点：

**速度：** 因为我们需要在 `description` 和 `name` 字段内部搜索文本，每次查询都必须搜索整个数据库。当执行数据库搜索时，这会显著降低 `HatShop` 的整体性能，尤其是在数据库中有大量产品的情况下。

**搜索结果的质量：** 这种方法不容易实现各种高级功能，例如按搜索相关性对匹配的产品进行排序。

**高级搜索功能：** 这些功能包括使用布尔运算符（`AND`、`OR`）进行搜索、搜索单词的屈折形式（例如复数形式和各种动词时态）或位置相近的单词。

那么，如何实现具备这些功能的更好的搜索呢？如果您有一个需要频繁搜索的大型数据库，如何才能在不压垮服务器的情况下进行搜索呢？

答案是使用 PostgreSQL 的 `tsearch2` 模块。

### 使用 PostgreSQL 的 tsearch2 模块进行搜索

`tsearch2` 是我们将用于实现网站搜索功能的搜索模块。该模块随 PostgreSQL 一起发布，它允许通过使用特殊的搜索索引对数据库执行高级搜索。有关安装说明，请阅读附录 A。

在构建目录搜索功能的数据层时，需要考虑两个方面：

- 准备要搜索的数据库

- 使用 SQL 搜索数据库

#### 创建支持搜索的数据结构

在我们的场景中，用于搜索的表是 `product`，因为这是访客要查找的对象。要使用 `tsearch2` 模块使该表可搜索，您需要分三步准备该表：

1.  向 `product` 表添加一个新字段，用于保存*搜索向量*。搜索向量是一个字符串，其中包含您要搜索的数据（例如产品名称和描述）的已准备（可搜索）版本。在我们的案例中，命令如下所示（现在不要执行，我们将在练习中处理这件事）：

```sql
ALTER TABLE product ADD COLUMN search_vector tsvector;
```

2.  通过在新添加的字段上添加 `gist` 索引来更新 `product` 表。`gist` 是执行实际搜索的引擎，它是伯克利广义搜索树的一种实现（有关 `gist` 的更多详细信息，请访问 <http://gist.cs.berkeley.edu/>）。在 `product` 表上添加 `gist` 索引的命令是：

```sql
CREATE INDEX idx_search_vector ON product USING gist(search_vector);
```

3.  用搜索向量填充 `product` 表的 `search_vector` 字段。这些搜索向量是每个产品可供搜索的单词列表。对于 `HatShop`，我们将考虑产品名称和产品描述中出现的单词，并赋予出现在名称中的单词更高的相关性。这样，如果多个产品匹配某个特定搜索字符串，在名称中有匹配项的产品将显示在结果列表的顶部。在此步骤中，我们还会过滤所谓的*停用词*（也称为噪声词），这些词与搜索无关，例如“的”、“或”、“在”等等。

以下命令使用 `to_tsvector` 函数（创建搜索向量）和 `setweight`（用于赋予名称中的单词更高的相关性）为每个产品设置搜索向量：

```sql
UPDATE product
SET search_vector =
setweight(to_tsvector(name), 'A') || to_tsvector(description);
```

`setweight` 的 `'A'` 参数赋予出现在产品名称中的单词最高相关性。有关这些函数如何工作的详细信息，请参考位于 <http://www.sai.msu.su/~megera/postgres/gist/tsearch/V2/docs/tsearch2-ref.html> 的 *`tsearch2` 参考*。您可以在 <http://rhodesmill.org/brandon/projects/tsearch2-guide.html> 找到一份 `tsearch2` 指南，并在 <http://www.devx.com/opensource/Article/21674> 找到一篇优秀的文章。
