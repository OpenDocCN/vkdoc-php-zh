# 检查您的网站是否支持 PHP

判断网站是否支持 PHP 最简单的方法是咨询您的托管公司。另一种方法是将一个 PHP 页面上传到您的网站，看看它是否能正常运行。即使您知道网站支持 PHP，也请执行以下测试来确认正在运行的版本：

1.  打开脚本编辑器，在空白页面中输入以下代码：

    `<?php echo phpversion();`

2.  将文件保存为 `phpversion.php`。务必确保您的操作系统不会在 `.php` 后添加 `.txt` 文件扩展名。如果您在 Mac 上使用 TextEdit，请确保它不会以多信息文本格式 (RTF) 保存文件。如果您有任何疑问，请使用本书附赠文件 `ch02` 文件夹中的 `phpversion.php`。

3.  将 `phpversion.php` 像上传 HTML 页面一样上传到您的网站，然后在浏览器中输入该 URL。假设您将文件上传到网站的顶级目录，URL 将类似于 [`www.example.com/phpversion.php`](http://www.example.com/phpversion.php)。

    如果您在屏幕上看到像 7.2.0 这样的三位数字，那么恭喜：PHP 已启用。该数字告诉您服务器上运行的 PHP 版本。*本书假定您运行的是 PHP 7.2.0 或更高版本。*

4.  如果您收到类似“Parse error”的消息，这意味着支持 PHP，但您在文件中输入代码时出现了错误。请改用 `ch02` 文件夹中的版本。

5.  如果您只看到原始代码，则表示不支持 PHP。

如果您的服务器运行的 PHP 版本低于 7.2.0，请联系您的托管商并告知他们您需要最新的稳定版 PHP。如果托管商拒绝，是时候更换托管公司了。

## 为什么不应再使用 PHP 5

PHP 版本由三个用点分隔的数字组成。第一个数字是主版本号；第二个是分支版本号；最后一个是修订版本号。主版本会引入可能破坏现有网站的显著更改，包括移除过时的特性。分支版本会引入新功能；但分支版本极不可能破坏与同一主版本的兼容性。修订版本包含针对错误和安全问题的修复。

当 PHP 5 于 2004 年发布时，开发团队仍在为 PHP 4 持续发布安全更新。这不仅延迟了对 PHP 5 的采用，也减缓了对这个新主版本的重要改进。政策的改变促使 PHP 每年发布一个新的分支版本。每个分支版本会获得两年的主动支持，用于修复错误和安全问题，随后再获得一年的关键安全修复支持。此后便不再提供支持。

PHP 5 的最终版本（5.6）获得了近两年半的额外支持；但其生命周期已于 2018 年 12 月 31 日结束。使用 PHP 5 会使您的网站和宝贵数据面临风险，因为安全漏洞将无法得到修复。此外，您也无法获得 PHP 7 新功能的优势，更不用说它的速度是 PHP 5 的两倍。对 PHP 7.0 的所有支持以及 PHP 7.1 的主动支持在本书出版前已经结束。这就是本书要求最低使用 PHP 7.2 的原因。尽管本书中的大部分代码可以在旧版 PHP 上运行，但有些则不能。

## 决定在何处测试页面

与普通网页不同，您不能直接在 Windows 文件资源管理器或 Mac 的 Finder 中双击 PHP 页面并在浏览器中查看。它们需要通过支持 PHP 的 Web 服务器进行解析或处理。如果您的托管公司支持 PHP，您可以上传文件到您的网站并在那里进行测试。但是，每次进行更改时都需要上传文件。在早期阶段，您可能会发现由于代码中的一些小错误，需要经常这样做。随着经验的增长，您仍然需要频繁上传文件，因为您会想尝试不同的想法。

如果您想立即开始使用 PHP，完全可以将您自己的网站用作测试平台。然而，您很快就会发现需要一个本地的 PHP 测试环境。本章的其余部分将向您展示如何做到这一点，并提供适用于 Windows 和 macOS 的说明。

## 本地测试环境所需的组件

要在本地计算机上测试 PHP 页面，您需要安装以下组件：

-   Web 服务器，这是一种用于显示网页的软件，而非独立的计算机
-   PHP
-   MySQL 或 MariaDB 数据库，以及 phpMyAdmin，一个用于管理数据库的基于 Web 的前端界面
