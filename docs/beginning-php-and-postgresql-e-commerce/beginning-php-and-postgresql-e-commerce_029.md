# 电子商务的核心

电子商务的核心——也是它真正区别于其他 Web 开发项目的地方——在于处理订单和信用卡。`PayPal`帮助我们推迟了这一点，但有很多充分的理由说明，最终你还是会想与`PayPal`分道扬镳：

**成本**：`PayPal`并不昂贵，但它提供的额外服务终究需要以某种方式支付。

转向更简单的信用卡处理服务可能意味着更低的交易成本（尽管这不是铁律），虽然开发自己的系统显然会带来前期投入。

**自由度**：`PayPal`有一套相当严格的条款和条件，并且仅适用于有限数量国家的居民。通过自行承担更多的信用卡处理责任，你可以更好地控制网站的工作方式。一个明显的例子是，你可以接受使用地区性的支付方式，例如在英国常见的`Switch`借记卡。

**集成度**：如果你使用自己的系统处理交易和订单，你可以根据自己的需求将商店和仓库的整合做到任何程度。你甚至可以自动联系第三方供应商，让供应商直接将商品发货给客户。

**信息**：当你全权处理整个订单时，你可以记录并整理交易中涉及的所有信息——然后将其用于营销和研究目的。

通过将订单处理与仓库、履行中心或供应商集成，你可以显著降低成本。这可能会减少履行中心对员工的需求，或者让业务在无需增加人手的情况下增长。

获取客户信息可以反哺整个流程，为你提供关于如何增加销售额的宝贵信息。最简单的方式是，你可以通过电子邮件向客户发送特别优惠，或者通过新闻简报保持联系。你还可以分析购买模式，并利用这些数据来制定有针对性的营销活动。

## 第三阶段

在第三阶段，你将学习如何：

-   构建一个客户账户模块，以便客户每次下单时都能登录并调取其详细信息
-   允许客户添加产品评论
-   使用`XML Web Services`将`Amazon.com`的产品集成到你的网站中
-   使用`SSL`（安全套接层）建立安全连接，确保用户在互联网上传输的数据得到加密
-   使用`DataCash`和`Authorize.net`处理信用卡收费
-   将信用卡号码安全地存储在数据库中

这第三阶段是所有阶段中最复杂的一个，需要艰苦细致的工作。然而，到第三阶段结束时，你将拥有一个包含可搜索产品目录、购物车、安全结账和完整订单处理系统的电子商务网站。

知道了这一点，我建议采用分阶段开发，因为：

-   公司不想把所有资金都投入到一个庞大的电子商务网站上，结果却发现人们实际上还是更喜欢邮购！或者，它可能在第一阶段后发现网站完全满足了它的需求，没有必要进一步扩展。无论哪种情况，我希望提供较低的初始成本能让我的投标更具优势。（这也可能意味着我能在总价上定得更高。）

因为这家公司已经是邮购业务，它很可能已经拥有一个商户账户并可以处理信用卡。因此，对于这家公司来说，尽快进入第三阶段是最好的，这样它就能享受到优惠的卡片处理费率。

## 总结

在本章中，我们看到了真实而充满竞争的商业世界中电子商务的一些原则，其中重要的是聚焦短期收入并降低风险。我们探讨了电子商务网站盈利的三个基本原因：

-   获取更多客户
-   让客户花费更多
-   降低订单履行成本

我们将这些原则应用到了一个三阶段计划中，该计划在每个阶段都交付一个可用的站点，并将在本书中持续扩展。
