本章讨论面向对象编程（简称 OOP）的原理如何在 PHP 中实现。让我们从引入*类*和*对象*的概念开始。这是 OOP 范式的基石，而主要概念点的掌握程度将在很大程度上决定使用 PHP 的丰富功能创建 OOP 项目的成功与否。

## OOP 原理

学习 OOP 原理通常从列举它们开始：*封装*、*多态*和*继承*。这听起来可能有点吓人。然而，并非所有事情都像初看起来那么可怕。要理解 OOP 的概念，有必要弄清楚它是什么以及为什么需要用其创建程序。OOP 范式必然意味着某种组织程序的方式。换句话说，OOP 是关于如何组织程序的。在这里，一个类比可以帮助我们。

假设有一个带厨房的小咖啡馆，你需要组织厨房的工作。厨房接收顾客的订单，并根据这些订单准备菜肴。如果咖啡馆不是很大，顾客也不多，你需要一名主厨来管理流程以确保厨房成功运作。你还需要几名助理厨师。每位助理厨师负责准备一道或多道特定的菜肴。例如，一位厨师专攻肉类菜肴，另一位专攻鱼类菜肴，下一位准备沙拉，有人专攻甜点，等等。可以使用一台或多台冰箱来存储食材，所有厨师都将食材存放在那里，并使用这些食材来准备餐点。所有厨师都可以使用所有冰箱。任务的分配由主厨负责。主厨还控制冰箱里的内容：所有食材都有吗？数量够吗？

> **注意：** 即使在实际中，厨房的工作组织方式不同，这也没关系。这个例子只是为了说明；你只需要解释方法上的差异，而不是组织一个真实厨房的工作。

我们来比较一下厨房的工作方式和程序的工作方式。助理厨师类似于函数，而冰箱里存放的食材类似于数据。总体目标是“连接”数据与处理该数据相关的“功能”。用“烹饪”的语言来说，这意味着你需要为每位助理厨师定义其职责范围以及在工作中可以使用的食材。

假设你把一切都安排妥当了，厨房运转得像时钟一样精准。什么会破坏这种局面呢？主要原因之一是“企业”的扩张。在编程语言中，这表现为代码量的增加。想象一下，顾客很多，厨房很大——很多助理厨师，很多台冰箱。不可避免的，由于有人错误地接单、拿走了另一位厨师的食材或从流程中退出，就会出现问题。这是一个纯粹的规模扩展问题。可以通过改变生产结构来解决。也就是说，我们把厨房分成几个部门。每个部门都有自己的专长。每个部门的食材存放在各自的冰箱里。每个部门都有自己的主厨。这可能会增加员工和总体成本，但它使厨房更可靠，因为由于生产组织方式的原因，一些问题会自动消除。例如，如果冰箱位于鱼类部门，肉类部门的人就无法使用它。

在这个新模型中，什么重要？重要的是你使用了一个相互交互的部门系统。而在最初的方案中，是单个助理厨师之间相互交互。在新的方法中，厨师们也在部门内部相互交互。在下一个层级，部门之间也存在交互。你可以用同样的方式处理程序。具体来说，你可以将程序组织为一些包含数据并能处理这些数据的块之间的交互。这些块被称为*对象*。这种方案本身就描述了面向对象编程（OOP）的范式。

OOP 原则旨在用于创建大型程序。一个大型程序通常包含数千行代码甚至更多。

因此，在面向对象编程框架内编写程序意味着要创建对象。对象是基于被称为*类*的模板创建的。

让我们用另一个比喻来理解类与对象之间的区别。假设你想建造一所房子（类比为对象）。但是要建房子，你需要一份建造所依据的计划或方案（类比为类）。你拿到一份计划，然后按照计划建造房子。类似地，你拿到一个类，然后基于这个类创建对象。基于同一份计划可以建造多所房子。它们类型相同，但在物理上是不同的。如果你根据不同的计划建造房子，就会得到不同的房子。同样，基于一个类也可以创建多个对象。它们类型相同（功能相同），但同时，物理上它们是不同的。如果你基于不同的类创建对象，那么对象也是不同的。还有一种情况是，有蓝图但没有建筑物。同样，你可以创建一个类，但不基于它创建任何对象。

这就是核心思想。接下来，你必须学习如何声明类并基于它们创建对象。

> **注意**

> 回到封装、多态和继承，有必要提及以下几点。它们是 OOP 的三种基本机制。封装意味着数据和用于处理数据的代码被组合成一个单一的单元，即对象。多态意味着使用通用接口与对象、函数和方法进行交互。最后，继承允许你通过继承现有类中的属性来创建新类。所有这些机制都将在示例中逐步讲解。

## 创建类和对象

现在是深入实践问题的时候了。让我们从创建类开始。

类的声明以 `class` 关键字开头，后跟类名。然后，在由大括号定义的代码块中，描述类的内容。类声明的模板如下所示。

```
class 名称{
```

## 类的内容

但关键问题依然存在：类中可以描述什么？简而言之，你可以在类中描述方法和字段。字段本质上是变量。与普通变量不同，字段是在类中描述的。无需给它们赋值，只需在类中声明它们的存在即可。方法与函数相同。但与函数不同的是，方法是在类中描述的。

> **注意**

> 字段有时也被称为属性或特性。类的字段和方法被称为类的成员。

以下是一个仅包含单个字段的简单类示例。

```
class MyClass{
public $number;
}
```

该类只包含一条 `public $number` 语句，这表示该类描述了一个名为 `$number` 的字段。`public` 关键字被称为*访问级别说明符*，表示该字段是公有的。字段的“公有性”反过来允许你不仅在类内部访问该字段，还能在类外部访问它。

> **注意**

> 除了 `public` 访问级别说明符，还可以使用 `private`（私有字段）和 `protected`（受保护字段）关键字。访问级别说明符如何影响字段属性将在后文讨论。

因此，存在一个包含单个 `$number` 字段的 `MyClass` 类。这意味着什么？这意味着如果基于 `MyClass` 类创建一个对象，该对象就拥有 `$number` 字段。

| 详细信息 |

| --- |

| 字段就像一个变量。如果把对象想象成一个容器，那么对象的字段就是放入该容器的变量。如果有多个基于同一个类创建的对象，它们拥有相同的字段集合。然而，这些是不同的变量，因为每个对象都有自己的字段。因此，在引用对象字段时，仅指定字段名是不够的。还必须指定该字段所属的对象。 |

要基于某个类创建对象，请使用 `new` 指令，后跟作为创建对象基础的类名。对该对象的引用会被写入到一个与该对象关联的变量（对象变量）中。创建对象的语句模板如下所示。

```
$variable=new ClassName;
```

例如，如果要创建一个 `MyClass` 类的对象，相应的命令可能如下所示。

```
$obj=new MyClass;
```

该命令创建一个 `MyClass` 类的对象，并且对该创建对象的引用被写入到 `$obj` 变量中。如果需要访问该对象，请使用 `$obj` 变量。

> **PHP 8 标准**

> 在 PHP 8 中，你可以使用 `$obj::class` 语句来获取创建 `$obj` 对象所基于的类的名称。在之前的 PHP 版本中，使用 `get_class()` 函数来实现此目的，相应的命令形式为 `get_class($obj)`。

但对象主要体现在其字段和方法上。因此，对对象的访问就是对其字段和方法的访问。为此，请在对象变量后面加上箭头 `->`，然后放置不带 `$` 符号的字段名。在这种情况下，你可以使用 `$obj->number` 指令访问 `$obj` 对象的 `$number` 字段。

清单 7-1 是一个程序，其中描述了一个类，基于该类创建了一个对象，然后对该对象字段执行了一些操作。

```
<?php
// 类描述：
class MyClass{
public $number;
}
// 创建对象：
$obj=new MyClass;
// 给字段赋值：
$obj->number=123;
// 显示字段：
echo "字段 \$number: $obj->number\n";
// 字段的新值：
$obj->number=321;
// 显示字段：
echo "字段 \$number: $obj->number\n";
?>

创建对象
```

```
程序输出（来自清单 7-1）

```

程序从描述 `MyClass` 类开始。基于该类，`$obj=new MyClass` 指令创建了 `$obj` 对象。它拥有 `$number` 字段，使用 `$obj->number=123` 语句为其赋值为 `123`。然后，使用 `echo "字段 \$number: $obj->number\n"` 语句显示 `$obj` 对象的 `$number` 字段。

之后，`$obj->number=321` 命令将新值 `321` 赋给 `$obj` 对象的 `$number` 字段。检查结果显示该字段值已更改。

### 方法

如前所述，除了在类中描述字段，你还可以在其中描述方法。方法的描述方式与函数相同，但仅在类体内。方法的主要特点与其调用方式有关。调用函数时，必须指定函数名及其参数列表。调用方法时，需要指定调用该方法的对象。该方法可以访问类中描述的所有字段和方法。

> **注意**
>
> 此处我们讨论的是普通（非静态）字段和方法。除了普通类成员，还存在静态成员。对它们的操作有其自身特点，将在后文讨论。

清单 7-2 是对前一个示例的小修改。现在，`MyClass` 类除了 `$number` 字段外，还包含两个方法。

```php
<?php
// 类描述：
class MyClass{
public $number;
// 显示字段的方法：
function show(){
print("字段 \$number: $this->number\n");
}
// 给字段赋值的方法：
function set($n){
$this->number=$n;
}
}
// 创建对象：
$A=new MyClass;
$B=new MyClass;
// 给字段赋值：
$A->set(100);
$B->set(200);
// 显示字段：
$A->show();
$B->show();
?>
清单 7-2
使用方法
```

以下是程序执行的结果。

```
程序输出（来自清单 7-2）
字段 $number: 100
字段 $number: 200
```

主要变化与 `MyClass` 代码有关。除了 `$number` 字段，该类还包含了 `show()` 和 `set()` 方法。`show()` 方法没有参数，允许显示调用该方法的对象的 `$number` 字段。`set()` 方法为调用该方法的对象的 `$number` 字段赋值。该方法的参数（名为 `$n`）决定了赋给 `$number` 字段的值。

在描述这些方法时，你会面临一个关键问题。即，方法是从对象调用的。这意味着，要调用方法，必须指定方法名和调用该方法的对象。而该方法反过来要访问对象的 `$number` 字段。访问字段时，必须指定（拥有该字段的）对象。但在描述方法时，对象尚不存在。换句话说，你需要以某种方式指代调用该方法的对象。这就是 `$this` 标识符的用途。在方法代码内部使用时，`$this` 指令指代调用该方法的对象。因此，例如，`$this->number` 命令是对调用该方法的对象的 `$number` 字段的引用。

具体来说，在 `show()` 方法体中，使用 `print("字段 \$number: $this->number\n")` 语句，它显示调用该方法的对象的 `$number` 字段。在 `set()` 方法体中，`$this->number=$n` 语句将方法的参数 `$n` 赋给 `$number` 字段。

在程序中，创建了两个 `MyClass` 类的对象 `$A` 和 `$B`。为此，分别使用了 `$A=new MyClass` 和 `$B=new MyClass` 命令。每个对象都有 `$number` 字段，但尽管名称相同，这些是不同的字段：一个属于 `$A` 对象，另一个属于 `$B` 对象。使用 `set()` 和 `show()` 方法对字段进行操作。`$A->set(100)` 指令将 `100` 赋给 `$A` 的 `$number` 字段，`$B->set(200)` 指令将 `200` 赋给 `$B` 的 `$number` 字段。之后，使用 `$A->show()` 命令显示 `$A` 对象的 `$number` 字段，并使用 `$B->show()` 命令显示 `$B` 对象的 `$number` 字段。

| 详细信息 |
| --- |
| 有趣的是，当尝试给对象中不存在的字段赋值时，该字段会出现（所谓的动态属性）。例如，如果 `$obj` 对象没有 `$name` 字段，那么在执行 `$obj->name="php"` 命令后，`$obj` 对象会获得值为 `"php"` 的 `$name` 字段。然而，自 PHP 8.2 起，这种用法已被弃用。如果仍然想使用它，应该在类描述之前加上 `#[\AllowDynamicProperties]` 注解。可以使用 `unset()` 函数从对象中删除一个字段。例如，要从 `$obj` 对象中删除 `$name` 字段，`unset($obj->name)` 指令可能会有用。另外值得一提的是，方法和字段一样，可以使用 `public`、`private` 和 `protected` 访问级别说明符来描述。如果方法在描述时没有访问级别说明符（如前面的示例所示），那么该方法默认为公有方法。良好的实践是显式为公有方法指定 `public` 访问级别。不过，为了演示 PHP 的灵活性和强大功能，某些方法将使用默认的（公有）访问级别（即不使用 `public` 关键字）。此外，你可以在类描述中为字段赋值。在这种情况下，当基于该类创建对象时，对象的相应字段将具有在类中赋给该字段的值。 |

> **PHP 8 标准**
>
> 在 PHP 8 版本中，访问对象字段和方法时，可以使用 `?->` 指令代替 `->`。例如，可以使用 `$object?->method` 表达式代替 `$object->method` 表达式。前一个选项更安全，因为如果 `$object` 变量的值是空的 `null` 引用，则不会发生致命错误。

### 构造函数与析构函数

在示例中，你基于类创建了对象，然后为对象的字段赋值。如果对象很多，并且它们有很多字段，那么这个过程可能会变得繁琐。因此，建议将其自动化。在这方面，两个特殊方法可能会很有用：*构造函数*和*析构函数*。

构造函数是在创建对象时自动调用的方法。构造函数的名称是 `__construct()`（名称以双下划线开头）。构造函数可以接受参数。通常，构造函数包含为字段赋值或执行其他操作的命令。

| 详细信息 |
| --- |
| 如果构造函数有参数，那么这些参数必须在创建对象时指定。构造函数的参数在基于 `new` 指令创建对象的命令中，类名后面的括号内指定。 |

析构函数是在从内存中移除对象时自动调用的方法。析构函数的名称是 `__destruct()`，并且它没有参数。清单 7-3 说明了如何使用构造函数和析构函数。

```php
<?php
// 类描述：
class MyClass{
public $code;
// 构造函数：
function __construct($code){
$this->code=$code;
echo "对象已创建：",$this->code,"\n";
}
```
