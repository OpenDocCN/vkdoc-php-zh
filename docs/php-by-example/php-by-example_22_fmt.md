## 总结

- 可以在现有类的基础上创建新类。这种机制称为继承。派生新类的类称为**父类**。从父类派生的类称为**子类**。父类的所有非私有字段和方法都会被子类继承。

- 在子类的声明中，类名后面跟着关键字`extends`和父类的名称。只能有一个父类。另一方面，一个子类也可以是另一个子类的父类。

- 子类中继承的方法（包括构造函数和析构函数）可以被重写。在这种情况下，子类中会描述继承方法（或构造函数/析构函数）的新版本。新版本方法中的参数必须与原始版本方法中的参数匹配。此规则不适用于构造函数。要访问父类中的方法（或构造函数/析构函数）版本，请使用`parent`关键字、双冒号(`::`)和方法名称（或构造函数/析构函数）。

- 为了防止方法被重写，可以在其描述中使用`final`关键字。如果希望禁用类的继承，也可以使用相同的关键字。

- 私有类成员不被继承。此类成员不能在子类中直接访问。但是，可以通过继承的公有方法访问它们。

- 受保护的类成员使用`protected`访问级别指定符声明。与类的私有成员一样，它们只能在类代码中访问，但与私有成员不同，受保护成员会被继承。

- 方法和构造函数是虚拟的。这意味着方法的版本由调用该方法的对象所属的类决定。

## 抽象类

创建类时，你可能不描述某些方法，而只声明它们。换句话说，你可以指定该类具有某个方法，但不指定该方法做什么。这样的方法称为**抽象方法**。

> **注意**
>
> 你为什么这样做是另一个问题。现在重要的是，这样做是可能的。

抽象方法使用`abstract`关键字声明。如果一个类至少有一个抽象方法，那么该类也是抽象的，并且必须使用`abstract`关键字定义。但问题仍然存在：你究竟为什么需要这些？

最概括的答案是，抽象类用于继承。为了理解这个答案的深度，你应该从稍微不同的角度来看待继承过程，而不是局限于可以将字段和方法从父类传递给子类这一点。关键在于，父类定义了子类的“结构”，而子类是基于父类创建的。更具体地说，让我们看一个例子（无需编写程序代码）。

假设您想要创建类来实现描述不同几何形状的对象。更具体地说，您想要创建实现正方形、圆形和等边三角形的对象。这些形状中的每一个都有一个特征尺寸（正方形的边长、圆的半径和三角形的边长）。对于每个图形，都可以计算面积。因此，每个实现几何图形的对象都必须有一个字段（图形的特征尺寸）和一个用于计算面积的方法。

> **注意**

### 计算几何图形面积的抽象类与方法

面积是通过方法计算的，因为几何图形的这一特性由其特征尺寸决定，并且必须根据图形特征尺寸的变化而改变。最好使用一个在被调用时能引用存储图形特征尺寸字段的方法。注意，边长为`R`的正方形面积等于`R²`。半径为`R`的圆面积等于`πR²`。边长为`R`的等边三角形面积由表达式`√(3)R² / 4`定义。

因此，在这三种情况下，对象都具有相似的结构。唯一的区别在于计算面积方法的实现方式。但它们是不同的对象，必须基于不同的类来创建。那么，你可能会问：如何让不同的类具有相同的结构（即相同的字段和方法集，但方法的实现不同）？答案如下：你可以描述一个抽象类，然后通过继承基于它创建不同的类。这些类具有由抽象类定义的相同结构。

一个圆形图标，中心有一个字母`i`。

### 抽象类优于普通类的原因

原则上，要实现你的策略，你可以使用一个普通类来替代抽象类，并重写继承的方法。但这不是一个好选择，因为如果某个方法意外未被重写，虽然不会出现形式上的错误，但代码无法正确执行。这种情况很难追踪。从这个意义上说，抽象类更可靠。

## 接口

因此，抽象类可以用作创建其他类的模板。它们具有相似的结构，这在实践中非常方便。然而，这里有一个问题。关键在于，一个子类只能有一个父类。如果你需要同时将多个“模板”组合到一个子类中，抽象类在这种情况下就帮助不大了。幸运的是，有另一种机制可以解决这个问题。它基于使用*接口*。

接口与抽象类非常相似。唯一根本的区别在于，抽象类可以包含非抽象方法，而接口则不能。此外，接口没有字段（但可以包含常量——它们的描述中使用了 `const` 关键字）。接口可以被视为一个“非常抽象”的类。从这个意义上说，接口的功能比抽象类更有限。然而，接口的优势在于，一个类可以同时实现多个接口。

要在实践中使用接口，你首先需要弄清楚如何创建接口，其次是如何在类中实现它。创建接口很容易。它以 `interface` 关键字开头，然后在花括号中描述接口的内容。这是对接口中包含的方法的声明（没有描述）。在这里，你可以使用访问级别说明符 `public` 和 `protected`。你也可以声明常量。

如果一个类实现了一个接口，那么该接口中的所有方法都必须在类中进行描述。类实现了某个接口这一事实在类描述中说明：在类名之后，使用关键字 `implements` 给出该类实现的接口名称。如果该类实现了多个接口，则它们以逗号分隔，列在 `implements` 关键字之后。在实践中，所有这些看起来是怎样的，如清单 9-2 所示。

这个例子与前一个类似，但存在一个本质区别。如前所述，声明了带有`show()`方法的`Alpha`接口。`Bravo`接口声明了`set()`方法。但除此之外，`Bravo`接口还继承了`Alpha`接口。因此，`Bravo`接口隐式获得了`show()`方法和`set()`方法。相应地，实现`Bravo`接口的`MyClass`类必须包含（并且确实包含）`set()`和`show()`方法的描述。

> **注意**

> 一个接口继承另一个接口，然后由类来实现这种方案并不好。我们通过例子来解释这一点。特别是，`Bravo`接口只明确声明了`set()`方法。你必须追溯接口继承链才能理解在实现该接口时必须定义`show()`方法。这不太理想。第一种方案，即类实现两个不同的接口，要方便得多，因为你可以立即识别出需要在类中实现的内容。

## 特征（Traits）

类的继承和接口的实现都解决了一个战略性问题：代码复用。还有另一种语法结构专门用于解决这个问题，那就是**特征**。

特征是一段可用于“构建”类的代码块。特征本身并不独立发挥作用，但你可以通过组合它们来得到一个类。特征的描述方式如下：使用`trait`关键字，然后在花括号内放置特征代码块。这是当你包含该特征时，会被添加到类中的代码块。要在类中包含特征，需要在类体中使用`use`指令，然后列出要包含的特征，用逗号分隔。清单 9-5 展示了其实际用法。

```
    number,"\n";
        echo "\$symbol = ",$this->symbol,"\n";
    }
}
// The second trait:
trait Second{
    function set($number,$symbol){
        $this->number=$number;
        $this->symbol=$symbol;
    }
}
// Including the traits in the class:
class MyClass{
    use First,Second;
    private $symbol;
    function __construct($number,$symbol){
        $this->set($number,$symbol);
        $this->show();
    }
}
// Creates an object of the class:
$obj=new MyClass(100,'A');
?>
```

## 对象类型控制

在某些情况下，确保程序中使用的对象属于特定类或实现特定接口非常重要。为什么需要这样做？例如，你描述了一个函数，并计划将一个对象作为参数传递给它，然后调用该对象的某个方法。该对象需要拥有该方法。你如何保证这一点？一种方法是在函数描述中显式指定参数类型。但如果只指定类名，那么只能将该类（或其子类）的对象作为参数传递给此类函数。还有一种更“精细”的方法。它包括创建一个接口，在其中声明所需的方法。然后，创建一个参数类型为接口名称的函数。如果是这样，任何实现该接口的类的对象都可以作为该函数的参数传递。对象实现了接口这一事实保证了该对象拥有所需的方法。

## 命名空间

命名空间的概念是通用的，并在许多编程语言中使用。为了理解它，让我们考虑一个与编程无关的虚构场景。假设一个学生群体中有两个学生同名（相同的姓、名和中间名）。在官方文件中区分这些学生将非常困难。但是，如果你按小组划分整个学生群体，并将同名者分配到不同的组中，那么学生识别的情况就会得到显著改善。使用命名空间时也会发生类似的事情。命名空间类似于学生组。在命名空间内，类、函数和其他工具应该是可区分的，或者换句话说，应该具有唯一的名称。但是，例如，如果函数属于不同的命名空间，它们可以有相同的名称。

## 匿名类

有些情况下，你需要一个类只基于它创建一个对象。在这种情况下，使用匿名类很方便。这意味着你在创建类的同时创建一个对象。相应的指令以`new`关键字开头，后跟`class`关键字以及花括号中的类描述。让我们看一个小例子来解释所描述的方法。程序如清单`[9-14]`所示。

```php

    // 获取字段值的方法：
    public function get(){
        return $this->number;
    }
    // 给字段赋值的方法：
    public function set($number){
        $this->number = $number;
    }
};
// 对对象的操作：
$obj->set(100);
echo "\$number = ", $obj->get(), "\n";
?>
Listing 9-14
An Anonymous Class
```

```
The output of the program (from Listing 9-14)
```

## 总结

* 抽象类用`abstract`关键字描述，并包含抽象方法。抽象方法没有实现，仅声明。抽象方法的声明也使用`abstract`关键字。除了抽象方法，抽象类还可以包含常规（非抽象）方法和字段。抽象类用于通过继承创建基于它们的子类。

* 接口就像一个“完全抽象”的类。接口没有字段（但可以有常量），并且其中只声明方法。接口用`interface`关键字描述。如果一个类实现了某个接口，该类必须实现接口中的每个方法。要在类中实现接口，在类描述中，需要在类名后放置`implements`关键字和要实现的接口名。如果一个类实现了多个接口，它们列在`implements`关键字后，用逗号分隔。

* 接口可以继承另一个接口。在这种情况下，使用`extends`关键字，就像在类继承中一样。

* 一个类可以同时继承另一个类并实现多个接口。

* Trait 包含可以包含在类描述中的代码块。Trait 使用`trait`关键字描述。要将 trait 内容包含到类中，需要放置`use`指令，然后列出要包含在类中的 trait，并用逗号分隔。

* 如果将接口名指定为函数参数类型，则可以将实现该接口的类对象作为参数传递给该函数。如果将类名（通常是抽象类）指定为参数类型，则可以将任何子类的对象作为参数传递。`instanceof`指令可用于检查对象或参数的类型。

* 要创建命名空间，使用`namespace`指令，后跟命名空间的名称。应将适当的文件导入到程序中以包含该命名空间。通过指定命名空间名称来访问命名空间中的工具。使用`as`指令创建别名。

* 当只需要基于一个类创建一个对象时，使用匿名类。在这种情况下，将一个表达式赋值给对象变量，该表达式以`new`和`class`关键字开头，后跟花括号中实际的类描述，基于该描述创建对象。
