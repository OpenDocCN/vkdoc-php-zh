## 选择包含文件（Include Files）的存放位置

PHP 包含文件的一个有用特性是它们可以放在任何位置，只要使用`include`命令的页面知道在哪里找到它们即可。包含文件甚至不需要放在 Web 服务器的根目录内。这意味着您可以将包含敏感信息（如密码）的包含文件保护在浏览器无法访问的私有目录（文件夹）中。因此，如果您的托管公司提供了服务器根目录之外的存储区域，您应该认真考虑将部分（如果不是全部）包含文件存放在那里。

`include`命令需要一个相对路径或完整路径。如果两者都未指定，PHP 会自动在 PHP 配置中指定的`include_path`中查找。下一节将解释如何更改 PHP 自动搜索包含文件的文件夹。

## 调整您的`include_path`

将包含文件存放在 Web 服务器的`include_path`中指定的文件夹中的好处是，您无需担心相对路径或绝对路径是否正确。您只需要文件名即可。如果您使用大量包含文件，或者您的站点层级较深，这将非常有用。更改`include_path`有三种方法：

- 编辑`php.ini`中的值：如果您的托管公司允许您访问`php.ini`，这是添加自定义包含文件夹的最佳方式。

- 使用`.htaccess`或`.user.ini`：如果您的托管公司允许通过`.htaccess`或`.user.ini`文件更改配置，这是一个很好的替代方案。

- 使用`set_include_path()`：仅在其他选项不可用时使用此方法，因为它仅影响当前文件的`include_path`。

运行`phpinfo()`时，您的 Web 服务器的`include_path`值会列在配置详情的 Core 部分。它通常以句点开头，表示当前文件夹，后跟每个要搜索的文件夹的绝对路径。在 Linux 和 Mac OS X 上，每个路径由冒号分隔。在 Windows 上，分隔符是分号。在 Linux 或 Mac 服务器上，您现有的`include_path`指令可能如下所示：

```
.:/php/PEAR
```

在 Windows 服务器上，等效内容如下所示：

```
.;C:\php\PEAR
```

### 在`php.ini`或`.user.ini`中编辑`include_path`

在`php.ini`中，找到`include_path`指令。要在您自己的站点中添加一个名为`includes`的文件夹，请在现有值的末尾添加一个冒号或分号（取决于服务器的操作系统），后跟`includes`文件夹的绝对路径。

在 Linux 或 Mac 服务器上，像这样使用冒号：

```
include_path=".:/php/PEAR:/home/mysite/public_html/includes"
```

在 Windows 服务器上，使用分号：

```
include_path=".;C:\php\PEAR;C:\sites\mysite\www\includes"
```

`.user.ini`文件的命令相同。`.user.ini`中的值会覆盖默认值，因此请确保从`phpinfo()`复制现有值并将新路径添加到其中。

### 使用`.htaccess`更改`include_path`

`.htaccess`文件中的值会覆盖默认值，因此请从`phpinfo()`复制现有值并将新路径添加到其中。在 Linux 或 Mac 服务器上，值应类似于以下内容：

```
php_value include_path ".:/php/PEAR:/home/mysite/public_html/includes"
```

在 Windows 上命令相同，只不过需要用分号分隔路径：

```
php_value include_path ".;C:\php\PEAR;C:\sites\mysite\www\includes"
```

注意：在`.htaccess`中，不要在`include_path`和路径名列表之间插入等号。

### 使用`set_include_path()`

虽然`set_include_path()`仅影响当前页面，但您可以轻松创建代码片段并将其粘贴到要使用它的页面中。PHP 还简化了获取现有`include_path`并以平台无关的方式将其与新路径结合的操作。

将新路径存储在一个变量中，然后与现有值结合，如下所示：

```php
$includes_folder = '/home/mysite/public_html/includes';
set_include_path(get_include_path() . PATH_SEPARATOR . $includes_folder);
```

看起来像是向`set_include_path()`传递了三个参数，但实际上只有一个；这三个元素由连接运算符（句点）连接，而不是逗号。

- `get_include_path()` 获取现有的`include_path`。

- `PATH_SEPARATOR`是一个 PHP 常量，它会根据操作系统自动插入冒号或分号。

- `$includes_folder`添加新路径。

这种方法的问题在于，新`includes`文件夹的路径在远程服务器和本地测试服务器上可能不相同。您可以使用条件语句来解决这个问题。超全局变量`$_SERVER['HTTP_HOST']`包含网站的域名。如果您的域名是`www.example.com`，您可以像这样为每个服务器设置正确的路径：

```php
if ($_SERVER['HTTP_HOST'] == 'www.example.com') {
    $includes_folder = '/home/example/public_html/includes';
} else {
    $includes_folder = 'C:/xampp/htdocs/phpsols/includes';
}
set_include_path(get_include_path() . PATH_SEPARATOR . $includes_folder);
```

对于不使用许多包含文件的小型网站来说，使用`set_include_path()`可能并不值得。但在更复杂的项目中，您可能会发现它很有用。

## 为什么我不能在 PHP 包含文件中使用站点根目录相对链接？

嗯，您可以使用，但也不能用。为清楚起见，我将首先解释文档相对链接和站点根目录相对链接之间的区别。

### 文档相对链接

当您单击链接转到另一个页面时，`<a>`标签中的路径告诉浏览器如何从当前页面到达下一个页面。大多数 Web 创作工具指定的路径是相对于当前文档的。如果目标页面在同一文件夹中，则仅使用文件名。如果目标页面比当前页面高一级，则文件名前会加上`../`。这被称为文档相对路径或链接。如果您的站点有许多文件夹层级，这种类型的链接可能难以理解（至少对人类来说如此）。

### 站点根目录相对链接

另一种类型的链接始终以正斜杠开头，它是站点根目录的简写。站点根目录相对路径的优点是，无论当前页面在站点层级中多深，开头的正斜杠都能保证 Web 服务器将从站点的顶层开始查找。尽管站点根目录相对链接更容易阅读，但 PHP 的`include`命令无法处理它们。您必须使用文档相对路径、绝对路径，或者在`include_path`指令中指定`includes`文件夹。

您可以通过将超全局变量`$_SERVER['DOCUMENT_ROOT']`连接到路径的开头来将站点根目录相对路径转换为绝对路径，如下所示：

```php
include $_SERVER['DOCUMENT_ROOT'] . '/includes/filename.php';
```

大多数服务器支持`$_SERVER['DOCUMENT_ROOT']`，但您应该检查`phpinfo()`显示的配置详情底部的 PHP Variables 部分以确认。

### 包含文件中的链接

这一点往往会让人困惑。尽管 PHP 的包含命令不理解站点根目录相对链接，但包含文件中的链接通常应该相对于站点根目录。这是因为包含文件可能被放入站点层级中的任何位置，因此当文件被包含在不同层级时，文档相对链接就会失效。

**注意：** `menu.php` 中的导航菜单使用的是文档相对链接，而非相对于站点根目录的链接。之所以特意保留这种写法，是因为除非你创建了虚拟主机，否则站点根目录是 `localhost`，而不是 `phpsols`。这是在 Web 服务器文档根目录的子文件夹中测试站点的一个缺点。本书全程使用的日本之旅网站只有一个层级，因此文档相对链接可以正常工作。当开发多级文件夹的站点时，请在包含文件中使用站点根目录相对链接，并考虑设置虚拟主机用于测试（详见第 2 章）。

### 嵌套包含文件

一旦一个文件被包含到另一个文件中，相对路径的计算将基于父文件，而不是被包含的文件。这给需要包含另一个外部文件的外部文件中的函数或类定义带来了问题。

如果两个外部文件位于同一文件夹中，你可以仅通过文件名来包含嵌套文件，像这样：

`require_once 'Thumbnail.php';`

在这种情况下，相对路径不应以 `./` 开头，因为 `./` 表示“从此文件夹开始”。对于包含文件来说，“此文件夹”指的是父文件的文件夹，而不是包含文件本身的文件夹，这会导致嵌套文件的路径错误。

当包含文件位于不同文件夹时，你可以使用 PHP 常量 `__DIR__` 构建目标文件的绝对路径。该常量返回包含文件所在目录（文件夹）的绝对路径，且不带尾部斜杠。将 `__DIR__`、一个正斜杠和文档相对路径拼接起来，就能将相对路径转换为绝对路径。例如，假设从一个包含文件到另一个包含文件的相对路径如下：

`'../File/Upload.php'`

你可以这样将其转换为绝对路径：

`__DIR__ . '/../File/Upload.php'`

为了方便，在文档相对路径的开头添加了正斜杠。这样做的效果是，先找到包含文件的父文件夹，然后向上返回一级，以找到正确的路径。

你将在第 8 章中看到一个使用示例，其中一个包含文件需要包含另一个位于不同文件夹中的文件。

## 包含文件的安全注意事项

包含文件是 PHP 的一个非常强大的特性。这种强大也带来了安全风险。只要外部文件可访问，PHP 就会包含它，并将其中的任何代码整合到主脚本中。但正如本章前面提到的，包含文件可以位于任何位置。从技术上讲，它们甚至可以在不同的服务器上。然而，这被认为是一种安全风险，以至于在 PHP 5.2 中引入了一个新的配置指令 `allow_url_include`。默认设置为 `Off`，因此除非你能完全控制服务器的配置，否则现在无法包含来自不同服务器的文件。与 `include_path` 不同，`allow_url_include` 指令除了服务器管理员之外，其他任何人都无法覆盖。

即使你同时控制着两个服务器，也绝不应该包含来自其他服务器的文件。攻击者有可能伪造地址，并尝试在你的站点上执行恶意脚本。
