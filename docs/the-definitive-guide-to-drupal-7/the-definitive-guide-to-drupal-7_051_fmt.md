# 为模块指定临时名称

既然你已经明确了方法并有示例可循，现在可以开始编码了。这就引出了第一个问题：如何命名模块？（你可以将函数直接添加到胶水代码模块中，但在开始主要的新功能时，最好在一个独立的模块中开始——尤其是当你可能将代码贡献给 Drupal 社区时。）如果灵感没有立刻涌现，为了避免在模块尚不存在时就耗费精力想出一个完美的名称，你可以先使用一个临时名称。

不过，选择临时名称时也有一些指导原则。如果你将模块命名为"tip"，后来决定将其贡献到`drupal.org`，你会想要重命名它，以免占用"tip"这个常见词汇——特别是当它并非对该模块功能的特别贴切描述时。问题是，对这些常见的三个字母进行查找和替换操作可不会太轻松。

即使对于你计划贡献的模块，临时名称也应遵循与自定义模块相同的做法，用你的站点项目名称作为命名空间。你的项目名称加上你选择的任何特色词汇，应该能让你通过查找和替换轻松修改所有函数名。

**提示** 本模块在其临时名称中使用了下划线，但某些方面（例如 CSS 类和 ID，历史上使用连字符而非下划线）如果临时名称和最终选定的名称都避免使用下划线会更方便。

那么开始吧！创建一个以你选择的模块名称命名的目录（本例中为`dgd7_tip`），并开始创建必要的模块文件，这些文件也以模块名称命名，从`dgd7_tip.info`开始（参见清单 33-23 和 33-24）。

***清单 33-23.** 使用 Vim 创建模块目录和.info 文件的命令行步骤*

```
cd sites/all/modules/custom
mkdir dgd7_tip
cd dgd7_tip/
vi dgd7_tip.info
```

***清单 33-24.** dgd7_tip.info 的初始工作内容*

```
name = Tip formatter
description = [dgd7_tip] 用于提示、注释、建议和其他强调段落文本的文本格式过滤器。
core = 7.x
```

**注意** 本文作者认为，管理员和开发者能够在`admin/modules`页面上看到模块的系统（或机器）名称是一项基本的可用性要求，并将在描述中继续包含系统名称，直到这一基本功能被核心接受。这种风格并非公认的实践，所以只有当你同样强烈认同时才遵循。

现在创建一个`.module`文件，并为其添加第一个钩子，即`hook_filter_info()`的实现（参见`api.drupal.org/hook_filter_info`），该实现改编自 Filter 示例模块，如清单 33-25 所示。

***清单 33-25.** dgd7_tip.module 的初始内容*

```
/**
 * 实现 hook_filter_info()。
 */
function dgd7_tip_filter_info() {
  $filters = array();
  $filters['dgd7_tip'] = array(
    'title' => t('Tip formatter'),
    'description' => t('允许使用简单符号来标记需要强调为提示、注释、建议或其他特殊插叙内容的文本段落。'),
    'process callback' => '_dgd7_tip_process',
    'tips callback' => '_dgd7_tip_tips',
  );
  return $filters;
}

/**
 * 实现过滤器处理回调。
 */
function _dgd7_tip_process($text, $filter) {
  return $text;
}

/**
 * 实现过滤器提示回调。
 */
function _dgd7_tip_tips($filter, $format, $long = FALSE) {
  $tips = '';
  return $tips;
}
```

搞定！看起来整洁干净。即使启用了你的模块，它也不会出现任何未定义函数的错误——尽管过滤器回调函数只是桩函数，因此它们也不会做任何事情。（在`hook_filter_info()`定义和`filter_example.module`中的'prepare callback'对于简单过滤器来说被证明是不必要的；它对于需要在其他过滤器工作之前对内容进行修改的复杂过滤很有用。本文作者最初为其创建了一个桩函数，但后来从示例中移除了。警告：空的 prepare 或 process 回调，或者由于任何原因未返回值的回调，将导致应用该输入格式的任何位置内容为空！）

**注意** 本模块只提供一个过滤器，但通过使用不同的键和值重复`$filters['dgd7_tip']`数组，它可以提供更多过滤器。如果你确实打算提供多个输入过滤器，键不应该是你的模块名称`dgd7_tip`，而应该是模块名称*加上*对该特定过滤器的描述性内容。相同的约定也适用于回调名称。本模块不打算提供其他输入过滤器，因此提前使用显式函数名还为时过早。（process 和 tips 回调是内部函数，仅供模块内部使用，如前缀下划线所示。对于任何 API 函数——可能被其他模块调用的函数——你可能需要采取预防措施，选择具有前瞻性的名称。）
