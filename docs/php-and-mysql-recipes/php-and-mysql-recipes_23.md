# 第 1 章：安装与配置

自《*PHP 5 实战：问题解决方案*》出版以来已过去约十年，PHP 以及整个网络技术在此期间取得了长足发展。是时候更新部分实战方案了。

本书中的示例代码专为当前版本（PHP 5.6 和 PHP 7）设计，但多数示例也可在更早的 PHP 5 版本上运行。PHP 7 仍较新，许多站点所有者尚未升级。PHP 7 基本兼容 PHP 5.6，但确实包含一些新功能和少数向后兼容问题。并非所有扩展都已更新以适配新的 Zend 引擎（PHP 的内部运行机制）。

强烈建议使用最新版本的 PHP。这将确保最佳性能和安全性，并能访问最新功能。

PHP 最初被设计为一种脚本语言，用于生成动态 Web 内容，其形式是对客户端（通常是浏览器）通过 HTTP 协议向 Web 服务器发出的请求进行响应。如今，浏览器已不再是这些脚本的唯一客户端类型。在越来越多数据通过返回 JSON、XML 或其他结构化格式的 Web 服务访问的世界里，我们看到客户端可以是任何东西：从智能手机上的应用程序、其他应用程序，乃至相互通信以交换数据的服务器。

## 方案 1-1：安装 PHP

### 问题描述

PHP 是一种开源脚本语言，已被移植到许多不同平台。它能运行在从树莓派到大型 IBM 主机的任何设备上。过去安装 PHP 需要用户从源代码编译解释器，但现在许多操作系统都提供了便于安装的软件包。无论你选择使用哪种操作系统和 Web 服务器，问题都是一样的：如何让 Web 服务器执行脚本并返回响应？

### 解决方案

所支持的操作系统、Web 服务器和服务器 API 的数量相当庞大，而不同的组合更是数不胜数。互联网上使用 PHP 作为脚本语言的大多数 Web 服务器都运行着某种形式的 Linux，也有一部分运行 Windows。在开发环境方面，常见的有 Windows、Mac OSX 和 Linux 系统。除此之外，至少还有三种类型的 Web 服务器：Apache、Nginx 和 Windows 上的 Internet 信息服务（IIS）。

所有这些配置共同要解决的核心问题，是将脚本解析器注入 Web 服务器，使得对某些文件的请求能够触发脚本执行并返回脚本生成的输出。对于 Web 服务器的其他类型请求，例如静态 HTML、图像、JavaScript 和 CSS 文件等，则由 Web 服务器直接处理——服务器会将请求解析为文件系统上的文件，并将原文件返回给请求者。

通常的配置是告诉 Web 服务器，具有特定文件扩展名（`.php`）的文件应由 PHP 解释处理。当收到对此类文件的请求时，Web 服务器会将请求传递给 PHP，并等待输出。

### 工作原理

鉴于操作系统、Web 服务器以及将 PHP 链接到 Web 服务器的方式可能存在大量组合，我选择了少数几种组合在以下示例中进行描述。第一个示例介绍如何使用包管理工具 `yum` 在全新的 CentOS 7 操作系统上安装 Apache 和 PHP。后续示例将展示如何在 Windows 上执行相同操作。

包管理器 `yum` 用于从 CentOS 仓库安装、更新或删除软件包，也可用来列出可用软件包。在安装新软件包之前，确保系统已完全更新始终是一个好习惯。这可以通过运行以下命令来实现：

```
yum update
```

这将确保所有内核和应用程序包都更新到最新版本。安装新内核后，必须重启系统。
