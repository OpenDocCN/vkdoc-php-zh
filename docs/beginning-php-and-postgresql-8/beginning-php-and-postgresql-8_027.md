# PEAR

好的程序员编写可靠的代码，而伟大的程序员重用优秀程序员的代码。对于 PHP 程序员来说，*PEAR*（`http://pear.php.net`），即 *PHP Extension and Application Repository*（PHP 扩展与应用库）的缩写，是查找和重用优秀 PHP 代码最有效的手段之一。受 Perl 广受欢迎的 CPAN（`http://www.cpan.org`）启发，该项目于 1999 年由著名 PHP 开发者 Stig Bakken 发起，第一个稳定版本随 PHP 4.3.0 一起发布。正式定义上，PEAR 是一个用于可重用 PHP 组件的框架和分发系统，目前提供 442 个*包*，分为 41 个不同主题（并且还在不断增加）。由于 PEAR 的贡献在被接受之前都经过社区的仔细审查，因此代码质量和遵守 PEAR 标准开发指南是有保证的。此外，由于许多 PEAR 包逻辑地实现了无论何种应用程序类型都会反复出现的常见任务，利用这个社区驱动的服务将为你节省大量的编程时间。

本章致力于对 PEAR 的全面讨论，提供以下主题：

- 几个流行 PEAR 包的概述，旨在让你了解这个仓库到底有多有用。
- 通过 PEAR 控制台安装和管理 PEAR 包的说明。
- 关于 PEAR 编码和文档指南的讨论，这不仅对构建通用应用程序有用，对审查和提交 PEAR 包也很有用。
- PEAR 提交流程的概述，以防你有兴趣向仓库做出自己的贡献。

### 流行的 PEAR 包

为了让你了解 PEAR 包到底有多受欢迎，在撰写本文时，托管的包迄今已被下载了近 1400 万次！事实上，有几个包非常受欢迎，以至于开发者从 4.0 版本开始就将它们默认包含在内。下面列出目前包含的包：



• `Archive_Tar`：`Archive_Tar` 包简化了 tar 文件的管理，提供了创建、列出、提取和向 tar 文件中添加内容的方法。此外，它还支持 Gzip 和 Bzip2 压缩算法，前提是安装了相应的 PHP 扩展。PEAR 的正常运行需要此包。

• `Console_Getopt`：通过在命令行执行时提供选项来修改脚本的行为通常很有用。例如，你可以通过向 `pear` 命令传递 `-V` 来验证已安装的 PEAR 版本：

```
%>pear -V
```

`Console_Getopt` 包提供了一种标准方法来读取这些选项，并在提供的语法不符合某些预定义规范（例如特定参数是否需要参数）时向用户提供错误消息。PEAR 的正常运行需要此包。

• `DB`：`DB` 包提供了一个面向对象的查询 API，用于抽象与数据库层的通信。这让你能够方便地将应用程序从一个数据库透明地迁移到另一个数据库，甚至可能像修改一行代码一样简单。目前支持 12 种数据库，包括：dBase、FrontBase、Informix、InterBase、Mini SQL、Microsoft SQL Server、MySQL、Oracle、ODBC、PostgreSQL、SQLite 和 Sybase。

• `Mail`：编写一个能够发送电子邮件的可移植 PHP 应用程序可能比你想象的要棘手，因为并非所有操作系统都提供相同的设施来支持此功能。例如，默认情况下，PHP 的 `mail()` 函数依赖于 `sendmail` 程序（或 `sendmail` 包装器），但 `sendmail` 在 Windows 上不可用。为了解决这种不兼容性，可以指定 SMTP 服务器的地址并通过它发送邮件。然而，你的应用程序如何确定哪种方法可用呢？`Mail` 包通过提供一个用于发送邮件的统一接口解决了这个难题，而无需修改 PHP 的配置。它支持从 PHP 应用程序发送电子邮件的三种不同后端（PHP 的 `mail()` 函数、`sendmail` 和 SMTP 服务器），并包含一个用于验证电子邮件地址语法的方法。通过使用简单的应用程序配置文件或基于 Web 的偏好表单，用户可以指定最适合其需求的方法。

• `Net_Socket`：`Net_Socket` 包用于通过提供一个用于执行连接以及在这些套接字之间读取和写入信息的通用 API 来简化 TCP 套接字的管理。

• `Net_SMTP`：`Net_SMTP` 包提供了 SMTP 协议的实现，使你可以轻松执行诸如连接和断开与 SMTP 服务器的连接、执行 SMTP 身份验证、识别发件人以及发送邮件等任务。

• `PEAR`：PEAR 的正常运行需要此包。

• `PHPUnit`：单元测试是一种特定的测试方法，用于确保代码块（或单元）的正确运行，通常针对类或函数库。`PHPUnit` 包通过指定一组通用的结构指南和一种自动化测试的方法，促进了单元测试的创建、维护和执行。
