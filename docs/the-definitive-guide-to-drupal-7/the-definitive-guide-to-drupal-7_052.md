# 为你的模块启动仓库

由于此模块可能独立于整个网站项目存在，你可以为其启动一个独立的版本控制仓库。使用 Git（更多信息和资源请参见第 2 章和第 37 章），只需在命令行中输入几个单词，如清单 33-26 所示。在你创建的`dgd7_tip`目录内执行此操作。

***清单 33-26.** 启动仓库并对临时命名的 dgd7_tip 模块进行初始提交的命令行步骤*

```
git init
git add .
git commit -m "包含桩过滤器 API 函数的.info 和.module 文件。"
```

你现在可以持续提交（如第 14 章所提倡的那样），无需提交消息，或者当你有特别需要记录的内容时再添加消息。本章代码中将不再标注`git add`命令和提交消息，但你可以确信，在每次重大更改以及许多微小的更改之后，都会有一次提交。

**注意** 本节创建的模块可从在线章节笔记`dgd7.org/other90`获取链接，你可以查看对它的每次提交。
