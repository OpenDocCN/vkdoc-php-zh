# 管理员身份验证

由于只希望特定用户访问目录管理页面，需要实施某种安全机制来控制对站点敏感页面的访问。

实现安全性需要处理两个重要概念：**认证**和**授权**。**认证**是唯一标识用户的过程（通常通过提供用户名和密码），而**授权**则指已认证用户可以访问哪些资源。

想要访问目录管理页面的用户应首先进行身份认证。在确认用户身份后，再决定该用户是否有权访问管理页面。

在 HatShop 中，将使用一种称为**HTTP 认证**的身份验证方法，它允许通过 HTML 表单控制登录过程。客户端认证后，PHP 会自动在客户端生成一个 cookie，用于验证所有后续请求。如果找不到该 cookie，客户端将显示登录 HTML 表单。

> **注意**：我们假设管理员从启用了 cookie 的客户端访问管理页面。

用户名和密码的组合可以以多种方式物理存储。例如，在第 11 章中，你将看到如何在数据库中存储经过哈希处理（加密）的客户密码。

> **提示**：**哈希**是存储密码的常用方法。密码的哈希值是通过对其应用数学函数（哈希算法）计算得出的。当用户尝试认证时，输入的密码会被哈希，生成的哈希值会与原始（正确）密码的哈希值进行比较。如果两个值相同，则输入的密码正确。哈希算法的一个基本特性是，理论上无法从哈希值中获取原始密码（该算法是单向的）。实际上，科学家最近发现了流行的`MD5`、`SHA-0`和`SHA-1`哈希算法的漏洞。

更简单的方法是将用户名和密码组合存储在 PHP 文件中。这种方法不如使用数据库灵活，但快速且易于实现。

将用户名/密码组合存储在脚本文件中时，可以选择以明文或使用哈希算法（如`MD5`或`SHA-1`）对密码进行哈希后存储。

在以下练习中，你将简单地将密码以明文形式存储，但了解还有其他选择也是好的。第 11 章将详细介绍哈希相关知识。
