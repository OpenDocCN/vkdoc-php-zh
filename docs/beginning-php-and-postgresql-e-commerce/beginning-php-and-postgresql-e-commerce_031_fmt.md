# 为增长而设计

在 Web 应用的语境下，“设计”一词可以指代很多方面。它最流行的用法可能指的是网站的视觉和用户界面设计。

这方面至关重要，因为，坦率地说，访问者通常更看重网站的外观和易用性，而不是后台使用了哪些技术或 Web 服务器运行的是什么操作系统。如果网站运行缓慢、难以使用或容易被遗忘，那么无论用什么尖端技术来创建它都无关紧要。

不幸的是，这种现实使得许多缺乏经验的程序员低估了网站不可见部分（代码、数据库等）实现方式的重要性。网站的视觉部分最初能吸引访问者，但功能性能让他们再次光顾。一个网站有时可以根据初始需求快速实现，但如果没有正确的架构，后续变更将会变得困难甚至不可能。

对于任何规模的项目，在开始编码之前都必须做一些准备工作。尽管如此，无论做了多少准备和设计工作，意外总会发生，隐藏的问题、新的需求和不断变化的规则似乎总是在与截止日期作对。

即使没有这些意外因素，网站设计者通常也会被要求，在项目完成并部署后，多次更改或添加新功能。`HatShop`也是如此，它将分三个独立的阶段实施，这在第 1 章中已经讨论过。

你将学习如何创建网站，以便在功能扩展或进行更新时，网站（或你）不会崩溃。因为这是一本关于编程的书，所以我们将重点关注如何设计使它们工作的代码，而不是聚焦于如何设计用户界面或营销技巧。

“设计代码”这个短语可能有不同的含义；例如，我们需要简短地讨论一下命名约定。尽管如此，我们需要考虑的最重要的方面是应用架构。架构指的是将单一功能（例如产品搜索功能）的代码拆分成更小组件的方式。虽然以尽可能快和简单的方式在单个组件中实现该功能可能更容易，但通过创建更多协同工作以实现预期结果的组件，你可以获得巨大的长期优势。

在讨论架构本身之前，你必须确定你希望从这个架构中获得什么。
