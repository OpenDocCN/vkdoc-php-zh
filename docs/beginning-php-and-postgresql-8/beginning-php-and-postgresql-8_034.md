# 第 12 章 日期和时间

### 日期函数

一些职业拳击手永远无法达到他们运动的顶尖水平，因为他们是单一维度的。也就是说，他们过于依赖自己战斗技巧中的某一特定方面，例如左勾拳。真正世界级的拳击手会充分利用他们掌握的一切，组合使用各种招式来进攻、消耗并最终击败对手。

这类似于有效使用日期函数：虽然有时只需要一个函数就足够了，但通常当你将两个或三个函数组合使用以产生预期结果时，它们的真正威力才会显现出来。本节将演示几种最常被请求的与日期相关的“招式”（任务），其中一些只涉及一个函数，而另一些则需要组合使用多个函数。

#### 显示本地化的日期和时间

在本章以及本书中，通常使用美国化的时间和货币格式，例如`04-12-05`和`$2,600.93`。然而，世界其他地区使用不同的日期和时间格式、货币，甚至字符集。鉴于互联网的全球覆盖范围，您可能需要创建一个能够适应外国（即*本地化*）格式的应用程序。事实上，忽略这一点可能会引起相当大的混乱。例如，假设您要创建一个为佛罗里达州奥兰多市一家热门酒店预订房间的网站。这家酒店特别受来自其他国家的游客欢迎，因此您决定创建该网站的多个本地化版本。您应该如何处理大多数国家使用其自己的货币和日期格式（更不用说不同的语言）这一事实？

虽然您可能费尽心思去创建一种繁琐的方法来管理这些事务，但这很可能容易出错，并且需要一些时间来部署。值得庆幸的是，PHP 提供了一组内置功能来本地化此类数据。

PHP 不仅可以方便地格式化日期、时间、货币等，还可以相应地翻译月份名称。在本节中，您将学习如何利用此功能，根据您想要的任何区域设置来格式化日期。这样做基本上需要两个函数：`setlocale()`和`strftime()`。下面将介绍这两个函数，并附上几个示例。

##### `setlocale()`

`string setlocale (mixed *category*, string *locale* [, string *locale*...]) string setlocale (mixed *category*, array *locale*)`

`setlocale()`函数通过为区域设置分配适当的值来更改 PHP 的本地化默认值。本地化字符串官方遵循以下结构：`language_COUNTRY.characterset`

例如，如果您想使用意大利语本地化，区域设置字符串应设置为`it_IT`。以色列语本地化应设置为`he_IL`，英国英语本地化为`en_GB`，美国英语本地化为`en_US`。当给定区域设置可能有多个字符集可用时，就会涉及到字符集组件。例如，区域设置字符串`zh_CN.gb18030`用于处理藏文、维吾尔文和彝文字符，而`zh_CN.gb3212`则用于简体中文。

您可以看到，区域设置参数可以作为几个不同的字符串传入，也可以作为区域设置值的数组传入。但为什么要传入多个区域设置呢？此功能（自 PHP 版本`4.2.0`起引入）是为了应对不同操作系统之间区域设置代码的差异。鉴于绝大多数 PHP 驱动的应用程序都针对特定平台，这很少会成为问题；然而，在您需要时，此功能可用。



最后，如果你在 Windows 上运行 PHP，请记住，微软显然是为了让我们保持警惕，也设计了自己的一套本地化字符串。你可以从 `http://msdn.microsoft.com` 获取语言和国家/地区代码列表。

**提示** 在某些基于 Unix 的系统中，你可以通过运行以下命令来确定支持哪些区域设置：`locale -a`。

可以为特定的数据分类指定区域设置。支持以下六种不同的类别：

-   `LC_ALL`：为以下所有五个类别设置本地化规则。
-   `LC_COLLATE`：字符串比较。这对于使用 â 和 é 等字符的语言很有用。
-   `LC_CTYPE`：字符分类与转换。例如，设置此类别可以让 PHP 使用 `strtolower()` 函数正确地将 â 转换为其对应的大写形式 Â。
-   `LC_MONETARY`：货币表示形式。例如，美国将 50 美元表示为 $50.00，而意大利则将 50 欧元表示为 50,00。
-   `LC_NUMERIC`：数字表示形式。例如，美国将一千四百一十二表示为 1,412.00，而意大利则将其表示为 1.412,00。
-   `LC_TIME`：日期和时间表示形式。例如，美国表示日期时，月份在前，然后是日，最后是年。例如，2005 年 2 月 12 日可能表示为 02-12-2005。然而，欧洲（以及世界大部分其他地区）将这个日期表示为 12-02-2005。设置好之后，你可以使用 `strftime()` 函数生成符合本地化要求的格式。

例如，假设我们正在处理货币值，并希望确保金额按照意大利语区域设置进行格式化：

```
setlocale(LC_MONETARY, "it_IT");
echo money_format("%i", 478.54);
```

将返回：

```
EUR 478,54
```

要本地化日期和时间，你需要结合使用 `setlocale()` 和接下来介绍的 `strftime()`。

#### strftime()

`string strftime (string $format [, int $timestamp])`

`strftime()` 函数根据 `setlocale()` 指定的本地化设置来格式化日期和时间。虽然它使用与 `date()` 相同的格式，接受用于确定请求日期和时间布局的转换参数，但不幸的是，其参数与 `date()` 使用的参数不同，因此有必要在**表 12-2** 中列出所有可用参数供你参考。请记住，所有参数都将根据所设置的区域设置生成输出。另外，请注意某些参数在 Windows 上不受支持。

##### 本地化日期格式化

通过将`strftime()`与`setlocale()`结合使用，可以根据用户所在地区的语言、标准和习惯来格式化日期。回顾旅游网站的例子，可以轻松地为用户提供包含旅行日期和机票价格的本地化行程单：

Benvenuto abordo, Sr. Sanzi

```php
<?php
setlocale(LC_ALL, "it_IT");
$tickets = 2;
$departure_time = 1118837700;
$return_time = 1119457800;
$cost = 1350.99;
?>
Numero di biglietti: <?php echo $tickets; ?><br />
Orario di partenza: <?php echo strftime("%d %B, %Y", $departure_time); ?><br />
Orario di ritorno: <?php echo strftime("%d %B, %Y", $return_time); ?><br />
Prezzo IVA incluso: <?php echo money_format('%i', $cost); ?><br />
```

此示例返回以下内容：

Benvenuto abordo, Sr. Sanzi
Numero di biglietti: 2
Orario di partenza: 15 giugno, 2005
Orario di ritorno: 22 giugno, 2005
Prezzo IVA incluso: EUR 1.350,99

#### 显示网页的最后修改日期

万维网诞生不过十年，却已开始变得像囤积狂的办公室。文档散落各处，许多都是过时、陈旧甚至完全无关的。帮助访问者判断文档有效性的常见策略之一是在页面上添加时间戳。当然，手动执行此操作难免会出错，因为页面管理员最终会忘记更新时间戳。不过，可以使用`date()`和`getlastmod()`来自动化此过程。您已经熟悉`date()`，因此这里将介绍`getlastmod()`。

`getlastmod()`

```php
int getlastmod()
```

`getlastmod()`函数返回页面的`Last-Modified`标头值，如果出错则返回`FALSE`。如果将其与`date()`结合使用，提供有关页面最后修改时间和日期的信息就变得非常简单：

```php
$lastmod = date("F d, Y h:i:sa", getlastmod());
echo "Page last modified on $lastmod";
```

这将输出类似以下的内容：

```
Page last modified on April 26, 2005 07:59:34pm
```

#### 确定当前月份的天数

要确定当前月份的天数，请使用`date()`函数的`t`参数。考虑以下代码：

```php
printf("There are %d days in %s.", date("t"), date("F"));
```

如果是在四月执行，则会输出以下结果：

```
There are 30 days in April.
```

#### 确定任意指定月份的天数

有时您可能想确定非当前月份的天数。单独使用`date()`函数行不通，因为它需要一个时间戳，而您可能只有月份和年份。不过，可以将`mktime()`函数与`date()`结合使用来产生所需结果。假设您想确定 2006 年二月的天数：

```php
$lastday = mktime(0, 0, 0, 3, 0, 2006);
printf("There are %d days in February, 2006.", date("t",$lastday));
```

执行此代码片段将产生以下输出：

```
There are 28 days in February, 2006.
```

#### 计算从当前日期起 X 天后的日期

确定未来或过去某个特定天数后的准确日期通常很有用。使用`strtotime()`函数和 GNU 日期语法，这类请求很容易实现。假设您想基于当前日期（2005 年 4 月 23 日）知道 45 天后的日期：

```php
$futuredate = strtotime("45 days");
```



`echo date("F d, Y", $futuredate);`

返回结果为：

`June 07, 2005`

通过在参数前面加上负号，可以计算出 45 天前的日期：`$pastdate = strtotime("-45 days");`

`echo date("F d, Y", $pastdate);`

返回结果如下：

`March 09, 2005`

那 10 周零 2 天后的日期呢？

`$futuredate = strtotime("10 weeks 2 days");`

`echo date("F d, Y", $futuredate);`

返回结果为：

`July 04, 2005`
