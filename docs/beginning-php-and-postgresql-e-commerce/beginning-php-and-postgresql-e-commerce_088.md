# 信用卡交易

在启动电子商务网站之前，您需要做的最后一件事情是实现信用卡处理功能。在本章中，我们将研究如何将此功能构建到您在上一章创建的管道中。

我们首先探讨信用卡交易背后的理论、帮助您实现信用卡处理的各类组织，以及可能实现的交易类型。接着，我们将以两个组织为例，讨论它们交易 API（应用程序编程接口，即您访问信用卡交易功能的手段）的具体细节。之后，您将构建一个新的类库，通过一些简单的测试代码来使用这些交易 API 之一。

最后，您将把该 API 与 HatShop 电子商务应用程序及订单处理管道集成。

## 信用卡交易基础

银行和其他金融机构使用基于 X.25 协议的安全网络进行交易，而非 TCP/IP（传输控制协议/互联网协议，这是数据在互联网上传输的主要方式）。关于 X.25，您无需了解太多，只需知道它是一种与 TCP/IP 不兼容的不同网络协议即可。因此，X.25 网络与互联网完全隔离。虽然有可能直接接入这些网络，但这不太可能是一个合理的选择。要做到这一点，您可能需要与您要使用的网络所有者进行严肃的谈判。该所有者会希望完全确定您是一位可靠的客户，有能力实施必要的安全措施来防止对其系统的攻击。因此，网络所有者不会轻易向任何人发放这些许可，因为大多数人负担不起所需的安全措施（包括将服务器锁在笼子里、每天通过安全滑道发送备份磁带、需要三个持有不同钥匙的人才能访问这些磁带，等等）。

另一种选择是通过网关提供商来访问这些网络。这使您能够通过互联网（使用安全协议）执行信用卡交易协议中属于您的那部分操作，同时依赖您选择的网关与 X.25 网络进行通信。

虽然这可能会产生一些费用，但网关提供商应与金融机构达成协议以保持低成本，并将节省的费用转嫁给您（在网关扣除其份额之后），因此这很可能比拥有自己的 X.25 连接要便宜得多。这种方法也可能比使用像 PayPal 这样的第三方更便宜，因为当您自行处理订单管道时，只需要最基本的功能。例如，您无需使用像 PayPal 这样的公司提供的所有订单审计功能，因为您已经在上一章中构建了所有这些功能。

## 使用信用卡支付网关

要与网关组织合作，您首先需要开设一个商户银行账户。在大多数银行都可以办理，您将获得一个商户 ID，用于向网关注册。下一步是找到一个合适的网关。不幸的是，这可能是一项艰巨的工作！

虽然找到网关并不难，但挑战在于找到一个能力胜任且能以您可接受的价格和质量提供服务的网关。实际上，有数百家公司渴望从您的销售额中分一杯羹。在互联网上快速搜索“信用卡网关”会产生一长串结果。这些公司的网站大部分是纯粹的宣传手册——您会发现自己读了好几页文字，内容无非是宣扬他们如何最出色、最安全地从事其业务，但最终只会得到一个需要填写的表格，以便客户服务代表可以致电您“讨论您的需求”。从长远来看，您可以放心，至少您很可能只需要经历一次这个过程。
