# 注意：十进制数与二元运算符一起使用时会被自动转换为二进制。也可以使用二进制表示法来指定用于运算的二进制数。

```
$x = 0b101 & 0b100; // 0b100 (4)
```

### 运算符优先级

当一个表达式包含多个运算符时，这些运算符的优先级决定了它们的求值顺序。优先级顺序见表 3-1。

**表 3-1. 运算符优先级顺序**

| 优先级 | 运算符 | 优先级 | 运算符 |
| --- | --- | --- | --- |
| `**` | | `&` |
| `++ --` | | `^` |
| `∼ - (unary)` | | `&#124;` |
| `!` | | `&&` |
| `* / %` | | `&#124;&#124;` |
| `+ - (binary)` | | `= op=` |
| `<< >>` | | `and` |
| `< <= > >= <>` | | `xor` |
| `== != === !== <=>` | | `or` |

例如，乘法优先级高于加法，因此在以下代码行中会先计算乘法。

```
$x = 4 + 3 * 2; // 10
```

可以使用括号强制改变优先级。括号内的表达式会在该语句中的其他表达式之前被求值。

```
$x = (4 + 3) * 2; // 14
```

## 额外的逻辑运算符

在优先级表中，请特别注意最后三个运算符：`and`、`or`和`xor`。`and`和`or`运算符的工作方式与逻辑运算符`&&`和`||`相同。唯一的区别是它们的优先级更低。

```
// 等同于：$a = (true && false);
$x = true && false; // $x 为 false
// 等同于：($a = true) and false;
$x = true and false; // $x 为 true
```

`xor`运算符是按位`^`运算符的布尔版本。如果只有一个操作数为`true`，则计算结果为`true`。

```
$x = (true xor true); // false
```
