# 第 5 章

## 缓存

在处理大量数据时，我们经常会遇到性能瓶颈。这些瓶颈可能由多种原因引起，而良好的缓存大多可以避免这些问题。如果你不清楚我所说的缓存是什么意思，我们稍后会通过一个例子来说明。

## 目标

- 首先，我们应该认识到使用缓存的必要性。除非我们理解缓存所解决的问题，否则我们无法充分发挥其潜力。

- 我们需要学习如何访问`Memcached`服务器，我们将用它来存储缓存数据。这需要我们编写与`Memcached`服务器交互的类。

## 性能瓶颈

设想一个社交网络，其中任何用户都可以拥有许多好友。如果要在网页上显示这些好友的列表及其所有相关信息，该页面必须进行数据库调用，以从数据库中检索好友记录。现在想象一下，这个好友列表会显示在网站的每个页面上。

这意味着网站的每个页面至少需要一次数据库查询。再加上成千上万的用户，每个用户的好友列表也在显示，最终你会得到海量的数据。

在任何增长或需求超过资源限制的应用中，这种数据需求很容易使应用不堪重负。那么，我们该如何利用缓存来减轻数据库的压力呢？

我们可以在首次从数据库检索好友列表时，轻松地将其缓存起来。这意味着网站无需频繁、重复地请求相同的数据。与其每次需要时都从数据库获取相同的数据，不如先检查数据是否已存储在缓存中。

回顾我们在第 4 章中创建的配置类。如果我们经常需要使用配置设置，那么将其缓存起来也是合理的。否则，我们将不得不为网站上的每个页面、每个用户反复加载相同的配置文件。

如果我们一次性加载配置数据，将其存储在缓存中，然后后续都从缓存中请求配置数据，我们将节省大量时间。

> **注意** 缓存的用途不仅限于存储数据库和配置数据。这两个例子只是我用来说明缓存效能的示例。凡是需要从比缓存系统慢的任何地方重复获取相同数据的情况，缓存都是有用的。
