# 第三部分：第三阶段开发

## 第 11 章：管理客户详细信息

在开发的前两个阶段，你已经构建了一个基本（但功能完整）的网站，并将其连接到 PayPal 以接收付款和确认订单。在本书的第三部分，你将更进一步。通过将 PayPal 从订单流程中移除，你可以获得更好的控制权并减少开销。这并不像你想象的那么复杂，但必须小心谨慎地做好。第 11 章通过实现一个客户账户系统，并研究交换和存储客户及信用卡详细信息的安全方面，为后续工作奠定了基础。

## 第 12 章：存储客户订单

在本章中，你将进行必要的修改，使客户能够下订单并与他们的用户资料相关联。这里的主要修改是，与订单关联的客户将通过 `orders` 表中的一条新信息来识别，其余大部分修改将用于应用此信息。

同样在本章中，你还将了解如何处理电子商务网站的另一个常见功能：税费和运费。实现此功能有多种选择，但我们只探讨一种简单的方法，并为你自己的进一步开发奠定基础。

## 第 13 章：实现订单处理管道：第一部分

HatShop 电子商务应用正在逐步成型。你已经添加了客户账户功能，并安全地跟踪和存储了客户地址及信用卡信息。但是，你当前并未使用这些信息——而是将这部分责任委托给了 PayPal。在本章和下一章中，你将构建自己的订单处理管道，处理信用卡授权、库存检查、发货、发送电子邮件通知等事宜。我们将把信用卡处理的具体细节留到第 15 章，但在此之前，我们会向你展示此流程在何处适用。

## 第 14 章：实现订单处理管道：第二部分

在本章中，你将添加所需的管道部分，以便从头到尾处理订单，不过要到下一章才会添加完整的信用卡交易功能。我们还将通过修改本书前面添加的订单管理页面，来考虑新的订单处理系统，从而研究订单的 Web 管理。

## 第 15 章：信用卡交易

在启动电子商务网站之前，你需要做的最后一件事是启用信用卡处理功能。在本章中，我们将探讨如何将其构建到你在第 13 章和第 14 章创建的管道中。你将学习如何使用两种流行的信用卡支付网关来实现这一点：DataCash 和 Authorize.net。到本章结束时，HatShop 将成为一个功能齐全、安全且可用的电子商务应用。

## 第 16 章：产品评论

至此，你已经拥有一个完整且功能齐全的电子商务网站。但这并不妨碍你为其添加更多功能，使其对访问者更加有用和愉悦。

通过在你的网站中添加产品评论系统，你可以增加访问者回访网站的机会，无论是为他们购买的产品撰写评论，还是查看其他人对该产品的看法。

## 第 17 章：连接到 Web 服务

在动态的互联网世界中，仅仅拥有一个重要的网络存在有时是不够的；你还需要与第三方提供的功能进行交互以实现你的目标。在本书中，到目前为止你已经看到了如何集成外部功能来处理客户付款。在第 17 章中，你将学习如何使用亚马逊网站的功能，通过 Web 服务来实现。Web 服务通过使用 HTTP 等标准互联网协议的 Web 接口暴露出来。客户端和服务器交换的消息使用基于 XML 的协议（名为 SOAP，即简单对象访问协议）或 REST（表述性状态传递）进行编码。这些消息通过 HTTP 发送。

在本章中，你将进一步了解这些技术。
