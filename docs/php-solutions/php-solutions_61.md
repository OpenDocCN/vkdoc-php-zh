# 16. 管理多个数据库表

上一章向你展示了如何使用 `INNER JOIN` 和 `LEFT JOIN` 检索存储在多个表中的信息。你还学习了如何通过向子表添加额外的列并单独更新每条记录以插入外键来链接现有表。然而，大多数时候你会希望同时在两个表中插入数据。这带来了一个挑战，因为 `INSERT` 命令一次只能操作一个表。你需要按照正确的顺序处理插入操作，从父表开始，这样你就可以获取新记录的主键，并与其他详细信息同时插入到子表中。在更新和删除记录时，也需要考虑类似的问题。涉及的代码并不难，但在构建脚本时，你需要清晰地将事件顺序牢记在心。

本章将指导你完成在 `blog` 表中插入新文章的过程，可选地选择关联图片或上传新图片，并将文章分配到一个或多个类别，所有这些都在一次操作中完成。然后，你将构建脚本以更新和删除文章，同时不破坏相关表的参照完整性。

你还将学习外键约束，它控制当你尝试删除在另一个表中仍存在外键关系的记录时会发生什么。并非所有数据库都支持外键约束，因此检查你的远程服务器是否支持非常重要。本章还解释了如果你的服务器不支持外键约束，你可以采取哪些措施来保护数据完整性。

具体来说，你将学习以下内容：

- 在相关表中插入、更新和删除记录
- 在记录创建后立即找到其主键
- 转换表的存储引擎
- 在 InnoDB 表之间建立外键约束
