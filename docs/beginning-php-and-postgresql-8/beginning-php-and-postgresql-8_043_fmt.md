# 第 14 章

## 身份验证

身份验证是当今 Web 应用程序中的常见做法。这不仅出于安全相关的原因，还为了根据用户偏好和类型提供个性化功能。通常，系统会提示用户输入用户名和密码，这两者的组合构成了该用户的唯一标识值。在本章中，你将学习如何使用 PHP 内置的身份验证功能来提示并验证这些信息。

具体来说，本章将学习以下内容：

- 基本的基于 HTTP 的身份验证概念

- PHP 的身份验证变量，即`$_SERVER['PHP_AUTH_USER']`和`$_SERVER['PHP_AUTH_PW']`

- 常用于实现身份验证程序的几个 PHP 函数

- 三种常见的身份验证方法：将登录对（用户名和密码）硬编码到脚本中、基于文件的身份验证以及基于数据库的身份验证

- 使用用户 IP 地址进一步限制身份验证凭据

- 利用 PEAR 的`Auth_HTTP`包

- 使用`CrackLib`扩展测试密码的可猜测性

- 使用一次性 URL 恢复丢失的密码

### HTTP 身份验证概念

HTTP 协议提供了一种相当简单但有效的用户身份验证手段，服务器用其来质询资源请求，客户端（浏览器）用其来提供与身份验证过程相关的信息。一个典型的身份验证过程如下：

1. 客户端请求一个受限资源。

2. 服务器以 401（未授权访问）响应消息来回应此请求。

3. 客户端（浏览器）识别出 401 响应，并弹出一个类似图 14-1 所示的身份验证提示窗口。大多数现代浏览器都能理解 HTTP 身份验证并提供相应的功能，包括 Internet Explorer、Netscape Navigator、Mozilla 和 Opera。

4. 如果用户提供了正确的凭据（用户名和密码），它们会被发送回服务器进行验证。随后用户被允许访问该资源。但是，如果用户提供了错误或空白的凭据，则访问被拒绝。

5. 如果用户验证通过，浏览器将身份验证信息存储在其身份验证缓存中。此缓存信息会保留在浏览器中，直到缓存被清除，或直到另一个 401 服务器响应被发送到浏览器。

你应该理解，尽管 HTTP 身份验证有效地控制了对受限资源的访问，但它并未保护身份验证信息传输的通道。也就是说，一个位置合适的攻击者可以轻易地嗅探或监控服务器与客户端之间发生的所有流量。提供的用户名和密码都包含在此流量中，且均未加密。因此，要消除通过此类方法被入侵的可能性，你需要实现一个安全的通信通道，通常通过使用安全套接字层（SSL）来实现。所有主流 Web 服务器都支持 SSL，包括 Apache 和微软 Internet 信息服务（IIS）。
