# 可访问性

**作者：Mike Gifford**

得益于过去三年近 400 人的贡献，Drupal 7 在实现普遍可访问性方面取得了巨大飞跃。Drupal 7 核心是当今可用的最易访问的内容管理系统/平台之一。其公共页面和管理页面都经过了可访问性问题审查，并且消除了许多参与障碍。Drupal 的模块化结构使用 API 和钩子将核心功能提供给主题和模块，这意味着开发者在定制站点时，核心中的许多增强功能将被继承。不幸的是，这并不意味着每个用其开发的站点都会是易访问的。

## 近期增强功能

表单 API 获得了大量增强，以确保站点的交互元素尽可能易于访问，包括以下内容：

* 跳过导航功能已引入所有核心主题，并且有处理焦点元素隐藏、不可见和可见显示设置的默认方法。
* 改进了颜色对比度和强度，以帮助低视力用户。此外，系统消息中添加了图片，为所有用户提供额外的视觉提示。
* 同样，密码管理系统也得到了增强，鼓励每个人使用强密码，包括一个使用 WAI-ARIA 的进度条，用于向屏幕阅读器提示安全级别。（有关 WAI-ARIA 的更多信息，请参阅本附录中的“引入 WAI-ARIA”部分。）
* 在大多数情况下，拖放界面会给盲人用户带来障碍。为了解决这个问题，用户可以根据需要禁用拖放重新排序功能。
* 已为 jQuery 添加的一些交互元素添加了警报。屏幕阅读器用户现在可以获知自动完成功能提供的简短列表。
* Drupal 核心现在可以由盲人和/或仅使用键盘的用户进行安装和管理。更多残障人士也能有效地查看网站内容。



### 标准是什么？

万维网联盟（W3C）的网页无障碍倡议（WAI）一直引领着无障碍技术的国际标准制定。在美国，国家标准是第 508 条；然而，该标准正在修订中，并很可能会与《网页内容无障碍指南》（WCAG）2.0 保持一致。

Drupal 是一个全球性社区，因此我们选择依据 WCAG 2.0 进行测试。我们还使用了部分《无障碍富互联网应用》（ARIA）套件草案中的选项，这些选项允许屏幕阅读器在内容动态变化时接收提醒。

WCAG 2.0 于 2008 年底发布，其范围已从网页扩展到更广泛的领域，旨在保持技术中立。这些指南并不仅仅局限于 HTML 的细节，而是提出了更通用的要求：网站应具备可感知性、可操作性、可理解性和健壮性（POUR）。这些指南中定义了许多技术方法，以帮助确保网站不会给用户带来障碍。

随着我们开始期望在更多类型的设备上获得动态、交互式的网络体验，拥有一套灵活、通用的指南变得愈发重要。这就是 WCAG 2.0 围绕 POUR 原则构建的原因，其核心要求如下：

-   所有用户都能感知你的内容吗？诸如测试色彩对比度、为图片和非文本内容提供替代文本和字幕等简单操作对此有所帮助。
-   每个人都能操作你的网站吗？人们现在可以用来与网站交互的工具，范围从口含棒和头部指针，到专用软件、键盘和屏幕阅读器。
-   你的网站易于理解吗？用户界面的操作方式是否可预测？用户能否快速识别并纠正自己的错误？
-   你的网站有多健壮？它是否基于开放标准且能与未来技术兼容？你将支持并使用哪些现有技术进行测试？

### 谁将受益？

是的，如果遵循了无障碍最佳实践，盲人将会受益，但你早已知道这一点。他们只是人口中极小的一部分，也仅仅是你需要考虑的残障用户群体之一。根据 2005 年美国收入与项目参与调查（SIPP）进行的一项调查，估计有 5440 万残障人士。

无障碍的经典标志是轮椅，但在使用互联网时，许多使用轮椅的人丝毫不会感到不便。上述对残障人士的估计包括了行动障碍者，但可能不包括色盲或阅读障碍者。我们只能粗略估计后两种障碍者的人数，但这部分人口占比相当可观——并且在一定程度上是隐形的，因为无法一眼看出某人是否有阅读障碍。

Drupal 7 的改进不仅会让那些我们轻易认定为残障的人受益。大多数国家正面临人口迅速老龄化的问题；这意味着视力、精细运动控制和听力存在问题的用户数量正在增加。通过针对更严格的无障碍需求进行开发，我们最终能够找到改善所有用户体验的解决方案。

无障碍问题会影响每一个人，即使只是暂时的。随着年龄增长，我们的身体和心智都会发生变化，使得技术更难使用，而无障碍设计则变得更加重要。随着人类整体寿命的延长，能够从无障碍改进中受益的人数也在增加。归根结底，无障碍问题关乎我们的人性。

### 这是法律规定

你很可能居住在一个已签署《联合国残疾人权利公约》的国家。该公约承诺允许所有人包容性地参与。此外，许多国家已经或正在制定自己的无障碍法规。在美国，《美国残疾人法案》通过承认残疾人独立生活和充分参与生活方方面面的权利，展现了领导力。澳大利亚、比利时、加拿大、丹麦、芬兰、法国、德国、香港、印度、爱尔兰、以色列、意大利、日本、韩国、荷兰、新西兰、葡萄牙、西班牙、瑞士和英国也都有关于无障碍和信息通信技术的法律。

无障碍正在成为法律要求，组织需要意识到这一点。事实上，政府机构和营利性公司曾因网站给残障人士设置障碍而被起诉。这种情况虽不常见，但塔吉特公司和国际奥委会都曾因未能实现网站无障碍而被成功起诉。

### 让你的网站无障碍的九种方法

Drupal 7 为你构建高度无障碍的网站提供了坚实的基础，但这并不足以确保你的网站完全无障碍。贡献的主题和模块的实现很容易为用户设置障碍。用户生成内容的结构以及图片字段的实现可能会造成更多障碍。

关于这个话题有很多可写的内容，但我们将仅限于选择好的模块和主题的基础知识。我们将探讨颜色和对比度，因为这是无障碍设计中经常被忽视的一个方面。然后，我们将介绍如何设置自动化测试并进行你自己的模拟测试。最后，我们将探讨如何让你的网站跟上新标准的步伐，以确保它对所有人保持无障碍。

#### 无障碍模块

Drupal 模块在复杂性和可靠性方面各不相同。Drupal 核心的钩子可以从模块和主题中调用，这可以改变 Drupal 的许多核心功能。这意味着，如果某个模块在开发时没有考虑无障碍性，它可能会降低你网站的无障碍性。你可以在问题队列中搜索模块的无障碍问题，以查看你正在安装的模块是否存在已知问题（参见 `drupal.org/project/issues/search/?issue_tags=accessibility`）。

如果你遇到问题队列中未列出的无障碍问题，请向问题队列添加错误报告，以便问题能够得到解决。只有通过这样的行动，我们才能改进 Drupal 的数千个模块和主题，并制定无障碍最佳实践。

无障碍助手模块（`drupal.org/project/accessible`）的目标是使你的网站更容易实现无障碍。此外，还有一些子模块，允许你轻松地为你网站的主题、模块和内容实现无障碍方面的改进。
