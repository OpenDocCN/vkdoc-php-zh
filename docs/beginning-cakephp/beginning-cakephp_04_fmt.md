# 第 1 章：引言

程序员使用框架已有多年历史，不过将框架用于 Web 开发是近期才兴起的。在任何项目（无论是否与 Web 相关）中使用框架的主要优势，可能源于“控制反转”的概念。许多程序以代码控制一切的方式运行。换句话说，代码决定了某个操作应在何时出现、如何处理用户响应等。

试想一下，如果这种控制顺序被反转会怎样？程序不再包含一系列操作的脚本或库，而是由一系列对象构成——这些对象在你扩展它们之前无法执行任何操作（尽管它们可能包含大量可用的工具）。这样一来，框架会调用你，而不是你去调用框架。

例如，假设你想在网站中安装一个投票程序。你浏览互联网，找到了一些声称能实现该功能的 PHP 脚本。在配置一些独特设置后，你将其中一个脚本上传到服务器并启动程序。程序运行正常，但如果你想做任何修改，就必须进入脚本，找到要更改的操作位置，然后手动进行调整。脚本掌控着控制流——程序运行时所有操作都会被执行，而若你想控制程序，就必须修改脚本。

相比之下，框架的控制流是反转的。要在框架中创建一个投票应用，你只需向框架添加处理投票的对象。框架会自动整合多个资源来实现投票流程，而你需要拦截这些资源或对其进行扩展以添加自己的功能。像脚本示例中的库一样，库会自行运行，任何修改都必须直接修改代码。框架则不同——在你扩展或添加功能之前，它无法真正为你执行任何操作。你无需直接修改框架的代码；相反，框架会采纳你的扩展，并用它们替代自身的库。

## CakePHP 入门指南

### 引言

`CakePHP`（简称 `Cake`）是一个框架，而非一组库，尽管它包含了许多像库一样简化 Web 开发的函数和方法。因此，`Cake` 等待您扩展其对象并添加您自己的自定义资源。使用 `Cake`，逐个编写每个脚本函数的日子已经一去不复返。相反，开发者现在使用一套专为 Web 开发设计的脚本、库和约定的捆绑包。

### 从新手到专业人士

本指南面向 `CakePHP` 的初学者。无论您对 `PHP` 脚本语言是否有丰富经验，在 `Cake` 中工作都需要一些您可能从未尝试过的新方法。如果您不了解什么是“has-and-belongs-to-many”关系，不知道如何构建自己的类对象，或者不知道如何解析数组，那么本书是您入门 `Cake` 的完美起点。

大多数现有的在线资源需要读者具备一定的 Web 开发背景知识，才能掌握如何在 `Cake` 中安装和工作。如果您像我刚开始使用 `Cake` 时一样，您可能只想要一些从头开始的教程和代码示例，这些示例能让您快速上手并引导您掌握更高级的技术。事实上，在论坛或聊天室提问时，许多初学者很少从专家那里得到帮助，或者得到令人困惑的指引。简单的问题可能只得到“哦，只需阅读在线手册和 API”这样的回复。有时新手需要一种非常简单的软件入门方法，而本指南正是为此而编写的。随着您开始掌握 `Cake`，本指南还将提供提示和参考，帮助您快速为项目添加更多功能并捕获错误。

本书将从展示如何在服务器和您自己的计算机上安装 `Cake` 开始，并提供详细的代码示例和可视化截图，逐步引导您完成整个过程。在第 2 章中，我将展示如何构建一个简单的 `Cake` 应用程序。您将熟悉模型-视图-控制器（`MVC`）结构，以及如何有效地组织您的 `Cake` 应用程序。在第二部分中，您将使用 `Cake` 构建更全面的 Web 应用程序，探索 `Cake` 内置的助手，包括 `Ajax` 助手，并使用更高级的功能。到本书结束时，您将能够创建自己的助手、插件和其他有用的功能，从而减少运行应用程序所需的总代码量，并且您还将拥有足够坚实的基础，可以自行尝试其他高级功能。

### 为什么选择 Cake？

自从 `Ruby on Rails` 成为一种流行的 Web 框架以来，开发者团队一直在为各种语言创建 `Rails` 的克隆或类 `Rails` 框架：`Python` 的 `TurboGears`；`PHP` 的 `Zend`、`Symfony` 等；`Perl` 的 `Catalyst`；等等。面对如此多的选择，为何要为您的 Web 项目选择 `CakePHP`？

#### 它是 PHP！

许多 Web 开发者抱怨切换到 `Ruby on Rails`，仅仅是因为该框架构建在 `Ruby` 语言之上。他们说，`PHP` 是支持更广泛的 Web 编程语言之一，并且是大多数 Web 服务提供商的标准配置，那么为什么要放弃它而选择 `Ruby` 呢？对于那些在 `PHP` 上学习 Web 开发或已将 `PHP` 作为主要开发工具的人来说，放弃 `PHP` 而选择其他语言似乎令人生畏或耗时。对于公司而言，切换到另一种语言可能需要重新分配资源、更换 Web 服务提供商或重新配置昂贵的服务器。无论如何，离开 `PHP` 转向另一个开发框架可能是昂贵且耗时的。有了 `Cake`，您可以在不学习另一种语言的情况下享受基于框架的开发的好处。

使用某些 `PHP` 框架的一个困难在于它们与 `PHP 4` 和 `PHP 5` 的兼容性。例如，`Symfony` 需要 `PHP 5`，并且不向后兼容 `PHP 4`。另一方面，`Cake` 兼容 `PHP` 的这两个版本，这对于许多拥有可追溯到几年前长期项目的开发者来说是一个必要的功能。

许多 `PHP` 开发者忽视了框架的好处，只是寻找预制的函数或类作为脚本的包含文件使用，或者像使用 `Perl` 那样，拉入一些在服务器上消耗大量时间且提供很少自定义功能的模块。然而，`Cake` 在其范围内是完全面向对象的。它提供了可以根据您的喜好实现和修改的对象，而不仅仅是某些给您很少控制的模块或包含文件集合。

#### 快速开发

启动一个 Web 项目可能既繁琐又技术要求高，尤其是在使用较旧的开发方法时。然而，`Cake` 使得构建 Web 应用程序的初始步骤变得容易。`Cake` 不是从命令行运行安装脚本，而是预先打包成一个文件夹，您只需将其放到服务器上即可运行。

一旦您开始在框架上进行构建，命令行就会派上用场。稍后，我将讨论`Cake`的脚手架特性，这些特性可以减少日常开发任务。使用`Cake`，在应用程序中早期创建用户流变得简单，并且可以改善与客户的沟通。在某些情况下，应用程序的初步运行可以在几分钟内完成，让客户了解项目的架构。

一旦项目充实并启动，得益于`Cake`，站点维护也得到了改进。由于其层次结构和组织性，以及其有效限制冗余的能力，`Cake`帮助开发者动态调整Web应用程序。`Cake`还支持测试数据库和`URL`路由，用于在线上环境中测试新功能或Web应用程序版本。

#### 模型-视图-控制器

`Cake`为您的Web应用程序强制执行`MVC`结构。基本上，它有效地将典型操作分离到特定区域：`Model`负责所有数据库交互，`View`负责所有输出和显示，`Controller`负责所有输入和程序流程的命令/脚本。典型的`PHP`应用程序将这三个功能混合在同一个代码中，使得维护和调试变得困难。

以下是`PHP`脚本的典型流程（见图1-1）：

1.  客户端通过输入`URL`或点击某种链接向`PHP`脚本发送请求。

2.  脚本处理数据，然后直接将数据库请求发送到数据库。

3.  脚本接收任何数据库输出并处理数据。

4.  脚本生成输出并将其转发到客户端的浏览器。

### MVC提升代码组织性

MVC改进了典型的PHP流程，是一种让类对象在整个应用中都可用的高效技术。MVC背后的主要目标是确保应用程序的每个功能只编写一次，从而通过减少冗余来精简代码。`Cake`通过提供实现MVC所需的资源，并采用一致的方法来定义操作在应用中的存储位置，从而实现了这一目标。只需以特定方式命名文件，`Cake`就能在不使用任何代码规范的情况下，将各种资源拼凑在一起。

MVC可能因你使用的框架而异，但通常其工作流程如下（参见图1-2）：

**1.** 客户端通过输入URL或点击某种链接，向应用程序发送页面请求。按照惯例，典型的URL通常结构如下：`http://{Domain}.com/{Application}/{Controller}/{Action}/{Parameter 1, etc.}`

**2.** 调度脚本解析URL结构，并确定要执行哪个控制器。同时，它还会将任何动作和参数传递给该控制器。

**3.** 控制器中的函数可能需要处理比调度器转发的参数更多的数据。它会向模型脚本发送数据库请求。

**4.** 模型脚本根据控制器提交的请求，决定如何与数据库进行交互。它可能会对数据库执行查询，并执行各种便捷的数据排序指令。

**5.** 一旦模型从数据库获取了数据或将数据发送到数据库，它会将输出返回给控制器。

**6.** 控制器处理数据，并输出到视图文件。

**7.** 视图将任何设计或展示数据添加到控制器的输出中，并将其输出发送到客户端的浏览器。

使用MVC开发网站的好处在于，可以将重复的功能或任务分离，从而实现更快速的编辑。它甚至有助于调试。例如，在与数据库交互时反复出现错误，问题通常出在模型的某个地方。知道所有数据库交互只发生在一个地方，使得解决问题更加容易。

### CRUD操作与Bake脚本

几乎所有网站都使用CRUD操作：创建、读取、更新和删除。例如，一个博客需要创建文章；用户需要能够阅读每篇文章；作者可能希望将来能够编辑或更新文章；作者也需要能够删除文章。

`Cake`通过其自动化的CRUD函数使这些操作变得轻而易举。它并非让你手动编写每个CRUD操作，而是提供了预构建的类来为你完成这些工作。`Cake`包含`Bake`脚本，这是一个便捷的命令行工具，能根据你的数据库模式和自定义参数生成可编辑的CRUD代码。

### 脚手架

借助Cake的脚手架功能，让网站快速起步变得容易得多。只需一行简单的代码，你就可以调用Cake的预构建脚手架，基于数据库渲染视图。换句话说，它能自动判断一些标准界面视图应如何与你的数据库协同工作，并输出HTML表单，而你无需编写任何HTML代码。

尽管脚手架功能并非用于完整的产品级视图，但它能让你开始测试逻辑和功能，而无需浪费时间构建视图或HTML输出。

### 助手

`Cake`提供了标准的HTML、Ajax和JavaScript助手，它们极大地简化了视图的创建。直观的助手代码字符串能为你渲染标记，极大地便利了你的HTML输出。而让Ajax工作起来，虽然初学时有点棘手，但与你需要担心DOM特性相比，使用助手要容易得多，也高效得多。此外，你还可以下载其他Cake开发者编写的助手来增强框架的能力，甚至可以自己编写助手，以减少重复或笨拙的标记。

### 可定制的元素

你可以定制Cake的每个功能来适应你的应用程序。例如，你可以将流行的浏览器所见即所得编辑器`FCKeditor`作为插件引入到`Cake`中。通过使用定制的助手，你可以将`FCKeditor`的所有功能集成到你的`Cake`应用程序中，并且实际上可以砍掉额外行的PHP代码来使其工作。稍后，我将讨论其他Cake元素，如组件、助手和插件，所有这些你都可以根据特定需求进行定制，或作为其他开发者提供的第三方资源引入到你的应用程序中。

### 庞大的社区

如果你在开发过程中需要帮助，有一个庞大的在线社区可以提供支持。实际上，PHP社区是网络上最大的开源编程群体，因此如果你需要一个`Cake`问题的快速解决方案，总有人能在几分钟内提供帮助。Cake专家还建立了在线论坛、聊天室和博客，以帮助其他人改进和学习这个框架。与其他PHP框架相比，这个社区是网络上最大的社区之一。

对于任何参与Web开发的人来说，代码示例都是必不可少的。PHP主导着这个领域，而`Cake`拥有一个不断增长的代码示例库。如果你正在考虑另一个框架，并且希望站在别人的成果之上，那么这个事实可能会让你倾向于选择`Cake`。

### 更多特性

`Cake`旨在通过提供一种组织数据库和其他资源文件的整体方法来简化构建Web应用程序的开发过程，从而减少代码。尽管这种Web编程的通用方法本身是`Cake`提供的一个主要特性，但它所拥有的其他强大资源库，如内置验证、访问控制列表(ACL)、数据净化、安全和会话处理组件以及视图缓存，使得`Cake`值得任何认真的开发人员投入时间研究。

### 总结

作为一个框架，`Cake`反转了控制流程，并为你（开发者）提供了一种在更短时间内扩展Web应用程序的有效方法。`Cake`是用PHP构建的，因此你可以利用一个基于Web的框架，而无需学习或使用另一种编程语言。使用`Cake`的其他好处包括：其MVC结构，它分离了应用程序的资源和功能；`Bake`脚本，它自动化了CRUD脚本编写；脚手架功能，它将基本的视图渲染简化为一行代码；内置的助手，它们将HTML输出操作缩减为单行调用；以及庞大且仍在增长的Cake社区。
