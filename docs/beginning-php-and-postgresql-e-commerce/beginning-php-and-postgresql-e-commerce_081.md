# 管理客户详细信息

到目前为止，你已经在这本书中构建了一个基础但功能完整的网站，并将其与 `PayPal` 集成，用于接收付款和确认订单。在本书的这一部分，你将更进一步。通过从订单处理流程中移除 `PayPal`，你可以获得更好的控制权，同时降低间接成本。这并不像你想象的那么复杂，但我们确实需要谨慎行事以确保一切正确。

在本章中，我们将通过实现一个客户账户系统为此奠定基础。

为了使电子商务网站更加用户友好，你通常会将信用卡号等详细信息存储在数据库中，这样用户就不必在每次下单时重新输入这些信息。你将实现的客户账户系统将存储这些信息，并包含输入此类详细信息所需的所有网页。

除了实现这些网页，我们还需要考虑其他几个因素。首先，简单地将信用卡号、有效期和其他重要信息以明文形式存储在数据库中并不理想，因为一旦服务器被攻破，这些数据就有可能被盗。这种情况可能远程发生，也可能由组织内部的个人实施。除了对此类数据实施严格限制的访问策略外，更简单的方法是对敏感信息进行加密，并在需要时通过编程方式检索。我们将创建一个安全库来简化此功能。

其次，安全通信非常重要，因为你将通过网络捕获信用卡详细信息等敏感信息。我们不能仅仅创建一个表单让人们通过 `HTTP` 访问并允许他们将其发送给我们，因为信息可能会被截获。相反，我们将使用基于 `HTTPS` 连接的 `SSL`。你将把 `HatShop` 应用程序推进到可以继续前进并在下一章实现自己的后端订单处理管道的阶段。

在本章中，你将学习如何：

- 存储客户账户
- 实现安全类
- 向 `HatShop` 添加客户账户功能
- 创建结账页面

## 存储客户账户

你可以通过多种方式处理网站中的客户账户功能。不过，一般来说，这些方法都具备以下共同特征：

- 客户登录以访问网站的安全区域。
- 一旦登录，Web 应用程序会记住该客户，直到客户注销（通过“退出”链接手动注销，或者在会话超时或发生服务器错误时自动注销）。
- Web 应用程序中的所有安全页面在允许访问之前都需要检查客户是否已登录。

首先，让我们看看 `HatShop` 电子商务网站的通用实现细节。

### HatShop 客户账户方案

判断客户是否登录的一种简单方法是将客户 ID 存储在会话状态中。然后你可以在安全页面的开头验证是否存在该值，如果不存在则警告用户。登录表单本身可以对用户进行身份验证，如果成功，则在会话状态中存储一个值，以备后续检索。要注销用户，只需从会话状态中移除该 ID 即可。

客户登录需要提供用户名（这里我们将使用客户的电子邮件地址，因为它保证是唯一的）和密码。通过互联网发送这些信息是一个敏感问题，因为第三方可能会窃听并捕获它们。在本章后面，我们将探讨如何启用通过互联网的安全通信。不过现在，我们将专注于身份验证方面，这一点不受用于传输客户电子邮件地址和密码的连接类型的影响。
