# 连接 Web 服务

在瞬息万变的互联网世界中，仅仅拥有一个重要的网络存在有时是不够的；你还需要与第三方提供的功能进行交互以实现你的目标。在本书的前面部分，你已经看到了如何集成外部功能来处理客户的支付。

在本章中，你将学习通过 Web 服务集成外部功能的新可能性。**Web 服务**是一种通过网络接口使用标准互联网协议（如`HTTP`）暴露出来的功能单元。客户端和服务器之间交换的消息采用基于 XML 的协议 `SOAP`（简单对象访问协议）或 `REST`（表述性状态转移）进行编码。

这些消息通过 `HTTP` 发送。稍后你将更详细地了解这些技术。

使用 Web 服务的美妙之处在于，客户端和服务器可以使用任何技术、任何语言和任何平台。只要它们通过标准协议（例如基于 `HTTP` 的 `SOAP`）交换信息，即使客户端是手机，而服务器是在 Solaris 上运行的 Java 应用程序，也毫无问题。

这些可能性令人振奋，我们建议你购买一本专门介绍 Web 服务的书籍，以深入了解其世界。可以查看 `http://www.xmethods.net/` 上的公共 Web 服务列表，了解你可以集成到应用程序中的外部功能类型。

在本章中，你将学习如何集成 **Amazon 电子商务服务**（`ECS`；Amazon.com 提供的 Web 服务接口，以前称为 Amazon Web Services——`AWS`），通过你的 HatShop 网站销售 Amazon.com 的产品。

你已经拥有一个向客户销售帽子的电子商务网站。你可以更进一步，通过将 Amazon.com 上其他一些与帽子相关的礼品整合到你的网站中，从他们对帽子的热情中赚取更多收入。免费吗？哦不……你将在网站上展示 Amazon.com 的详细信息，但最终结账将由 Amazon.com 处理，并且 Amazon.com 会将通过你网站产生的购买佣金存入你的银行账户。听起来像是轻松赚钱，不是吗？

在本章中，你将学习如何使用 `ECS` 向你的网店添加一个名为“Amazon Super Hats”的特殊部门，如图 17-1 所示。这将是一个“特殊”部门，其处理方式与其他部门不同——例如，当访问者想要购买产品时，付款直接由 Amazon 处理。本章仅探讨了 `ECS` 功能的一小部分，因此，如果你真的想通过此服务发财，应该更深入地挖掘以找到更多内容。
