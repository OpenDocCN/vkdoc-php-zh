# 创建 `department` 表

你在第 2 章中创建了 `hatshop` 数据库。在接下来的练习中，你将使用 `pgAdmin III` 向其中添加 `department` 表。或者，你可以使用源代码/下载中的 SQL 脚本创建并填充 `department` 表（你也可以通过 `pgAdmin III` 执行它们）。

> **注意** 与其他数据库服务器不同，`PostgreSQL` 仍然允许你在添加新行时，如果需要，手动为自动编号字段指定值。

> **注意** 你可以在本书的源代码/下载部分找到数据库创建脚本，该部分位于 Apress 网站 (`http://www.apress.com`)。你也可以在作者的个人网站 `http://www.cristiandarie.ro` 和 `http://www.emilianbalanescu.ro` 上找到这些文件。

## 练习：创建 `department` 表

1. 启动 `pgAdmin III`，并使用 `hatshopadmin` 用户名登录你的数据库服务器，就像你在第 2 章最后一个练习中所做的那样。

2. 展开 `hatshop` 数据库节点，展开 `Schemas`，展开 `public`，然后选择 `Tables` 节点。右键单击此节点，并从上下文菜单中选择**新建表**。

3. 在**名称**文本框中输入 `department`，然后单击**列**选项卡。

4. 在**列**窗口中，单击**添加**。按照图 3-10 所示填写详细信息，然后单击**确定**。

   **图 3-10.** *添加 `department_id` 字段*

5. 再次单击**添加**以添加 `name` 字段，如图 3-11 所示。

   **图 3-11.** *添加 `name` 字段*

6. 再次单击**添加**以添加 `description` 字段。这应该也是一个 `varchar` 字段，最大长度为 1000。让**非 NULL** 复选框保持未选中状态，然后单击**确定**。

7. 创建表的最后一步是指定一个主键。单击**约束**选项卡，确保组合框中选中了**主键**，然后单击**添加**。

8. 为键名输入 `pk_department`，然后切换到**列**选项卡，并将 `department_id` 添加到列表中。单击**确定**关闭对话框。

9. 再次单击**确定**以创建表。你的新表现在应该出现在数据库的**表**列表中（参见图 3-12）。如你所见，除了你的新表之外，还有一些 `PostgreSQL` 默认创建的表——忽略它们是安全的。如果你从列表中选择 `department` 表，`pgAdmin III` 将显示创建你刚刚通过可视化界面构建的结构的 SQL 代码。你可以检查在你的计算机上是否看到相同的代码，以确保你已正确完成了练习的步骤。

   **图 3-12.** *`pgAdmin III` 中的 `department` 表*

   > **注意** 你将在后续章节中学习更多关于 SQL 的知识，但现在只需知道 SQL 是一种可用于与数据库交互、创建数据表、从这些表中读取数据等的语言就足够了。

10. 让我们用一些示例数据填充该表。右键单击 `department` 表，然后选择**查看数据**。

11. 使用显示的表单，添加表 3-2 中提到的记录。

    **表 3-2.** *`department` 表的三个记录*

    | `department_id` | `name`          | `description`                                                                                   |
    |-----------------|-----------------|-------------------------------------------------------------------------------------------------|
    | Holiday         | 用我们特别的季节性帽子系列为假期做准备！                                                              |
    | Caps and Berets | 适合工作和化装派对的完美帽子！                                                                   |
    | Costume Hats    | 为你的新戏服找到相配的帽子！                                                                     |

    > **注意** 由于你已经设置了序列，最好让它生成部门 ID，而不是自己输入值。你只需输入名称和描述。如果你手动写入 `department_id` 值而不是让序列生成它，你还需要更新序列，因为这不会自动发生。如果你不更新序列，它将生成数据库中已存在的值。

## 工作原理：创建 PostgreSQL 数据表

你已经创建了你的第一个数据库表！你还设置了主键、设置了自增列，并用一些数据填充了该表。

正如你所看到的，一旦你对表的结构有了清晰的概念，使用 `pgAdmin III` 在你的数据库中创建它就相对容易了。让我们继续！
