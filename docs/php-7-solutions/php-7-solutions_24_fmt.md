# 嵌套包含文件

当一个文件被包含到另一个文件中时，相对路径是基于父文件计算的，而不是基于被包含的文件。这给需要包含另一个外部文件的外部函数或类定义带来了问题。

如果两个外部文件位于同一文件夹中，你可以仅使用文件名来包含嵌套文件，如下所示：

```php
require_once 'Thumbnail.php';
```

在这种情况下，相对路径*不应*以`./`开头，因为`./`表示“从当前文件夹开始”。对于包含文件而言，“当前文件夹”指的是父文件的文件夹，而非包含文件本身的文件夹，这会导致嵌套文件的路径错误。

当包含文件位于不同文件夹时，你可以使用 PHP 常量`__DIR__`为目标文件构建绝对路径。该常量返回包含文件所在目录（文件夹）的绝对路径，且不带尾部斜杠。将`__DIR__`、一个正斜杠和文档相对路径拼接起来，即可将相对路径转换为绝对路径。例如，假设这是从一个包含文件到另一个包含文件的相对路径：

```php
'../File/Upload.php'
```

你可以像这样将其转换为绝对路径：

```php
__DIR__ . '/../File/Upload.php'
```

在文档相对路径开头添加正斜杠的效果是：先找到包含文件的父文件夹，然后向上跳一级以找到正确的路径。在第 10 章中你将看到一个这样的实际示例，其中某个包含文件需要包含另一个位于不同文件夹的文件。
