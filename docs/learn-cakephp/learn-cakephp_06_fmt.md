# 命名

我们应该为类、方法、变量、参数等进行命名。名称应该具有描述性和意义。“Foo” 和 “bar” 在简短的代码示例中还行，但在生产代码中可能会非常令人困惑。

当你完全沉浸在代码中时，一个名为 `$lld` 的变量可能很清楚，但 `$last_login_date` 难道不是更具可读性、更容易理解吗？

按照惯例，CakePHP 对数据库表、字段名和视图文件使用下划线命名。对于类名，它使用驼峰命名法（CamelCase）。对于变量名和参数名，你可以选择其中任何一种，但请坚持使用一种。因此，如果你更喜欢带下划线的变量名，那么所有变量名都使用下划线，不要混用驼峰命名。

最好遵循 PHP 标准建议 (PSR) (`www.php-fig.org/psr`)。`PSR-1` 建议使用驼峰命名的方法名。使用动词作为方法名是一个最佳实践。
