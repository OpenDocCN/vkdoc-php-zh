# 处理错误与异常

PHP 7 改变了报告大多数错误的方式，现在通过**抛出异常**（即生成一个包含错误详情和发生位置的特殊类型对象）。如果你使用过 PHP 5，可能唯一注意到的区别是错误消息的措辞或错误类型有所不同。然而，由内部错误（如解析错误或缺少包含文件）抛出的异常与由脚本抛出的异常之间存在细微差别。


  
当 PHP 由于内部错误而抛出异常时，它会立即停止脚本运行。如果你已开启错误信息显示（建议在测试环境中这样做），PHP 会显示一条消息说明发生了什么。有时这些消息难以解读，因此**捕获异常**通常是个好主意。你可以通过将主脚本包装在 `try` 块中，并将错误处理代码放在 `catch` 块中来实现，如下所示：  

```
try {
    // main script goes here
} catch (Throwable $t) {
    echo $t->getMessage();
}
```

提示  
`catch` 块中的 `Throwable` 类型声明是 PHP 7 新增的。它涵盖了内部错误和脚本抛出的异常（用户异常）。你可能从 PHP 5 中了解的 `Exception` 类型声明仅涵盖用户异常。  

这产生了一条错误消息，通常比某些错误生成的长篇消息更容易理解。  

你可以使用 `throw` 关键字抛出自定义异常，如下所示：  

```
if (error occurs) {
    throw new Exception('Houston, we have a problem.');
}
```

括号内的字符串用作错误消息，可以在 `catch` 块中被捕获。  

警告  
错误消息在开发过程中至关重要。它们帮助你解决问题。但是，当你将脚本部署到实时网站时，它们不仅看起来丑陋；还可能泄露对恶意攻击者有用的信息。在上线时，将 `catch` 块中的错误消息显示替换为中性消息。或者，使用 `catch` 块将访问者重定向到错误页面。  

### 动态创建新变量  

PHP 支持创建所谓的**变量变量**。虽然这看起来像是打字错误，但并非如此。简单来说，变量变量创建一个新变量，该变量从现有变量派生其名称。以下示例展示了其工作原理（参见 `variable_variables.php`）。  

```
$location = 'city';
```

上述语句将字符串“city”赋给一个名为 `$location` 的变量。你可以通过使用*两个*美元符号来创建一个变量变量，如下所示：  

```
$$location = 'London';
```

变量变量将原始变量的值作为其名称。换句话说，`$$location` 等同于 `$city`。  

```
echo $city; // London
```

你将在第 6 章的邮件处理脚本中看到此技术的实际应用示例。  

提示  
为了表明双 `$` 是有意为之，请将用于创建变量变量的变量用花括号括起来，如下所示：`${$location}`。花括号是可选的，但能使代码更易读。

### 从外部文件包含代码  

从其他文件包含代码的能力是 PHP 的核心部分。只需要使用 PHP 的包含命令之一，并告诉服务器在哪里找到该文件。  

#### 介绍 PHP 包含命令  

PHP 有四个命令可用于从外部文件包含代码，分别是：  

- `include`  
- `include_once`  
- `require`  
- `require_once`  

它们都执行基本相同的事情，那为什么要有四个呢？根本区别在于，即使找不到外部文件，`include` 也会尝试继续处理你的脚本，而 `require` 则具有强制性的含义：如果文件缺失，PHP 引擎会停止处理并抛出致命错误。实际上，这意味着如果你即使没有外部文件的内容，页面仍能保持可用，则应使用 `include`。如果页面依赖于外部文件，则应使用 `require`。  

另外两个命令 `include_once` 和 `require_once` 阻止同一文件在一个页面中被多次包含。尝试在脚本中多次定义一个函数或类会触发致命错误。因此，`include_once` 或 `require_once` 确保函数和类只定义一次，即使脚本尝试多次包含外部文件（例如当这些命令位于条件语句中时）也是如此。  

提示  
如果有疑问，始终使用 `require`，但对于定义函数和类的文件，应使用 `require_once`。依赖即使找不到外部文件脚本仍能正常工作，可能会使你面临安全风险。  

### PHP 查找包含文件的位置  

要包含一个外部文件，请使用四个包含命令之一，后跟引号（单引号或双引号均可）中的文件路径。文件路径可以是绝对路径，也可以是相对于当前文档的路径。例如，以下任何一项都可以工作（只要目标文件存在）：  

```
require 'includes/menu.php';
require 'C:/xampp/htdocs/phpsols-4e/includes/menu.php';
require '/Applications/MAMP/htdocs/phpsols-4e/includes/menu.php';
```

注意  
PHP 接受包含命令中 Windows 文件路径的正斜杠。  

你也可以选择性地在包含命令中使用括号，因此以下方式也可以工作：  

```
require('includes/menu.php');
require('C:/xampp/htdocs/phpsols-4e/includes/menu.php');
require('/Applications/MAMP/htdocs/phpsols-4e/includes/menu.php');
```

使用相对文件路径时，建议使用 `./` 来表示路径从当前文件夹开始。因此，将第一个示例重写如下会更高效：  

```
require './includes/menu.php'; // path begins in current folder
```

*不*起作用的是使用相对于站点根目录的文件路径，如下所示：  

```
require '/includes/menu.php'; // THIS WILL NOT WORK
```

这不起作用，因为 PHP 包含命令将前导斜杠解释为硬盘的根目录。换句话说，PHP 将其视为绝对路径，而不是相对于站点根目录的路径。PHP 还会在 PHP 配置中定义的 `include_path` 中查找。我将在本章后面回到这个主题。在此之前，让我们将 PHP 包含投入实际使用。
