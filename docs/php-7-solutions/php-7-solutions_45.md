## 创建 PHP 文件上传类

正如你刚才所见，上传文件仅需几行代码，但这本身还不足以宣告任务完成。你需要通过实施以下步骤来使过程更加安全：

-   检查错误级别。
-   在服务器上验证文件未超过允许的最大大小。
-   检查文件类型是否可接受。
-   从文件名中移除空格。
-   重命名与现有文件同名的文件，以防止覆盖。
-   自动处理多个文件上传。
-   告知用户结果。

每次你想要上传文件时，都需要实施这些步骤，因此构建一个易于重用的脚本是合理的。这就是我选择使用自定义类的原因。构建 PHP 类通常被视为一个高级主题，但别因此而却步。**类** 是一个旨在协同工作的函数和变量的集合。这虽然过于简化，但足以让你理解基本概念。类中的每个函数通常应专注于单一任务，因此你将构建独立的函数来实现前述列表中的步骤。代码还应该是通用的，不局限于特定的网页。一旦构建好类，你可以在任何表单中重用它。尽管类定义很长，但使用该类只需编写几行代码。类的另一个优势在于，你可以通过创建**子类**（也称为**子类**）来添加或修改其功能。你将在第 10 章中看到一个例子，该章基于本章开发的类进行构建。

要使用一个类，你需要调用其构造方法来创建该类的一个实例（即一个**对象**）。该对象可以访问该类的所有特性，即其函数（称为**方法**）和变量（称为**属性**）。通常会创建类的多个实例来处理不同的情况。例如，你可能希望创建单独的对象来将图片和文本文件上传到不同的位置。当你通读本章的其余部分时，这些概念会变得更加清晰。

如果你时间紧迫，完成的类位于 `ch09/PhpSolutions` 文件夹中。即使你自己不构建脚本，也请通读相关描述，以便清楚了解其工作原理。

## 定义 PHP 类

定义 PHP 类非常简单。你使用 `class` 关键字，后跟类名，然后将该类的所有代码放在一对花括号之间。按照惯例，类名以大写字母开头，并存储在与类同名的单独文件中。

### 使用命名空间避免命名冲突

如果你编写自己的脚本，很少需要担心命名冲突。我们将创建一个用于上传文件的类，因此 `Upload` 或 `FileUpload` 看起来是合乎逻辑的名称。一旦你开始使用他人编写的脚本和类（包括本书中的），就存在多个类同名带来的风险。PHP 通过使用命名空间将相关的类、函数和常量分组来解决这个问题。

一种常见的策略是将类定义存储在一个描述其功能的文件夹结构中，并为顶层文件夹赋予一个基于域名或公司名称的唯一名称。命名空间可以有子级别，因此文件夹结构会复制为用反斜杠分隔的子命名空间。命名空间也会单独声明，允许你使用简单的类名。我们将要构建的类名为 `Upload`，但为了避免命名冲突，它将被创建在名为 `PhpSolutions\File` 的命名空间中。

你可以在文件顶部使用 `namespace` 关键字，后跟命名空间，像这样声明命名空间：

```
namespace PhpSolutions\File;
```

#### 注意

在所有操作系统上，PHP 都使用反斜杠作为命名空间分隔符。不要试图在 Linux 或 macOS 上将其更改为正斜杠。

因此，如果我们在此命名空间中创建一个名为 `Upload` 的类，其完全限定名就是 `PhpSolutions\File\Upload`。

```
use PhpSolutions\File\Upload;
```

```
## PHP 解决方案 9-2：创建基本的文件上传类

在本 PHP 解决方案中，你将创建一个名为`Upload`的类的定义来处理文件上传。你还将创建该类的一个实例（一个`Upload`对象）并使用它来上传图像。请花足够的时间完成以下步骤。它们并不难，但会引入一些概念，如果你之前没有使用过 PHP 类，这些概念可能不熟悉。

1.  在`phpsols-4e`站点根文件夹中创建一个名为`PhpSolutions`的子文件夹。文件夹名称中使用相同的大小写字母组合。

2.  在`PhpSolutions`文件夹中创建一个名为`File`（大写 F）的子文件夹。

3.  在新的`PhpSolutions/File`文件夹中，创建一个名为`Upload.php`的文件。同样，文件名中使用相同的大小写字母组合。然后插入以下代码：

    ```
    <?php
    namespace PhpSolutions\File;
    class Upload {
    }
    ```

    所有剩余代码都放在花括号之间。该文件将只包含 PHP 代码，因此你不需要关闭 PHP 标签。

4.  PHP 类通过将一些变量和函数声明为`protected`来隐藏其内部工作原理。如果你在变量或函数前面加上关键字`protected`，它只能在该类或子类内部访问。这可以防止值被意外更改。

`Upload`类需要以下项目的受保护变量：

*   上传文件夹的路径
*   最大文件大小
*   报告上传状态的消息
*   允许的 MIME 类型

通过将它们添加到花括号内来创建变量，如下所示：

```
    class Upload {
    protected $destination;
    protected $max = 51200;
    protected $messages = [];
    protected $permitted = [
    'image/gif',
    'image/jpeg',
    'image/pjpeg',
    'image/png',
    'image/webp'
    ];
    }
    ```

这些属性可以在类中使用`$this->`访问，它指的是当前对象。例如，在类定义内部，你访问`$destination`为`$this->destination`。

**注意**
当你首次在类内部声明属性时，它像任何其他变量一样以美元符号开头。但是，你需要省略`->`运算符之后的属性名称中的美元符号。

除`$destination`外，每个受保护属性都赋予了默认值：

*   `$max`将最大文件大小设置为 50 KB（51200 字节）。
*   `$messages`是一个空数组。
*   `$permitted`包含一个图像 MIME 类型数组。

`$destination`的值将在创建类的实例时设置。其他值将由类内部控制，但你也会创建函数（或在类中称为**方法**）来更改`$max`和`$permitted`的值。

1.  当你创建对象时，类定义文件会自动调用类的构造函数方法，该方法初始化对象。所有类的构造函数方法都称为`__construct()`（双下划线）。与你在上一步中定义的属性不同，构造函数需要在类外部可访问，因此你在其定义前加上`public`关键字。

**注意**
`public`和`protected`关键字控制属性和方法的**可见性**。公共属性和方法可以在任何地方访问。尝试直接在类定义或子类外部访问受保护的属性或方法会触发致命错误。

`Upload`类的构造函数接受一个参数：你想要上传文件的文件夹路径，并将其赋值给`$destination`。在受保护属性列表之后，添加以下代码，确保它位于类定义的闭合花括号之前：

```
public function __construct($path) {
if (is_dir($path) && is_writable($path)) {
$this->destination = rtrim($path, '/\\') . DIRECTORY_SEPARATOR;
} else {
throw new \Exception("$path must be a valid,writable directory.");
}
}
```

构造函数内部的`if`语句将`$path`传递给`is_dir()`和`is_writable()`函数，这些函数检查提交的值是一个有效的、可写的目录（文件夹）。如果是，`rtrim()`函数会删除`$path`末尾的任何空格和斜杠，然后连接适用于操作系统的目录分隔符。这确保了路径以斜杠结尾，无论用户在创建`Upload`对象时是否添加了斜杠。当只向`rtrim()`传递一个参数时，它只删除空格。第二个可选参数是一个字符串，包含所有其他要删除的字符。需要两个反斜杠来转义闭合引号。如果传递给构造函数的值不是有效的、可写的目录，构造函数会抛出异常。

**注意**
类可以定义自己的异常，并且由于`Upload`类是在命名空间内定义的，因此构造函数应该使用自定义异常还是 PHP 核心部分的`Exception`类是不明确的。要访问命名空间内的核心命令，你需要在它们前面加上反斜杠。这就是`Exception`前面有反斜杠的原因。我们使用的是核心的`Exception`类，而不是自定义的。
    ```
