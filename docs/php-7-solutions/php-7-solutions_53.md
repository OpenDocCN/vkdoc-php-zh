# 良好数据库设计的检查点

数据库设计没有所谓的*正确*方法——每个数据库都各不相同。不过，以下指南应该能为你指明正确方向：

- 为表中的每条记录赋予一个唯一标识符（主键）。
- 将每组相关数据放入其独立的表中。
- 通过使用一个表中的主键作为其他表中的外键来交叉引用相关信息。
- 在每个字段中仅存储一条信息。
- 遵循 DRY 原则（不要重复自己）。

在早期阶段，你很可能会犯下一些日后会后悔的设计错误。试着预见未来的需求，并让你的表结构保持灵活。你可以随时添加新表以应对新的需求。
