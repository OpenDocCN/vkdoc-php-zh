谁在意代码呢？只要它能运行，那就没问题了。对吧？我不这么认为。

有几个理由让你应该关注代码质量。

首先，你代码的质量将决定软件的质量。如果你认为可以用混乱的代码构建出优秀的软件，那你是在做梦。其次，整洁代码节省时间。你花在阅读代码上的时间将多于编写它的时间。你团队中的其他人也会花相当一部分时间来阅读你的代码。你同样会花时间阅读别人的代码。整洁代码易于阅读，因此整洁代码能节省时间。第三，你的代码是你自身的反映。如果你不重视代码的质量，你就不会被录用。混乱的代码意味着你对自己或自己的作品没有坚持高标准。客户和雇主都要求质量。

整洁代码是可维护的。它易于阅读和理解。这意味着它易于使用和修改。整洁代码是直截了当的。

## 如何编写整洁代码

编写整洁代码不需要花费大量时间。编写整洁代码是一种习惯，在编写和重构过程中需要多加注意。幸运的是，这是一种可以学会的习惯。
