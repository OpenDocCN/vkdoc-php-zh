# 第 3 章

## 创建产品目录：第一部分

在学习完三层架构并实现了网站主页的一部分之后，是时候继续你的工作了——开始创建 HatShop 产品目录。

由于产品目录包含许多组件，我们将分两章来创建它。在本章中，你将创建第一个数据表，在中间层实现访问方法，并学习如何处理数据层。到本章结束时，你最终将在网页上获得动态生成的内容。在第 4 章中，你将通过添加对类别、产品列表、产品详情页面等的支持，完成产品目录的构建！

本章将涉及的主要主题有：

- 分析产品目录的结构及其应支持的功能
- 为目录及目录的数据层创建数据库结构
- 实现运行目录所需的业务层对象
- 为产品目录实现一个可用的用户界面

### 向访问者展示你的商品

任何在线商店的基本功能之一，就是让访问者能够轻松浏览产品。想象一下，如果亚马逊没有其出色的产品目录会是什么样子！

无论你的访问者是在寻找特定商品，还是仅仅在浏览，确保他们在你网站上的体验是愉快的，这一点至关重要。毕竟，你希望访问者能够尽可能轻松、无痛地找到他们想要的东西。这就是为什么你需要在网站上添加搜索功能，并找到一种巧妙的方法将产品结构化为类别，以便他们能够快速直观地访问。

根据商店的规模，将产品归入若干类别可能就足够了，但如果产品很多，你就需要找到更多的方法来分类和构建产品目录。

确定目录的结构是本章首先要完成的任务之一。请记住，在专业的做法中，这些细节在开始编码之前，即在构建项目需求文档时就应该确定下来，如附录 B 所述。然而，就本书而言，我们更倾向于一次处理一件事。

在目录结构确定之后，你将开始编写使目录按计划运行的代码。

### 产品目录是什么样子的？

如今的网络用户比以前要求更高。他们期望能快速找到他们心目中任何产品或服务的信息，如果找不到，他们很可能会在给网站第二次机会之前就转向竞争对手。当然，你不希望这种事情发生在你的访问者身上，因此你需要构建目录，使其尽可能直观和有用。

由于这家在线商店最初将有大约 100 种产品，而且未来可能会更多，仅将它们分组到类别中是不够的。商店还设有多个部门，每个部门将包含若干类别。每个类别下可以关联任意数量的产品。

> **注意** 在本书后面，你还会创建网站的行政部分，通常称为控制面板，它允许客户更新部门、类别和产品数据。在此之前，你将手动填充数据库数据（或者你可以通过使用 Apress 网站 `http://www.apress.com` 源码/下载部分提供的 SQL 脚本来“投机取巧”，稍后你会看到）。

另一个需要特别考虑的重要细节是：一个类别是否可以存在于多个部门中，以及一个产品是否可以存在于多个类别中。你可能猜到了，这类决策会对编写产品目录的方式产生影响，因此你需要就此问题咨询你的客户。

对于 HatShop 产品目录，每个类别只能存在于一个部门中，但一个产品可以存在于多个类别中。例如，产品“军用贝雷帽”将同时出现在“贝雷帽”和“军用帽”这两个类别中。这一决定将对数据库设计产生影响，我们在讲到相关内容时会重点强调这些影响。

最后，除了将产品按类别分组之外，你还需要设置特色产品。对于这个网站，一个产品可以成为首页或部门页面的特色产品。下一节将展示一些截图来说明这一点。
