# 目录管理

在前面的章节中，您操作的是数据库中已存在的目录信息。您可能已经手动插入了一些记录，或者从本书附带的源代码中下载了部门、分类和产品信息。

显然，这两种方法对于真实的网站来说都是不可接受的，因此您需要编写一些代码来轻松管理网店数据。也就是说，在发布网站之前需要处理的最后一个细节是创建其管理界面。虽然访问者永远不会看到这部分内容，但它对于向客户交付高质量的网站至关重要。

在本章中，您将实现一个目录管理页面。通过此功能，您将完成网站开发的第一阶段！由于此页面可以有多种实现方式，因此需要与客户进行认真讨论，以确定所需的具体功能列表。在我们的案例中，目录管理页面应允许客户执行以下操作：

- 添加和删除部门
- 修改现有部门信息（名称、描述）
- 查看属于某个部门的分类列表
- 管理部门分类
- 编辑现有分类信息（名称、描述）
- 查看特定分类中的产品列表
- 编辑产品详细信息
- 将现有产品分配到其他分类（一个产品可以属于多个分类），或将其移至另一个分类
- 查看与某个部门关联的分类
- 从分类中移除产品
- 从目录中删除产品

为了确保敏感页面（例如网站的管理部分）的安全，您还需要执行以下操作：

- 实现一个登录表单，管理员需要提供用户名和密码。
- 学习如何使用 SSL 保护登录表单和管理页面。
