# 连接数据库

连接数据库可能需要花费一些时间，如果所有或大多数对 Web 服务器的请求都使用同一个数据库，那么保持一个连接池并反复重用这些连接是合理的。

### 解决方案

各种数据库扩展中的`持久连接`功能已经存在很长时间了。尽管持久连接可以节省一些时间，但也可能引发问题。如果使用持久连接锁定了某个表，而脚本无法释放该锁，那么该表可能会一直保持锁定状态，直到手动终止连接（重启 Web 服务器，可能还需要重启数据库服务器），因此需谨慎使用。

在较旧的数据库驱动中，曾使用专门的连接函数（例如用`mysql_pconnect()`代替`mysql_connect()`），但在`mysqli`驱动中，持久连接是通过在主机名前添加前缀`'p:'`来创建的。

允许的持久连接数量可在`php.ini`中配置，并且应匹配网站的同时请求数。如果该数值设置过低，某些用户可能会遇到连接问题。

当 Web 服务器关闭或重启时，持久连接会被关闭。如果持久连接是由 CLI 模式下运行的脚本创建的，那么当脚本终止时连接也会关闭。在这种情况下，持久连接和非持久连接没有区别。

### 工作原理

以下示例展示了如何使用主机名的`'p:'`前缀来创建持久连接。

```php
<?php
// 16_5.php

$mysqli = new mysqli('p:127.0.0.1', 'php', 'secret', 'music');

if ($mysqli->connect_error) {
    die("Error connecting: {$mysqli->connect_errno} {$mysqli->connect_error}\n");
}
else {
    echo "Connected to music\n";
}
```

不要在持久链接上使用`close()`函数，因为这会将其从内存中移除。

可以使用`phpinfo()`函数来显示`mysqli`连接的状态。`Active Persistent Links` 是当前正在使用的链接数。`Inactive Persistent Links` 是当前未使用但可用的链接数，而`Active Links` 是当前正在使用的非持久链接数。
