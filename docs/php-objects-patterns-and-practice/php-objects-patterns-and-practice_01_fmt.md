# 引言

当我最初构思这本书时，PHP 中的面向对象设计还是一个深奥的话题。过去的这些年不仅见证了 PHP 作为面向对象语言的不可阻挡的崛起，也见证了框架的蓬勃发展。当然，框架非常有用。它们管理着众多（或许在当今，是大多数）Web 应用程序的核心与粘合剂。更重要的是，它们通常恰好体现了本书所探讨的设计原则。

然而，对于开发者而言，这里存在一个危险，就像所有有用的 API 一样。这种危险在于，人们可能发现自己被降级到用户层，不得不等待远方的专家凭个人喜好修复 bug 或添加功能。从这个立场出发，很容易陷入一种流放状态：人们将框架的内部机制视为高级魔法，而将自己的工作视为不过是架设在强大而未知的底层基础设施之上的小小装饰。

尽管我是一个积习难改的重复造轮子者，但我的论点主旨并非我们都应该抛弃框架，从头构建 MVC 应用程序（至少不总是如此）。相反，我认为作为开发者，我们应当理解框架所解决的问题，以及它们用来解决问题的策略。我们不仅应该从功能上评估框架，还应该从创建者所做的设计决策角度去评估，并判断其实现质量。当然，当条件合适时，我们也应该继续前进，构建我们自己精简而专注的应用程序，并随着时间的推移，编写我们自己的可重用代码库。

我希望本书能在帮助 PHP 开发者将面向设计的见解应用到他们的平台和库方面起到一些作用，并为他们在需要独立前行时提供一些概念性工具。
