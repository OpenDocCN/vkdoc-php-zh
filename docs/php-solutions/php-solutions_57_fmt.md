# 15. 从多个表中提取数据

正如我在第 11 章中所述，关系型数据库的主要优势之一是能够通过使用一个表中的主键作为另一个表中的外键来链接不同表中的数据。`phpsols` 数据库有两个表：`images` 和 `blog`。现在是时候添加更多表并将它们关联起来，以便你可以为博客文章分配类别，并将图像与单个文章关联。

你并不是物理上地连接多个表，而是通过 SQL 来实现。通常，你可以通过识别主键和外键之间的直接关系来连接表。然而，在某些情况下，关系更为复杂，需要通过第三个表来作为其他两个表之间的交叉引用。

在本章中，你将学习如何建立表之间的关系，以及如何将一个表的主键作为外键插入到另一个表中。虽然概念上听起来很困难，但实际上相当简单——你使用一个数据库查询来查找第一个表中的主键，保存结果，然后在另一个查询中使用该结果将其插入到第二个表中。

具体来说，你将学习以下内容：

- 理解不同类型的表关系
- 使用交叉引用表处理多对多关系
- 修改表结构以添加新列或索引
- 将主键作为外键存储在另一个表中
- 使用 `INNER JOIN` 和 `LEFT JOIN` 连接表
