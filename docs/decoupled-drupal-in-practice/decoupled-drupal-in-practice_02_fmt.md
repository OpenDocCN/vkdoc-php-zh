# 引言

在内容管理的历史上，从未有过如此令人眼花缭乱的设备扩散，以及我们遇到和消费的数字体验的增长。长期以来作为组织数字形象最关键元素的网站，正日益被视为内容渠道和形态万花筒中的单一侧面。当今的许多用户在获取内容或数据的过程中，甚至从未接触过传统的 Web 浏览器。

用户每天都在与数量惊人的不同客户端（也称为消费者）进行交互：智能手机上的原生移动应用程序、个人电脑上的原生桌面应用程序、电视上的机顶盒、聊天应用程序、对话式界面和物联网设备。如今，这些消费者倾向于与一个同时充当内容仓库或数据存储的单一服务器进行交互。

这种现象颠覆了整个行业的架构范式和方法。传统上，网站（尤其是大型网站）通过内容管理系统进行管理，该软件支持内容的创建、操作及其格式化和布局。许多长期专注于网站管理的内容管理系统，对于当前内容渠道的持续爆炸性增长准备不足。

Drupal 是由 Dries Buytaert 创建的免费开源内容管理系统，是这一趋势中的一个显著例外。近年来，Drupal 正在重塑自己，不仅作为传统网站的端到端系统，还用于与摄取数据的客户端和其他服务器进行通信，以及两者紧密结合的混合体。这为当今最常用的内容管理框架之一预示着激动人心的新可能性。

市场上的其他内容管理系统在新的数字格局中存在某些劣势。WordPress 虽然提供了名为 WP-API 的 RESTful API，但缺乏像 Drupal 那样灵活的内容模型，并且采用了自定义规范，这给构建数据消费者的开发者带来了学习曲线。与此同时，Contentful 和 Prismic 等专有替代方案虽然市场份额增长迅速，但对于那些偏好从头到尾使用开源的人来说并不理想。

在本书中，我们将从最微小的细节（例如如何发出请求以获得消费者期望的响应）到更宏观的图景（包括解耦式 Drupal 对 Drupal 乃至整个 CMS 架构未来的影响）来审视解耦式 Drupal 的概念和思想。到本书结束时，你将深入理解 Drupal 的 API 及其消费方式，并成功使用多种技术构建简单的内容应用程序。

第 1 部分“解耦式 Drupal 基础”概述了单体 CMS 架构如何日益演变为解耦式或“无头”CMS 架构，以满足当今多渠道世界的需求。前三章将 CMS 的历史与网页的演变进行类比，概述了解耦式 CMS 架构是如何因打破架构和技术壁垒的需求而出现的。然后，我们将尽可能全面地定义解耦式 Drupal，评估其最常见的架构方法，并列举解耦 Drupal 的风险与回报。对于已经熟悉 RESTful 方法和解耦 CMS 架构的读者，可以安全地跳过第 1 部分。

第 2 部分“解耦 Drupal”详细审视了 Drupal 作为内容服务、Web 服务提供者和内容仓库的能力。在这些章节中，我们将研究 Drupal 8 中可用的 Web 服务解决方案，包括作为 Web Services and Context Core Initiative 成果的 Drupal 核心部分，以及贡献模块。我们将详细描述如何将 Drupal 配置为 RESTful 数据服务，以及可用的认证机制。

第 3 部分“消费与操作 Drupal 8”介绍了如何弥合 Drupal 与其 API 消费者之间的鸿沟，这通常通过`XMLHttpRequests` (XHR) 发生。这些章节涉及如何形成请求，以使用核心 REST 以及 JSON API、RELAXed Web Services 和 GraphQL 等贡献解决方案，在 Drupal 服务器端创建、读取、更新和删除数据。

第 4 部分“解耦式 Drupal 生态系统”涉及围绕 Drupal 新兴的生态系统，涉及 API 优先的分发版（这有助于 Drupal 新手），以及软件开发工具包（SDK）和参考应用程序，它们为开发者提供了构建基础，弥合了 Drupal 与消费技术之间的鸿沟。在此过程中，我们将介绍 Contenta、Reservoir、Headless Lightning 和 Waterwheel 生态系统等项目。

第 5 部分“与消费者集成”探讨了 Drupal 驱动（即消费或操作来自 Drupal 的数据）的应用程序本身的开发，而不仅仅是支撑它们的请求和响应。这些章节使用各种广泛使用的技术，解释了如何将解耦式 Drupal 与基于 JavaScript 的消费者集成。

第 6 部分“解耦式 Drupal 的高级主题”涉及解耦式 Drupal 在生产环境中的问题，例如 REST 插件系统、自文档化 API、代码生成和缓存。此外，这些章节还从批判性角度评估了关于解耦式 Drupal、内容管理和解耦式 CMS 未来的问题。

本书揭示了作为即将迎来黄金时期的扩展范式的解耦式 Drupal，同时也回应了围绕新兴方法提供更清晰度、标准化和最佳实践的日益增长的需求。解耦式 Drupal 为用户体验和数字生态系统带来了激动人心的新进展，但风险和缺点也限制了其在所有用例中的适用性。解耦式 Drupal 横跨内容管理的两个截然不同的时代，不仅是我们概念化和交付内容的转折点，也是我们设想自己与这些体验之间关系的转折点。
