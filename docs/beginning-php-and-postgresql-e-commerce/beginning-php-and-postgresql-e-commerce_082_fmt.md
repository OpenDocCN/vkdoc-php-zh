# 另一个与安全相关的问题涉及存储用户密码

在数据库中明文存储用户密码不是一个好主意，因为这类信息是潜在的攻击目标。相反，你应该存储所谓的密码**哈希**（`hash`）。哈希是一个表示密码的唯一字符串，但无法被转换回原始密码。要验证用户输入的密码，你只需为输入的密码生成一个哈希，并将其与存储在数据库中的哈希进行比较。如果哈希匹配，则输入的密码也匹配，因此你可以确信客户是真实的。

这引出了另一个重要任务——你需要为新用户提供注册方式。

注册的结果是在数据库中添加一个新客户，包括用户名和密码哈希信息。

要在你的应用程序中实现此方案，你需要完成以下任务：

- 创建两个新的数据库表：第一个名为`customer`，用于保存客户详情；第二个名为`shipping_region`，用于存储客户可以居住的可能配送区域。

- 在数据层和业务层中实现相关方法，用于添加、修改和检索`customer`和`shipping_region`中的信息。

- 修改`cart_details`组件化模板，该模板现在会将用户重定向到一个结账页面，该页面将在名为`checkout_info`的新组件化模板中实现。

- 创建一个名为`customer_login`的客户登录组件化模板。

- 创建一个名为`customer_details`的组件化模板，用于客户注册或编辑基本账户详情。

- 创建一个名为`customer_credit_card`的组件化模板，允许客户输入信用卡详细信息。

- 创建一个名为`customer_address`的组件化模板，供客户输入配送地址。
