# 安全与错误消息

在使用 PHP 和数据库开发网站时，显示错误消息至关重要，以便在出现问题时调试代码。然而，原始错误消息在正式运行的网站中显得不够专业，还可能向潜在攻击者透露数据库结构的相关信息。因此，在将脚本部署到互联网之前，你应该用自定义的中性消息替换数据库生成的错误消息，例如“抱歉，数据库暂时不可用。”

## 章节回顾

通过数据库进行内容管理涉及插入、选择、更新和删除记录。每条记录的主键在更新和删除过程中起着至关重要的作用。大多数情况下，当记录首次创建时，主键的生成由数据库自动处理。此后，查找记录的主键只需通过 `SELECT` 查询即可实现，无论是通过显示所有记录列表，还是通过搜索你知道的关于该记录的信息（如标题或文章中的词语）。

`MySQLi` 和 `PDO` 预处理语句通过消除确保引号和控制字符正确转义的需求，使数据库查询更加安全。如果在脚本运行过程中需要使用不同变量重复执行同一查询，它们还能加速应用程序的运行。脚本只需用占位符验证一次 SQL 语句，无需每次都进行验证。

尽管本章重点介绍内容管理，但相同的基本技术适用于大多数与数据库的交互。当然，SQL 和 PHP 的内容远不止于此。在下一章中，我将讨论一些最常见的问题，例如仅显示长文本字段的第一句左右内容以及处理日期。然后在第 15 章中，我们将探索如何操作数据库中的多个表。
