# for 循环语句

`for` 循环语句具有以下语法结构。

```
for(第一个代码块;第二个代码块 (条件);第三个代码块){
# 命令
}
```

`for` 关键字之后紧跟三个放在括号中的代码块。代码块之间用分号分隔。如果某个代码块包含多条命令，则用逗号分隔。右括号之后，在花括号中紧随另一个代码块。这些命令构成了 `for` 循环语句的主体。

![](img/613529_1_En_3_Figi_HTML.jpg)一个信息图标。备注

如果语句主体中的命令块只包含一条命令，则可以省略花括号。不过，最好还是保留花括号。

我们来讨论一下 `for` 循环语句是如何运行的。在最开始，会执行第一个代码块中的命令。这些命令仅执行一次，且只执行一次。然后，会检查第二个代码块中的条件。如果条件为真，则执行循环语句主体中的命令以及第三个代码块中的命令。之后，再次检查该条件。如果条件为真，则再次执行循环语句主体中的命令以及第三个代码块中的命令，随后再次检查条件。以此类推。当在后续某次测试中发现条件为假时，循环语句的执行即终止。

还有一种调用 `for` 循环语句的替代方式。这种方式不使用花括号。取而代之的是，在 `for` 指令之后（在语句主体的命令块之前）放置一个冒号。并且整个语句以 `endif` 指令（以及一个分号）结束。

```
for(第一个代码块;第二个代码块 (条件);第三个代码块):
# 命令
endif;
```

此形式的语句执行方式与前述相同。

## `switch` 选择语句

`switch` 语句允许你根据某个表达式的值，执行某一个命令块。在某种意义上，这条语句类似于嵌套的条件语句。但是，在条件语句中，检查的是条件（一个布尔类型值或可转换为布尔类型值的表达式），而在选择语句中，检查的是表达式的值，其类型可能不同于布尔类型。

选择语句的调用语法如下。

```
switch(expression){
case value:
# 命令
break;
case value:
# 命令
break;
default:
# 命令
}
```

`switch` 关键字后面跟着一个括号内的表达式，其值将被检查。然后，跟着 `case` 块。在上面的模板中有两个 `case` 块，但通常，这种块的数量几乎没有限制。最后一个可以是 `default` 块（它是可选的）。整个结构用花括号括起来。

每个 `case` 块在 `case` 关键字之后包含一个控制值。其后跟一个冒号和一个命令块，该命令块通常以 `break` 指令结束。在 `default` 块中，如果存在，则不使用 `break` 指令（在那里使用它没有任何意义）。

选择语句是如何执行的？首先计算 `switch` 关键字后括号内的表达式。然后将该表达式的值与 `case` 块中的控制值依次比较，直到找到第一个匹配项。

| 详细信息 |
| --- |
| 这是普通的比较，而不是检查全等。 |

如果找到匹配项，则执行相应 `case` 块中的命令。这是在命令块以 `break` 指令结束时的情况。如果 `case` 块末尾没有 `break` 指令，则会执行下一个块中的命令，然后是它后面的块，依此类推。此过程会一直持续，直到遇到 `break` 指令或到达选择语句的右花括号。

选择语句有一种替代语法。

```
switch(expression):
case value:
# 命令
break;
case value:
# 命令
break;
default:
# 命令
endswitch;
```

这里不使用花括号：用冒号代替左花括号，用 `endswitch` 语句（后跟分号）代替右花括号。

## `goto` 指令

PHP 提供了`goto`指令，可用于跳转到程序中用标签标记的特定位置。坦白说，使用`goto`指令通常被认为是不太好的编程风格。不过，既然存在这种可能性，了解它总归是有益的。

因此，你可以在程序中放置一个标签。它是一个标识符，用于确定程序中的跳转点。声明标签很简单：指定标签名称后跟一个冒号。要跳转到程序中的该位置，必须调用`goto`语句，后跟标签（该标签确定要跳转的程序位置）。清单 3-12 展示了一个程序，它使用条件语句、标签和`goto`指令模拟了循环语句。结果是在控制台中显示一个数字序列。

## `match` 选择语句

PHP 8 引入了另一种选择语句，即`match`语句。从某种意义上说，`match`语句类似于`switch`语句和三元运算符的组合。与`switch`语句一样，`match`语句允许你检查表达式。但与`switch`语句不同的是，`match`语句会返回一个结果（类似于三元运算符）。下面展示了使用`match`语句的通用语法。

```
$variable=match(expression){
value=>result,
value=>result,
value=>result,
...
$value=>result,
default=>result
};
```

在该语句的最开始，放置`match`关键字，后跟需要检查值的表达式（放在括号中）。下一个代码块用花括号括起来。如前所述，`match`语句会返回一个结果，因此整个表达式会被赋值给某个变量。在语句的末尾，需要加上分号。

在`match`语句的主体中，在花括号内放置形式为`value=>result`的代码块。这些元素用逗号分隔。最后一个可以是带有`default`关键字的`default=>result`代码块，但这是可选的。

让我们讨论一下`match`语句是如何执行的。首先，计算`match`关键字后括号内表达式的值。然后，将该值依次与`match`语句主体中指定的值进行比较（直到找到第一个匹配项）。如果找到匹配项，则返回对应`=>`箭头后面的结果。如果没有找到匹配项，则返回带有`default`关键字的代码块的结果。

| 详细信息 |
| --- |
| `match`选择语句必须返回一个结果。如果执行语句时发现没有任何代码块生效（未找到匹配项且没有`default`代码块），则会发生错误。比较操作会持续到找到第一个匹配项为止。如果找到匹配项，则不再检查其余值。 |

如果被测试表达式的多个值应返回相同的结果，那么这些值可以收集在同一个代码块中（用逗号分隔）。让我们更详细地看一下这种情况。

| 条件 `$num<=3`、`$num<8` 和 `$num>=5` 的设计使得至少其中一个为 `true`。因此，对于变量 `$num` 的任何值，选择语句都能返回一个结果。另外值得一提的是，对于变量 `$num` 的值 `5`、`6` 和 `7`，条件 `$num<8` 和 `$num>=5` 同时为 `true`。但条件 `$num<8` 在条件 `$num>=5` 之前被检查。因此，如果发现条件 `$num<8` 为 `true`，则不会检查条件 `$num>=5`。 |
