### LDAP 简介

LDAP 是当今访问目录服务器的事实标准，它提供了一种用于存储、检索、操作和保护目录数据的明确模型。或许对 LDAP 的最佳描述出现在 IBM 的《LDAP 红皮书》（http://www.redbooks.ibm.com/redbooks/SG244986/）中，该书将 LDAP 描述为一种由四个关键模型组成的协议：



• **信息：** 正如关系型数据库定义了表中数据必须遵循的列属性，LDAP 也定义了目录服务器中存储信息的结构。

• **命名：** LDAP 提供了一种明确定义的结构，用于确定如何导航、标识和检索 LDAP 信息。这种结构被称为通用目录结构或模式（schema），它非常类似于常用于组织信息的层级模型。此类实体的例子包括动植物分类法、公司组织层级（类似于图 17-1 所示的结构）、同义词词典和家谱。

• **功能：** LDAP 定义了可以对目录服务器中存储的信息执行的操作，规定了如何检索、插入、更新和删除数据。此外，LDAP 还定义了 LDAP 客户端与服务器之间通信所使用的格式和传输方法。

• **安全性：** LDAP 提供了一种方案，用于确定如何以及由谁来访问 LDAP 目录中存储的信息。它提供了多种访问级别，包括读取、插入、更新、删除和管理等访问权限级别。此外，LDAPv3 的传输层安全（TLS）扩展通过使用加密技术，提供了一种在客户端和服务器之间进行身份验证和数据传输的安全手段。

从上述总结中你可能已经推断出，LDAP 既定义了信息存储方式，也定义了通信方法。LDAP 在实现方面几乎不留想象空间，这也是它被广泛使用的原因之一。

#### 连接到 LDAP 服务器

使用 LDAP 与使用数据库服务器非常相似，因为在进行任何交互之前，必须先建立与服务器的连接。PHP 的 LDAP 服务器连接函数称为 `ldap_connect()`。

`ldap_connect()`

`resource ldap_connect ([string *hostname* [, int *port*]])` `ldap_connect()` 函数建立到由 `hostname` 指定的 LDAP 服务器的连接，端口号为 `port`。如果未指定可选的 `port` 参数，并且服务器名称前带有 `ldap://` URL 方案，或者完全省略了该 URL 方案，则假定使用 LDAP 的标准端口 `389`。如果使用 `ldaps://` 方案，则假定使用端口 `636`。如果连接成功，则返回一个链接标识符；如果出错，则返回 `FALSE`。一个简单的使用示例如下：

```php
<?php

$ldapHost = "ldap://ad.example.com";

$ldapPort = "389";

$ldapconn = ldap_connect($ldapHost, $ldapPort)

or die("Can't establish LDAP connection");

?>
```
