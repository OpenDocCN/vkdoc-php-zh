# 设置 phpsols 数据库

在本地测试环境中，你可以在 MySQL 中创建的数据库数量没有限制，并且可以随意命名它们。我假设你正在本地测试环境中工作，因此将向你展示如何创建一个名为 `phpsols` 的数据库，以及两个分别名为 `psread` 和 `pswrite` 的用户账户。

#### 注意

在共享托管环境中，你可能仅限于使用托管公司设置的一个数据库。如果你在远程服务器上进行测试，并且无法自由创建新的数据库和用户账户，请将托管公司分配给你的数据库名称和用户名分别替换 `phpsols` 和 `pswrite`。

## MySQL 命名规则

MySQL 对于数据库、表和列的基本命名规则如下：

- 名称长度最多为 64 个字符。

- 合法字符包括数字、字母、下划线和 `$`。

- 名称可以以数字开头，但不能完全由数字组成。

一些托管公司似乎对这些规则一无所知，会分配给客户端名称中包含一个或多个连字符（非法字符）的数据库。如果数据库、表或列的名称包含空格或非法字符，在 SQL 查询中必须始终用反引号（`` ` ``）将其括起来。请注意，这不是单引号（`'`），而是一个单独的字符。在我的 Windows 键盘上，它直接位于 Tab 键上方。在我的 Mac 键盘上，它位于左 Shift 键旁边，与波浪号（`~`）在同一个键上。

在选择名称时，你可能会不小心选到 MySQL 众多保留字中的一个（<https://dev.mysql.com/doc/refman/8.0/en/keywords.html>），例如 `date` 或 `time`。避免这种情况的一种技巧是使用复合词，例如 `arrival_date`、`arrival_time` 等。或者，用反引号将所有名称括起来。phpMyAdmin 会自动执行此操作，但在 PHP 脚本中编写自己的 SQL 时，你需要手动执行此操作。

#### 注意

由于很多人使用过 `date`、`text`、`time` 和 `timestamp` 作为列名，MySQL 允许在不使用反引号的情况下使用它们。但是，你应该避免使用它们。这是一种不良实践，如果将来将数据迁移到不同的数据库系统，可能无法正常工作。

### 名称的大小写敏感性

Windows 和 macOS 将 MySQL 名称视为不区分大小写。然而，Linux 和 Unix 服务器则区分大小写。为了避免在将数据库和 PHP 代码从本地计算机传输到远程服务器时出现问题，我强烈建议你仅在数据库、表和列名称中使用小写字母。当使用多个单词构建名称时，请用下划线连接它们。
