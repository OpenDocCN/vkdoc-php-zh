## PHP 与 Web 表单

虽然使用超链接作为交互手段确实很有用，但通常你需要一种允许用户实际向应用程序输入原始数据的方法。例如，如果你想允许用户输入姓名和电子邮件地址以订阅新闻通讯该怎么办？当然，你会使用表单。因为你肯定非常清楚什么是 Web 表单，并且毫无疑问已经作为最终用户使用过数百次甚至数千次 Web 表单，所以本章不会介绍表单语法。如果你需要一个关于如何创建基本表单的入门或复习课程，可以考虑查阅 Web 上提供的许多教程。以下两个提供特定表单教程的特别有用的网站：

-   **W3 Schools**：http://www.w3schools.com/
-   **HTML Goodies**：http://www.htmlgoodies.com/

相反，我们将回顾如何将 Web 表单与 PHP 结合使用来收集和处理有价值的用户数据。

将数据从一个脚本传递到另一个脚本有两种常见方法：`GET` 和 `POST`。虽然 `GET` 是默认方法，但你通常希望使用 `POST`，因为它能够处理更多的数据，这在您使用表单插入和修改大块文本时是一种重要的行为。如果你使用 `POST`，发送到 PHP 脚本的任何已发布数据都必须使用 `$_POST` 语法进行引用，正如第 3 章首次介绍的那样。例如，假设表单包含一个名为 `email` 的文本字段值，如下所示：

```html
<input type="text" name="email" size="20" maxlength="40" value="" />
```

一旦此表单被提交，你可以像这样引用该文本字段值：`$_POST['email']`

当然，为了方便起见，没有什么能阻止你先将此值分配给另一个变量，如下所示：

```php
$email = $_POST['email'];
```

请记住，除了奇怪的命名约定外，`$_POST` 变量与其他任何变量一样。它们只是以这种方式被引用，以明确地将外部变量的来源进行分区。如你在第 3 章所学，这种约定适用于来自 `GET` 方法、Cookie、会话、服务器和上传文件的变量。可以将其视为变量的命名空间。

本节介绍了许多场景，其中 PHP 不仅在管理表单数据方面，而且在实际创建表单本身方面都可以发挥非常有效的作用。不过，首先让我们看一个概念验证示例。

### 一个简单的示例

以下脚本渲染一个表单，提示用户输入他们的姓名和电子邮件地址。一旦完成并提交，该脚本（名为 `subscribe.php`）将在浏览器窗口中显示此信息。

```php
<?php

// If the submit button has been pressed

if (isset($_POST['submit']))

{

echo "Hi ".$_POST['name']."!<br />";

echo "The address ".$_POST['email']." will soon be a spam-magnet!<br />";

}

?>

<form action="subscribe.php" method="post">

<p>

Name:<br />

<input type="text" name="name" size="20" maxlength="40" value="" />

</p>

<p>

Email Address:<br />

<input type="text" name="email" size="20" maxlength="40" value="" />

</p>

<input type="submit" name = "submit" value="Go!" />

</form>
```

假设用户填写了两个字段并点击了 `Go!` 按钮，将显示类似于以下的输出：

```
Hi Bill!
The address bill@example.com will soon be a spam-magnet!
```

请注意，在此示例中，表单引用的是它所在的脚本，而不是另一个脚本。虽然两种做法都经常使用，但引用原始文档并使用条件逻辑来确定应执行哪些操作是非常常见的。在这种情况下，条件逻辑规定，只有在用户提交（发布）了表单时，`echo` 语句才会执行。

同样值得注意的是，在将数据发回其来源的同一脚本的情况下，如前面的示例所示，你可以使用 PHP 超级全局变量 `$_SERVER['PHP_SELF']`。执行脚本的名称会自动分配给此变量；因此，使用它代替实际文件名将节省一些额外的代码修改，以防文件名以后更改。例如，前面示例中的 `<form>` 标签可以修改如下，并仍然产生相同的结果：

```php
<form action="<?php echo $_SERVER['PHP_SELF']; ?>" method="post">
```
