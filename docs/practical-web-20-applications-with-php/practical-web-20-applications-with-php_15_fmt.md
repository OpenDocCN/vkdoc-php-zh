## 第 3 章：用户认证、授权与管理

在第 2 章中，我们了解了模型-视图-控制器设计模式，该模式使我们能够轻松地将应用程序逻辑与显示逻辑分离，并且我们使用`Zend_Controller_Front`实现了它。现在，我们将扩展应用程序控制器以处理用户认证、用户授权和用户管理。

在这一阶段，您可能会想知道*认证*（Authentication）和*授权*（Authorization）之间的区别。

- **认证：** 确定用户是否确实为其所声称的身份。这通常通过使用唯一的用户名（他们的*身份*）和密码（他们的*凭证*）来执行。

- **授权：** 确定用户是否被允许访问特定的*资源*，前提是我们已通过认证过程知道他们是谁。授权还决定了未经身份验证的用户可以执行哪些操作。在我们的应用程序中，资源本质上是一个特定的操作或页面，例如提交新博客文章的操作。

在本章中，我们将使用 Zend Framework 的`Zend_Auth`组件在应用程序中设置用户认证。这包括创建用于存储用户详细信息的数据库表。然后，我们将使用`Zend_Acl`组件来管理应用程序中每个用户可以访问的资源。此外，我们必须将权限系统与`Zend_Controller_Front`集成起来。

### 创建用户数据库表

由于我们的应用程序将为多个用户维护用户账户，我们需要跟踪这些用户账户。为此，我们将创建一个名为`users`的数据库表。该表将为每个用户包含一条记录，并存储其用户名和密码，以及其他重要详细信息。

访问我们 Web 应用程序的用户将分为三类：*访客*、*会员*和*管理员*。访问应用程序的用户在登录为会员之前将被自动归类为访客。为了区分会员和管理员，`users`表将包含一个用于表示每个用户*角色*的列。在实现基于`Zend_Acl`的访问控制列表时，我们将使用此列。

**注意** 在更复杂的系统中，你可能会为用户分配多个角色；然而，为简单起见，我们只允许每个用户拥有一个角色。任何被归类为管理员的用户也将能够执行普通成员的所有功能。此外，你也可以使用另一个表来存储用户类型，但同样为了简化，我们将放弃这种做法，而在代码中维护一个静态的用户类型列表。

我们在`users`表中为每个用户存储的核心数据如下：

- `user_id`：用于表示用户的内部整数。

- `username`：用于登录的唯一字符串。实际上，这将是用户的公共标识符。我们会在博客文章和其他公开内容中显示`username`，而不是他们的真实姓名，因为许多用户希望保持匿名。

- `password`：用于验证用户身份的字符串。我们将使用`md5()`函数以哈希形式存储密码。请注意，这意味着密码无法被找回，必须重新设置。我们将实现所有必要的代码来完成此操作。

- `user_type`：用于对用户进行分类的字符串（可以是`admin`或`member`，尽管根据你从本书学到的内容，未来可以轻松添加其他用户类型）。

- `ts_created`：表示用户账户创建时间的时间戳。

- `ts_last_login`：表示用户最后登录时间的时间戳。我们允许此字段具有`null`值，因为在创建记录时用户尚未登录。

清单 3-1 展示了在 MySQL 中创建`users`表所需的 SQL 命令。所有 SQL 模式定义都存储在主应用目录的`schema-mysql.sql`文件中。如果你使用的是 PostgreSQL，可以在`schema-pgsql.sql`中找到相应的模式。

**注意** 选择如何为你的 Web 应用存储数据库模式完全由你决定。我这样组织只是为了让你在需要时能轻松查阅（并且在下载本书代码时也能方便地获取）。

**清单 3-1.** *用于在 MySQL 中创建用户表的 SQL (`schema-mysql.sql`)*

```
create table users (
  user_id serial not null,
  username varchar(255) not null,
  password varchar(32) not null,
  user_type varchar(20) not null,
  ts_created datetime not null,
  ts_last_login datetime,
  primary key (user_id),
  unique (username)
) type = InnoDB;
```

`user_id`列被定义为`serial`类型，这与使用`bigint unsigned not null auto_increment`等效。我个人更喜欢使用`serial`，因为它更短、更易输入，并且在 PostgreSQL 中也能正常工作。

`username`列最大长度为 255 个字符，但我们会在代码中对此长度施加限制。`password`将以 MD5 加密字符串的形式存储，因此该列只需要 32 个字符长度。

接下来是`user_type`列。该列的长度并不太重要，尽管你添加的任何新用户类型将被限制在 20 个字符以内（这只是一个内部名称，因此不需要过于描述性）。此字符串用于执行 ACL 检查。

最后是两个时间戳列。MySQL 实际上有一种名为`timestamp`的数据类型，但我选择了`datetime`类型，因为 MySQL 会自动更新使用`timestamp`类型的列。在 PostgreSQL 中，你需要改用`timestamptz`数据类型（参见`schema-pgsql.sql`文件中的表定义）。下面的“时间戳”部分提供了关于时间戳在 PHP 中如何工作的更多细节。

**提示** 清单 3-1 指示 MySQL 在创建表时使用`InnoDB`表类型，从而为我们提供 SQL 事务能力并强制外键约束。否则使用的默认表类型是`MyISAM`。

现在你必须在数据库中创建此表。有两种方法可以实现。首先，你可以使用以下命令将整个`schema-mysql.sql`文件通过管道导入数据库：

```
# mysql -u phpweb20 -p phpweb20 < schema-mysql.sql
```

当你输入此命令时，系统会提示你输入密码。这将从头开始创建整个数据库。
